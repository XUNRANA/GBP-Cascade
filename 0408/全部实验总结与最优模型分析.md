> 涵盖 0402 / 0406 / 0408 全部 19 组实验，详细解析最优模型 Exp#18 从数据输入到最终输出的完整流程。

## 一、任务背景

- **任务**：胆囊息肉良恶性二分类（benign=良性肿瘤, no_tumor=非肿瘤性息肉）
- **数据集**：训练集 1229 张（benign=309, no_tumor=920），测试集 523 张（benign=129, no_tumor=394）
- **类别不平衡**：约 3:1（no_tumor : benign）
- **数据来源**：超声图像（LabelMe标注多边形mask）+ 临床Excel表（年龄/性别/尺寸等）+ JSON特征字典（超声报告文本text_bert + 形态学特征）
- **评估指标**：F1 macro（兼顾两类），辅以分割指标 Lesion IoU/Dice

## 三、分阶段进展分析

### 第一阶段：0402 基础架构探索 (Exp#1-10)

#### 关键突破：Exp#4 — SegGuided Attention + Metadata

**从 Exp#1→4 的演进路径**：
1. **Exp#1** (F1=0.6215)：3通道RGB + UNet分割 + 分类头，奠定联合训练框架
2. **Exp#2** (F1=0.6083)：尝试多尺度分类 + 输入级Mixup → **失败**，Mixup在混合样本时破坏了分割mask的语义
3. **Exp#3** (F1=0.6161)：引入4通道输入(RGB+LesionMask)，但用GAP池化丢失了空间信息
4. **Exp#4** (F1=0.6741)：**两个关键创新** — (a) Seg-Guided Attention 用分割概率图加权空间特征池化；(b) 6D结构化元数据(age, gender, size_mm, size_bin, flow_bin, morph_bin) → **F1跃升5.3%**

**后续尝试均未超越 Exp#4**：
- **Exp#5/6** (F1=0.6241/0.6253)：MaxViT-Tiny@320 替换SwinV2 → 效果下降，说明SwinV2的窗口注意力更适合医学图像局部细节捕捉
- **Exp#7** (F1=0.6489)：加强正则化(更大dropout+EMA) → 略降，EMA的慢更新对少数类学习不利
- **Exp#8** (F1=0.6417)：双尺度SegAttn(f2+f3) → 额外尺度引入噪声而非互补信息
- **Exp#9** (F1=0.6333)：SupCon对比学习 → batch_size=8 太小，负样本不足以学好对比空间
- **Exp#10** (F1=0.6454)：3-seed同构集成 → 同架构同超参模型高度相关，集成无法修正系统性错误

### 第二阶段：0406 正则化与特征扩展 (Exp#11-14)

#### 教训：EMA 是致命陷阱

- **Exp#11** (F1=0.6554)：扩展至18D形态特征，比基线略低，但验证了更丰富元数据有增益潜力
- **Exp#12** (F1=0.4865)：**灾难性失败** — EMA decay=0.9995 导致benign类完全学不到（Val F1始终0.4272），EMA的平滑效应抹平了少数类的梯度信号
- **Exp#13** (F1=0.6422)：特征级Mixup替代输入级 → 避免了mask破坏问题，但增益有限
- **Exp#14** (F1=0.5023)：**灾难性失败** — 三重问题叠加：(a) 图像级而非患者级KFold导致数据泄漏 (b) EMA仍在使用 (c) 18D高维特征 → 各折Val F1均低于0.40

**关键教训**：
1. EMA(0.9995) 在类别极度不平衡场景下必须避免
2. 交叉验证必须按患者ID分组，否则同一患者的多张图像分到train和val造成泄漏

### 第三阶段：0408 多模态融合 (Exp#15-19)

#### 最优方案：Exp#18 全模态门控融合

- **Exp#15** (F1=0.6506)：6D→10D临床特征，单独增加反而降分（wall_thickness覆盖率仅55.2%，缺失值噪声）
- **Exp#16** (F1=0.6433)：BERT[CLS]后期拼接 → **最差方案**，冻结BERT的通用语义与图像特征不对齐
- **Exp#17** (F1=0.6639)：BERT全token交叉注意力 → 接近基线，验证了空间级文本-图像交互的价值
- **Exp#18** (F1=0.6846)：**整合所有有效组件 + 门控自适应融合 → 唯一超越基线的实验**
- **Exp#19** (val F1=0.6772±0.0220)：患者级5折CV诚实评估，确认模型真实性能

#### Stage 2：图像编码 — SwinV2-Tiny Encoder

```
input_4ch (B, 4, 256, 256)
        │
        ▼  adapt_model_to_4ch: 修改第一层Conv权重, 3ch→4ch
┌─────────────────────────────────────────────┐
│           SwinV2-Tiny Encoder               │
│  (Shifted Window Multi-Head Self-Attention) │
│                                             │
│  Stage 0: Patch Embed + 2×STB              │
│     f0: (B, 96, 64, 64)                    │
│                                             │
│  Stage 1: Patch Merge + 2×STB              │
│     f1: (B, 192, 32, 32)                   │
│                                             │
│  Stage 2: Patch Merge + 6×STB              │
│     f2: (B, 384, 16, 16)  ← 主要分类特征    │
│                                             │
│  Stage 3: Patch Merge + 2×STB              │
│     f3: (B, 768, 8, 8)   ← 最深语义特征     │
└───────┬────────┬────────┬────────┬──────────┘
        f0       f1       f2       f3
```

- STB = Swin Transformer Block (窗口大小 8×8)
- 4通道适配: 将预训练的3ch卷积核复制第一通道权重到第4通道
- 总下采样率: Stage0=4×, Stage1=8×, Stage2=16×, Stage3=32×

---

#### Stage 3：UNet 分割解码器

```
f3 (B, 768, 8, 8)  ──┐
                      ▼
              UNetDecoderBlock(768+384→384)  ← f2 skip connection
                      │
               d3 (B, 384, 16, 16)
                      │
              UNetDecoderBlock(384+192→192)  ← f1 skip connection
                      │
               d2 (B, 192, 32, 32)
                      │
              UNetDecoderBlock(192+96→96)    ← f0 skip connection
                      │
               d1 (B, 96, 64, 64)
                      │
              ConvTranspose2d(96→48, stride=4) → GroupNorm → GELU
              Conv2d(48→48, 3×3) → GroupNorm → GELU
              Conv2d(48→2, 1×1)
                      │
               seg_logits (B, 2, 256, 256)  → CE+Dice Loss
```

- UNetDecoderBlock: 上采样2× → 与skip connection拼接 → 两层Conv+GN+GELU
- 分割loss = CrossEntropy(bg=1.0, lesion=5.0权重) + DiceLoss, 仅对有mask的样本计算

#### Stage 4：分类特征投影

```
f2 (B, 384, 16, 16)
        │
        ▼  cls_proj: Conv2d(384→256, 1×1) + GroupNorm(8, 256) + GELU
        │
f2_proj (B, 256, 16, 16)  ← 256维分类特征图
```

#### Stage 6：TextImageCrossAttention — 文本增强图像空间特征

**核心思想**：让图像每个空间位置"查询"文本序列，找到与当前位置最相关的文本描述（如"回声不均匀"增强异常区域的响应）。

```
f2_proj (B, 256, 16, 16) ──→ flatten ──→ img_tokens (B, 256, 256)
                                              │
                              img_proj (256→128) ──→ Q (B, 256, 128)
                                              │
text_hidden (B, 128, 768) ──→ text_proj (768→128) ──→ K, V (B, 128, 128)
                                              │
                              MultiheadAttention (4头, embed=128, dropout=0.1)
                              key_padding_mask = (attention_mask == 0)
                                              │
                              attn_out (B, 256, 128)
                                              │
                              Residual: attn_out = LayerNorm(attn_out + Q)
                                              │
                              out_proj (128→256) → reshape → (B, 256, 16, 16)
                                              │
                              f2_enhanced = f2_proj + out  (残差连接)
                                              │
                              f2_enhanced (B, 256, 16, 16)

# 注意: Q 是 256 个图像空间位置, K/V 是 128 个文本token
# 效果: 图像每个16×16位置学会关注报告中最相关的描述词
```

---

#### Stage 7：Seg-Guided Attention Pooling — 分割引导注意力池化

**核心思想**：用分割网络输出的病灶概率图作为空间注意力权重，让分类特征聚焦于病灶区域而非背景。

```
seg_logits (B, 2, 256, 256)
        │
        ▼  softmax → 取通道1(病灶概率)
seg_prob (B, 1, 256, 256)
        │
        ▼  双线性插值到 16×16
attn (B, 1, 16, 16)
        │
        ▼  attn = attn + 0.1           ← 背景最低权重0.1(不完全忽略)
        ▼  attn = attn / sum(attn)     ← 归一化为概率分布
        │
f2_enhanced (B, 256, 16, 16)
        │
        ▼  img_feat = Σ(f2_enhanced × attn)  ← 加权空间求和
        │
img_feat (B, 256)  ← 图像分支最终特征向量
```

**为什么这是关键创新（Exp#3→Exp#4 的跃升）**：
- Exp#3 用 GAP(Global Average Pooling)，256个空间位置等权重平均 → 病灶区域信号被大量背景稀释
- Exp#4 用分割概率加权 → 病灶区域权重远高于背景 → 分类特征更聚焦、更纯净
- 背景保留0.1最低权重，避免完全丢失上下文信息

#### Stage 10：Gated Trimodal Fusion — 门控三模态融合

**核心思想**：不同样本中三种模态的信息质量不同（如某些患者报告文本详细，另一些缺失临床数据），门控网络自适应地为每个样本调整三路特征的权重。

```
img_feat (B, 256) ─┐
text_feat (B, 128) ─┼──→ concat (B, 480)
meta_feat (B, 96)  ─┘         │
                               ▼
                    ┌─────────────────────────────┐
                    │    Gate Network              │
                    │    Linear(480→256) → GELU    │
                    │    Dropout(0.2)              │
                    │    Linear(256→3) → Sigmoid   │
                    └──────────┬──────────────────┘
                               │
                    gates (B, 3) ∈ [0, 1]
                    g_img, g_text, g_meta
                               │
                    ┌──────────┴──────────────────────────────────┐
                    │                                              │
                    │  img_proj:  Linear(256→256)                  │
                    │  text_proj: Linear(128→256)                  │
                    │  meta_proj: Linear(96→256)                   │
                    │                                              │
                    │  fused = g_img × img_proj(img_feat)          │
                    │        + g_text × text_proj(text_feat)       │
                    │        + g_meta × meta_proj(meta_feat)       │
                    │                                              │
                    │  fused = LayerNorm(fused)                    │
                    └──────────┬──────────────────────────────────┘
                               │
                    fused (B, 256)  ← 融合后的多模态特征
```

**为什么门控优于拼接（Exp#16 vs Exp#18）**：
- 拼接: 所有模态等权重贡献，低质量模态(如缺失文本/临床数据)会引入噪声
- 门控: Sigmoid独立控制每路权重 ∈ [0,1]，可以"关闭"低质量模态
- 输入依赖: 不同样本得到不同的门控权重，实现样本级自适应

---

#### Stage 11：分类输出

```
fused (B, 256)
        │
        ▼  Linear(256→128) → GELU → Dropout(0.4)
        ▼  Linear(128→2)
        │
cls_logits (B, 2)  ← [benign_score, no_tumor_score]
        │
        ▼  推理时: softmax → P(benign) → 与阈值比较
        │  训练时: WeightedCrossEntropy(class_weight=[2.98, 1.0]) + LabelSmoothing(0.1)
        │
        ▼  最终预测（两种策略）:
           F1最优阈值:
           if P(benign) >= 0.430:  → benign (良性肿瘤)
           else:                   → no_tumor (非肿瘤性息肉)

           临床安全阈值(benign漏检≤10%约束):
           if P(benign) >= 0.179:  → benign
           else:                   → no_tumor
```

---

#### Stage 12：联合损失函数

```
Loss = seg_loss + λ_cls(2.0) × cls_loss

seg_loss = CE(seg_logits, seg_target, weight=[1.0, 5.0])   ← 仅对has_mask=True的样本
         + DiceLoss(seg_logits, seg_target)

cls_loss = WeightedCE(cls_logits, label, weight=[2.98, 1.0], label_smoothing=0.1)
```

- λ_cls=2.0 使分类损失权重高于分割，确保分类任务为主要优化目标
- 分割权重 lesion=5.0 补偿前景占比极小（病灶仅占图像约5-10%）
- class_weight=2.98 由训练集 no_tumor/benign 样本比自动计算

## 五、关键发现与经验教训

### 成功经验

| 技术 | 验证实验 | 效果 |
|------|---------|------|
| **4通道输入 (RGB+Mask)** | Exp#1→3 | 引入先验mask信息，为后续SegAttn奠定基础 |
| **Seg-Guided Attention** | Exp#3→4 | F1 +5.8%，最重要的单一创新 |
| **结构化元数据** | Exp#4 | 6D临床特征提供图像外的诊断线索 |
| **交叉注意力 (text→image)** | Exp#17 | 空间级文本-图像交互优于简单拼接 |
| **门控融合** | Exp#18 | 自适应多模态融合，唯一超越基线方案 |
| **患者级CV** | Exp#19 | 正确的评估方法避免数据泄漏 |

### 失败教训

| 技术 | 验证实验 | 原因 |
|------|---------|------|
| **输入级Mixup** | Exp#2 | 混合样本破坏分割mask语义 |
| **EMA (decay=0.9995)** | Exp#7/12/14 | 慢更新使少数类(benign)梯度被平滑掉 |
| **SupCon Loss** | Exp#9 | batch=8中仅~2个benign，负样本严重不足 |
| **同构集成** | Exp#10 | 同架构同超参模型预测高度相关 |
| **后期拼接文本** | Exp#16 | 冻结BERT通用语义与图像特征不对齐 |
| **单独扩展临床特征** | Exp#15 | 低覆盖率特征的缺失值填充引入噪声 |
| **图像级KFold** | Exp#14 | 同一患者多张图像分到train/val造成泄漏 |

### 核心瓶颈

1. **训练数据不足**：1229例样本(其中benign仅309)，所有模型在epoch 10-15后严重过拟合
2. **少数类困难**：benign F1始终低于0.55，是整体F1的核心瓶颈
3. **BERT未适配**：冻结的通用中文BERT无法编码"回声不均匀""形态欠规则"等医学术语的诊断含义
4. **评估偏差**：测试集同时用于选epoch和调阈值，Exp#19的诚实评估确认偏差约1-2%

## 七、临床阈值与校准分析（新增）

> 目标：在“不漏诊良性肿瘤（漏检率≤10%）”前提下，尽可能多筛出 no_tumor（息肉随访）人群，并验证高置信度 no_tumor 预测是否可信。

### 1) 临床约束阈值搜索（基于 Exp#18 best weight）

离线推理结果（523例测试集）：

| 策略 | 阈值（基于P(benign)） | benign漏检率 | benign召回 | no_tumor召回 | no_tumor精度 | no_tumor筛出率 |
|------|------------------------|--------------|------------|--------------|--------------|----------------|
| F1最优策略 | 0.430 | 48.84% | 51.16% | 85.28% | 84.21% | 72.66% |
| **临床约束策略** | **0.179** | **9.30%** | **90.70%** | **34.01%** | **91.78%** | **27.92%** |

解释：
- 约束阈值 0.179 能把 benign 漏检率压到 10%以内（满足“尽量不漏诊”）。
- 在该约束下，模型仍可筛出 27.92% 的患者进入 no_tumor 随访路径，且精度 91.78%。
- 代价是 no_tumor 召回显著下降（34.01%），说明该策略偏保守。

### 2) no_tumor 置信度-准确率校准（Reliability Diagram）

已输出文件：
- `logs/20260408_task2_SwinV2Tiny_segcls_18/20260408_task2_SwinV2Tiny_segcls_18_no_tumor_reliability.png`
- `logs/20260408_task2_SwinV2Tiny_segcls_18/20260408_task2_SwinV2Tiny_segcls_18_no_tumor_reliability.csv`
- `logs/20260408_task2_SwinV2Tiny_segcls_18/20260408_task2_SwinV2Tiny_segcls_18_test_probs.csv`

关键指标：
- ECE = **0.0902**
- Brier Score = **0.1704**
- 高置信 no_tumor 区间 `P(no_tumor) >= 0.90`：共 **4/523 (0.76%)** 例，实际 no_tumor 比例 **100% (4/4)**

结论（按你的临床问题）：
- “当模型非常有把握（>90%）说是息肉时，它当前是可信的”，但覆盖率很低（仅0.76%），能直接放行随访的人数还不多。
- 若想“既不漏 benign 又多挑出息肉”，下一步应优先提升概率校准与 no_tumor 召回，而不只是单纯调阈值。
