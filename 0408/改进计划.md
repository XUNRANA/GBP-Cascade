### Exp#16: BERT超声报告编码 + 后期拼接融合

**目标**: 首次引入超声报告文本，用预训练中文BERT编码，验证文本语义对分类的增益

**文本编码方案**:
- 模型: `bert-base-chinese` (冻结全部参数，仅训练投影层)
- 输入: text_bert 字段，tokenizer截断至max_length=128 (覆盖绝大部分报告)
- 输出: [CLS] token embedding (768D) → Linear投影到128D

**融合方式**: 后期拼接 (Late Fusion)
```
img_feat (256D) || text_feat (128D) || meta_feat (64D) → 448D → MLP → 2类
```

**关键设计决策**:
1. **冻结BERT**: 训练数据仅~1200例，微调BERT必然过拟合，冻结是唯一选择
2. **保留6D元数据**: 不扩展到10D，独立验证文本的增量价值（与Exp#15对比）
3. **投影层可训练**: `Linear(768, 128) + LayerNorm + GELU + Dropout(0.3)`

**数据加载修改**:
```python
from transformers import BertTokenizer, BertModel

class GBPDatasetWithText(Dataset):
    def __init__(self, ...):
        self.tokenizer = BertTokenizer.from_pretrained("bert-base-chinese")
        # ...
    
    def __getitem__(self, idx):
        # ... 原有图像/mask/meta处理 ...
        text = self.text_dict.get(case_id, "")
        text_inputs = self.tokenizer(
            text, max_length=128, padding="max_length",
            truncation=True, return_tensors="pt"
        )
        return input_tensor, seg_target, meta_tensor, text_input_ids, text_attention_mask, label
```

**模型修改**:
```python
class SwinV2SegGuidedCls4chWithText(SwinV2SegGuidedCls4chModel):
    def __init__(self, ...):
        super().__init__(...)
        self.text_encoder = BertModel.from_pretrained("bert-base-chinese")
        for param in self.text_encoder.parameters():
            param.requires_grad = False  # 冻结BERT
        
        self.text_proj = nn.Sequential(
            nn.Linear(768, 128),
            nn.LayerNorm(128),
            nn.GELU(),
            nn.Dropout(0.3),
        )
        
        # 更新分类头: 256(img) + 128(text) + 64(meta) = 448
        self.cls_mlp = nn.Sequential(
            nn.Linear(448, 128), nn.GELU(), nn.Dropout(0.4),
            nn.Linear(128, num_cls_classes),
        )
    
    def forward(self, x, metadata=None, input_ids=None, attention_mask=None):
        seg_logits, img_feat = ...  # 原有图像+seg流程, 获取256D特征
        
        with torch.no_grad():
            text_out = self.text_encoder(input_ids=input_ids, attention_mask=attention_mask)
        text_feat = self.text_proj(text_out.last_hidden_state[:, 0])  # [CLS] → 128D
        
        meta_feat = self.meta_encoder(metadata.float())  # 64D
        fused = torch.cat([img_feat, text_feat, meta_feat], dim=1)  # 448D
        cls_logits = self.cls_mlp(fused)
        return seg_logits, cls_logits
```

**训练配置**:
- BERT冻结，不参与梯度更新，显存增加但可控
- 优化器只更新: backbone(2e-5) + text_proj/cls_mlp(2e-4)
- 其余超参数沿用Exp#4

**预期**:
- 文本包含医生对形态、回声、血流的综合描述，信息密度远高于离散编码
- 即使冻结BERT，[CLS]表征也应能捕捉到"轮廓不清、形态不规则"等恶性信号

### Exp#17: BERT文本 + 交叉注意力融合

**目标**: 用Cross-Attention实现文本与图像的深层交互，而非简单拼接

**核心思路**: 
- BERT输出不仅用[CLS]，而是用**全部token的hidden states** (seq_len × 768D)
- 图像空间特征 f2 (B, 256, 16, 16) 作为Query
- 文本token特征作为Key和Value
- 让图像的每个空间位置"查询"最相关的文本描述

**交叉注意力模块**:
```python
class TextImageCrossAttention(nn.Module):
    def __init__(self, img_dim=256, text_dim=768, hidden_dim=128, num_heads=4):
        super().__init__()
        self.img_proj = nn.Linear(img_dim, hidden_dim)
        self.text_proj = nn.Linear(text_dim, hidden_dim)
        self.cross_attn = nn.MultiheadAttention(
            embed_dim=hidden_dim, num_heads=num_heads,
            dropout=0.1, batch_first=True,
        )
        self.norm = nn.LayerNorm(hidden_dim)
        self.out_proj = nn.Linear(hidden_dim, img_dim)
    
    def forward(self, img_feat_2d, text_hidden_states, text_attention_mask):
        """
        img_feat_2d: (B, C, H, W) → reshape → (B, H*W, C)
        text_hidden_states: (B, seq_len, 768)
        """
        B, C, H, W = img_feat_2d.shape
        img_tokens = img_feat_2d.flatten(2).permute(0, 2, 1)  # (B, H*W, C)
        
        Q = self.img_proj(img_tokens)       # (B, H*W, hidden)
        K = self.text_proj(text_hidden_states)  # (B, seq_len, hidden)
        V = K
        
        # text_attention_mask → key_padding_mask
        key_padding_mask = (text_attention_mask == 0)  # True = padding
        
        attn_out, _ = self.cross_attn(Q, K, V, key_padding_mask=key_padding_mask)
        attn_out = self.norm(attn_out + Q)  # residual
        
        # 映射回图像维度并reshape
        out = self.out_proj(attn_out)  # (B, H*W, C)
        out = out.permute(0, 2, 1).view(B, C, H, W)
        return out
```

**整体前向流程**:
```
Input 4ch → SwinV2 Encoder → f0, f1, f2, f3
                                    ↓
                              f2 (384ch, 16×16)
                                    ↓
                              cls_proj → (256ch, 16×16)
                                    ↓
text_bert → BERT(frozen) → hidden_states (seq_len, 768)
                                    ↓
                    ┌─── TextImageCrossAttention ───┐
                    │  Q: img (256, 16×16)          │
                    │  K,V: text (seq_len, 768→128) │
                    └──────────────────────────────┘
                                    ↓
                          enhanced_img_feat (256, 16×16)
                                    ↓
                    Seg-Guided Attention Pool → 256D
                                    ↓
                    cat[256D, meta(64D)] → 320D → MLP → 2类
```

**关键点**:
- 交叉注意力放在seg-guided attention**之前**，让文本信息增强图像空间特征
- 保留残差连接：即使文本无用，模型也能退化为原始Exp#4
- 4头注意力，hidden_dim=128，参数量可控

**训练配置**:
- BERT冻结，cross-attention模块可训练 (lr=2e-4)
- 其余同Exp#4

**预期**:
- 当报告提到"胆囊壁增厚"时，注意力应集中在壁区域
- 比Exp#16的简单拼接更能建模文本-图像的空间对应关系

### Exp#18: 全模态融合 (扩展临床 + BERT文本 + 门控融合)

**目标**: 将前三个实验的有效组件整合，设计一个完整的多模态融合架构

**三路特征**:
1. **图像分支**: Seg-guided attention → 256D (与Exp#17相同，含交叉注意力增强)
2. **文本分支**: BERT [CLS] → 投影128D (与Exp#16相同)
3. **临床分支**: 10D扩展特征 → MLP → 96D (与Exp#15相同)

**门控融合机制 (Gated Fusion)**:
```python
class GatedTrimodalFusion(nn.Module):
    """三路特征门控融合: 自适应加权图像/文本/临床特征"""
    def __init__(self, img_dim=256, text_dim=128, meta_dim=96, fusion_dim=256):
        super().__init__()
        total_dim = img_dim + text_dim + meta_dim  # 480
        
        # 门控网络: 为每路特征生成权重
        self.gate = nn.Sequential(
            nn.Linear(total_dim, fusion_dim),
            nn.GELU(),
            nn.Dropout(0.2),
            nn.Linear(fusion_dim, 3),  # 3路权重
            nn.Sigmoid(),
        )
        
        # 各路特征投影到统一维度
        self.img_proj = nn.Linear(img_dim, fusion_dim)
        self.text_proj = nn.Linear(text_dim, fusion_dim)
        self.meta_proj = nn.Linear(meta_dim, fusion_dim)
        
        self.norm = nn.LayerNorm(fusion_dim)
    
    def forward(self, img_feat, text_feat, meta_feat):
        concat = torch.cat([img_feat, text_feat, meta_feat], dim=1)
        gates = self.gate(concat)  # (B, 3)
        g_img, g_text, g_meta = gates[:, 0:1], gates[:, 1:2], gates[:, 2:3]
        
        fused = (g_img * self.img_proj(img_feat) + 
                 g_text * self.text_proj(text_feat) + 
                 g_meta * self.meta_proj(meta_feat))
        return self.norm(fused)  # (B, fusion_dim)
```

**完整架构图**:
```
Input 4ch → SwinV2 Encoder → UNet Seg Decoder → seg_logits
                ↓
          f2 (384→256, 16×16)
                ↓
text_bert → BERT(frozen) → hidden_states
                ↓               ↓
       CrossAttention(img, text)  [CLS] → text_proj → 128D ──┐
                ↓                                              │
       Seg-Guided Attn Pool → img_feat (256D) ────────────────┤
                                                               │
       10D Clinical → MLP → meta_feat (96D) ──────────────────┤
                                                               ↓
                                                    GatedTrimodalFusion
                                                         ↓ (256D)
                                                    MLP → 2类
```

**训练配置**:
- BERT冻结
- backbone_lr: 2e-5, head_lr: 2e-4 (cross-attn, gate, projections, cls_mlp)
- 其余同Exp#4
- 考虑增加warmup到10 epochs (新模块多，需要更长预热)

**预期**:
- 门控机制能自动学习不同样本依赖哪路信息更多
  - 报告描述详细的样本 → 文本权重高
  - 报告简短/缺失的样本 → 图像/临床权重高
- 是信息利用最充分的方案，预期效果最好

## 注意事项

1. **BERT必须冻结**: 训练集仅~1200例，微调110M参数的BERT必然过拟合
2. **显存预估**: BERT inference (batch=8, seq=128) 约需额外1-2GB显存，总计应在8GB以内
3. **文本缺失处理**: 若某样本无text_bert，用全零向量替代（文本投影层加Dropout可处理）
4. **递进验证**: Exp#15→16→17→18逐步加入组件，每步都能与前一步对比，明确每个组件的增量价值
5. **依赖安装**: 需要 `pip install transformers` (huggingface transformers库)
6. **BERT下载**: 首次运行需下载bert-base-chinese (~400MB)，建议提前缓存到本地
