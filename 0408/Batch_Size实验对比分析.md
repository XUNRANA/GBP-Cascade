## 三、分组分析

### 3.1 1:1 平衡数据组 (Exp#20b → 21a → 21b)

| 指标 | BS=8 (20b) | BS=16 (21a) | BS=32 (21b) | 趋势 |
|------|-----------|-------------|-------------|------|
| Benign Precision | 0.6544 | **0.6964** (+6.4%) | 0.6443 (-1.5%) | 16最高 |
| Benign Recall | 0.6899 | 0.6047 (-12.4%) | **0.7442** (+7.9%) | 32最高 |
| **Benign F1** | 0.6717 | 0.6473 (-3.6%) | **0.6906** (+2.8%) | **32最优** |
| No_tumor F1 | 0.6534 | **0.6909** (+5.7%) | 0.6387 (-2.3%) | 16最高 |
| F1 macro | 0.6625 | **0.6691** (+1.0%) | 0.6647 (+0.3%) | 16最高 |
| 最优阈值 | 0.530 | 0.445 | 0.475 | — |
| 最优Epoch | 15 | 10 | 30 | 32收敛更慢 |

**发现**：
- **BS=32 获得了所有实验中最高的 Benign F1 (0.6906)** 和最高的 Benign Recall (0.7442)
- BS=16 在 Benign Precision (0.6964) 和整体 F1 macro (0.6691) 上最优
- BS=32 收敛更慢（最优 epoch 30 vs 10/15），但最终 Benign 类效果最好
- 三个 1:1 实验的阈值都接近 0.5，说明模型校准良好

### 3.2 1:3 不平衡数据组 (Exp#18 → 21c → 21d)

| F1 macro | **0.6846** | 0.6707 (-2.0%) | 0.6602 (-3.6%) | **8最优, 越大越差** |
| 最优阈值 | 0.430 | 0.400 | 0.430 | 都偏离0.5 |
| 最优Epoch | 15 | 15 | 25 | — |

**发现**：
- **1:3 数据上增大 batch size 全面负面**，Benign F1 和整体 F1 都下降
- BS=32 的 Benign F1 仅 0.4841，是所有6组实验中最低的
- 阈值始终偏离 0.5（0.400~0.430），说明模型严重偏向 no_tumor 预测

## 五、深度分析

### 1. 为什么大 Batch Size 在 1:1 数据上有效？

**梯度估计更准确**：
- 1:1 数据中，每个大 batch 包含近似等量的 benign 和 no_tumor 样本
- BS=32 时，每 batch 约 16 benign + 16 no_tumor → 两类梯度同时得到充分估计
- 梯度噪声降低，参数更新方向更准确
- 配合线性缩放的学习率，模型能在更准确的梯度方向上迈更大步

**BS=32 特别适合小数据集 (618张)**：
- 618张 / 32 = ~19 batches/epoch，每 batch 覆盖数据比例更高
- 模型每个 epoch 能看到数据的更"全局"的统计特征
- 虽然收敛更慢（epoch 30 vs 15），但最终效果更好

### 2. 为什么大 Batch Size 在 1:3 数据上有害？

**类别偏差放大**：
- BS=8 时，每 batch 约 2 benign + 6 no_tumor
- BS=16 时，每 batch 约 4 benign + 12 no_tumor
- BS=32 时，每 batch 约 8 benign + 24 no_tumor
- **大 batch 中 no_tumor 的梯度主导更严重**，模型更倾向学习多数类特征
- class_weight 只放大 benign 梯度幅度，不改变 batch 内的多样性分布

**小 batch 的"隐式正则化"丧失**：
- 小 batch 的梯度噪声本身起到正则化作用
- 对于少数类 benign，这种噪声帮助模型探索更多决策边界
- 大 batch 减少噪声，反而让模型更快收敛到偏向多数类的局部最优

### 3. 阈值分析

| 实验 | 最优阈值 | 偏离 0.5 | 解读 |
|------|---------|---------|------|
| Exp#18 (1:3, bs=8) | 0.430 | -0.070 | 中度偏向 no_tumor |
| Exp#21c (1:3, bs=16) | 0.400 | **-0.100** | 更严重偏向 no_tumor |
| Exp#21d (1:3, bs=32) | 0.430 | -0.070 | 中度偏向 no_tumor |
| Exp#20b (1:1, bs=8) | 0.530 | +0.030 | 近似校准 |
| Exp#21a (1:1, bs=16) | 0.445 | -0.055 | 轻微偏向 no_tumor |
| Exp#21b (1:1, bs=32) | 0.475 | -0.025 | **近似校准** |

- 1:3 组阈值始终需要大幅调低（0.400~0.430），模型输出概率系统性偏向 no_tumor
- 1:1 组阈值接近 0.5，特别是 **BS=32 (0.475) 几乎完美校准**
- BS=32 + 1:1 数据 = 最佳概率校准，这意味着模型真正学会了区分两类

- 大 batch 的学习率更大，需要更多 epoch 来精细调整

## 六、关键发现

### 发现1：Batch Size 效果强烈依赖数据平衡性

| 数据状态 | BS 增大效果 | 原因 |
|---------|-----------|------|
| **1:1 平衡** | Benign F1 提升 (特别是 bs=32) | 梯度估计更准确，两类等权更新 |
| **1:3 不平衡** | Benign F1 下降 | 大 batch 放大多数类偏差 |

这是本次实验最重要的发现。**"大 batch 一般更好"的经验规律，在类别不平衡场景下不成立。**

### 发现2：BS=32 + 1:1 数据产生了最优的 Benign 识别

| 维度 | Exp#21b (1:1, bs=32) | 值 |
|------|---------------------|-----|
| Benign F1 | **所有实验最高** | 0.6906 |
| Benign Recall | **所有实验最高** | 0.7442 |
| 阈值校准 | **近似完美** | 0.475 |

对于临床场景（漏检良性肿瘤比误诊更危险），**Exp#21b 是目前最佳方案**。

### 发现3：1:3 数据的最佳 batch size 仍然是 8

增大 batch size 无法解决数据不平衡问题。在 1:3 数据上：
- Benign F1: 0.5217 (bs=8) > 0.5076 (bs=16) > 0.4841 (bs=32)
- F1 macro: 0.6846 (bs=8) > 0.6707 (bs=16) > 0.6602 (bs=32)

**数据平衡是比调 batch size 更根本的因素。**

## 八、后续优化方向

### 方向1：保留全部数据 + WeightedRandomSampler + BS=32

结合本轮实验发现，最有潜力的方案：
```
训练集: 全部 1229 张 (不丢弃数据)
采样: WeightedRandomSampler (每 epoch 有效 1:1 采样)
Batch Size: 32
LR: backbone 8e-5, head 8e-4
class_weight: [1.0, 1.0] (采样已平衡)
```
预期：兼顾 Exp#21b 的 Benign F1 优势 + Exp#18 的 No_tumor 数据多样性

### 方向2：Benign 专属强增强 + 1:1 + BS=32

在 Exp#21b 基础上，对 benign 样本额外增强（弹性变形、CutMix），增加等效多样性

### 方向3：两阶段训练

1. 阶段一：1:3 全部数据 + BS=8 训练到 epoch 15 → 学好 no_tumor 特征
2. 阶段二：冻结 backbone，1:1 + BS=32 微调分类头 → 校准 benign 决策边界

## 九、总结

| 结论 | 支撑数据 |
|------|---------|
| **大 batch 在平衡数据上有效** | 1:1 组: BS=32 Benign F1 最高 (0.6906) |
| **大 batch 在不平衡数据上有害** | 1:3 组: BS 越大 Benign F1 越低 (0.52→0.51→0.48) |
| **数据平衡比 batch size 更重要** | 同 BS=32: 1:1 Benign F1=0.69 vs 1:3 Benign F1=0.48 |
| **BS=32 + 1:1 = 最佳 Benign 识别** | Benign F1=0.6906, Recall=0.7442, 阈值 0.475 |
| **BS=8 + 1:3 = 最佳整体 F1** | F1 macro=0.6846 (但 Benign F1 仅 0.52) |

**核心结论**：Batch Size 的效果取决于数据平衡性。在类别不平衡场景下，先解决数据平衡问题，再调 batch size，才能获得真正的提升。**Exp#21b (1:1, BS=32) 目前是 Benign 识别的最优方案**。
