# 0408 实验总结：充分利用文本信息

## 一、实验概述

基于 0402 Exp#4 基线模型（SwinV2-Tiny@256 + 4ch + UNet Seg + Seg-Guided Attention + 6D Metadata，F1 macro=0.6707/阈值优化0.6741），设计了4组递进实验，探索超声报告文本和扩展临床特征对胆囊息肉良恶性分类的增益。

- 数据集：训练集 1229 张（benign=309, no_tumor=920），测试集 523 张（benign=129, no_tumor=394）
- 4个实验在4张GPU上并行训练，均训练100 epochs
- BERT (bert-base-chinese) 在所有实验中全部冻结，仅训练投影层

## 三、各实验分析

### Exp#15：扩展结构化临床特征 (6D→10D)

**改动**：新增回声(echo_type)、个数(lesion_count)、壁厚(wall_thickness)、超声直径(us_diameter) 4个临床特征，meta_hidden 64→96。

**结果**：F1=0.6506（阈值优化），**低于基线 0.6741**。

**分析**：
- 扩展临床特征反而导致性能下降，benign F1 从 0.51 降至 0.4628
- 新增特征中壁厚覆盖率仅55.2%，大量缺失值（中位数填充）可能引入噪声
- 4个新特征与原有6D特征存在信息冗余（如 us_diameter 与 size_mm）
- 10D特征空间增大但训练样本不变，可能导致元数据分支过拟合

### Exp#16：BERT[CLS] 后期拼接融合

**改动**：冻结BERT编码超声报告，取[CLS] token投影到128D，与图像256D和元数据64D拼接（448D）后分类。

**结果**：F1=0.6433（阈值优化），**四组实验中最差**。

**分析**：
- 简单拼接融合未能有效利用文本信息，benign F1仅0.4319
- 冻结BERT的[CLS]表征是通用语义，未经医学领域适配，可能与图像特征语义空间不对齐
- 后期拼接缺乏模态间交互，文本和图像特征各自独立，无法建立跨模态关联
- 阈值需大幅偏移到0.600才能提升，说明模型置信度校准有问题
- 最优epoch仅在第10轮出现，之后持续退化，过拟合严重

### Exp#17：BERT 交叉注意力融合

**改动**：用BERT全token hidden states作为K/V，图像空间特征f2作为Q，通过4头交叉注意力实现文本-图像空间级交互，增强后的图像特征再经过Seg-Guided Attention。

**结果**：F1=0.6639（阈值优化），**接近但仍低于基线**。

**分析**：
- 交叉注意力比简单拼接效果好很多（+0.0206 F1），验证了空间级文本-图像交互的价值
- Benign的recall达到0.4961，是四组实验中最高的单项recall，说明交叉注意力有助于发现恶性线索
- 阈值仅需微调到0.505，说明模型置信度校准良好
- 但未超越基线，可能是冻结BERT的表征质量限制了交叉注意力的上限

### Exp#18：全模态门控融合（最佳方案）

**改动**：整合所有有效组件——交叉注意力增强图像(256D) + BERT[CLS]投影(128D) + 10D扩展临床(96D)，通过门控融合网络自适应加权三路特征。Warmup增加到10 epochs。

**结果**：F1=0.6846（阈值优化），**唯一超越基线的实验**（+0.0105）。

**分析**：
- 门控融合有效协调了三路特征，避免了Exp#15中单纯扩展临床特征的负面效果
- Benign F1达到0.5217，是所有实验中最好的，相比基线0.51有明显提升
- Benign的精确率(0.5323)和召回率(0.5116)较为均衡，没有严重偏向某一方
- No_tumor F1达到0.8474，也是最好的
- 最优阈值偏低(0.430)，说明模型对benign的预测偏保守，门控可能给予no_tumor更高的基础权重
- Warmup增加到10 epochs有助于多模块协同训练的稳定性

## 四、关键发现

### 1. 严重的过拟合问题
所有实验的训练集分类准确率均在epoch 30后达到98%+，最终接近99.5%，而测试F1在epoch 10-15后即开始下降。最优模型均出现在极早期（epoch 10-15），说明：
- 1229例训练样本对于当前模型复杂度明显不足
- 后85-90个epoch的训练基本无效，反而损害了泛化能力
- 需要更强的正则化策略或早停机制

### 2. 文本信息引入方式至关重要
- **简单拼接（Exp#16）效果最差**，甚至不如不用文本的基线
- **交叉注意力（Exp#17）显著优于拼接**，空间级交互能让图像特征"理解"文本描述
- **门控融合（Exp#18）效果最好**，自适应加权避免了低质量模态对高质量模态的干扰

### 3. 扩展临床特征需谨慎
- 单独增加临床特征（Exp#15）反而性能下降
- 但在门控融合框架下（Exp#18），同样的10D特征配合其他模态反而带来增益
- 说明特征的价值取决于融合策略，不能孤立评估

### 4. 冻结BERT的局限性
- 通用中文BERT的[CLS]表征缺乏医学领域专业性
- 即使通过交叉注意力/门控融合改善了利用方式，文本带来的提升仍然有限
- 未来可考虑使用医学领域预训练的中文BERT（如MC-BERT）或在更大数据集上微调

---

## 五、分割任务表现

所有实验的分割性能均保持优异水平：
- Lesion IoU: 0.96~0.99
- Lesion Dice: 0.98~0.99

分割任务在所有方案中都非常稳定，未受到文本/临床特征引入的影响。

## 七、后续建议

1. **早停策略**：将patience设为15-20 epochs，避免后期过拟合浪费训练时间
2. **数据增强**：针对benign类（少数类）加强增强策略，如Mixup、CutMix
3. **医学BERT**：尝试替换为医学领域预训练的中文语言模型
4. **BERT微调**：可尝试仅微调BERT最后2-3层，配合更强的Dropout
5. **Exp#18改进方向**：
   - 调整门控网络结构，尝试Softmax归一化替代Sigmoid
   - 尝试降低临床特征中低覆盖率特征的权重
   - 增加交叉注意力的层数或heads
6. **集成学习**：将Exp#17和Exp#18的预测结果进行集成
