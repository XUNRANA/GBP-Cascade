# GBP-Cascade 改进计划1

## Context

当前最优模型 Exp#18 (全模态门控融合) F1 macro = 0.6846，但存在严重问题：
1. **评估不可信**：测试集同时用于选epoch和调阈值（数据泄漏），真实性能可能低2-3%
2. **严重过拟合**：最优epoch仅在第10-15轮，训练acc达99%而测试F1持续下降
3. **Benign瓶颈**：少数类F1仅0.52，是整体性能的核心限制
4. **BERT未适配**：冻结的通用中文BERT对医学超声报告文本编码质量有限
5. **骨干网络未探索**：只试过SwinV2-Tiny，从未测试更大变体

目标：F1 macro > 0.72，理想 0.75+

---

## 8个递进实验

### Exp#19：修复评估体系（必须最先做）

**假设**：当前0.6846含评估偏差，需建立诚实基线

**改动**：
- 基于Exp#4架构（最简单的强模型），不用Exp#18（避免混淆变量）
- **患者级** 5折交叉验证：用 `StratifiedGroupKFold(n_splits=5, group=case_id_norm)` 按患者分组
  - 0406 Exp#14的KFold失败原因之一：图像级分折，同一患者出现在train和val中
- 每折训练30 epochs，**每epoch评估**，patience=8 早停
- **不用EMA**（0406 Exp#12的EMA=0.9995导致benign类完全学不到）
- 523张测试集**仅最终评估一次**
- 汇报：5折平均val F1 + 最终test F1 + 5模型集成test F1

**关键文件**：
- 基于 `0408/scripts/seg_cls_utils_v5.py` 新建 `seg_cls_utils_v6.py`
- 修改 `run_seg_cls_experiment` 增加CV逻辑
- 新增 `sklearn.model_selection.StratifiedGroupKFold`

**预期**：诚实基线 val F1 约 0.62-0.65（低于0.6846但真实）

---

### Exp#20：抗过拟合训练策略

**假设**：模型在epoch 15后开始记忆训练集，需延长有效训练窗口

**改动**（在Exp#19的CV基础上，仍用Exp#4架构）：
- **R-Drop正则化**：每个输入前向两次（不同dropout mask），加KL散度损失
  - 为什么选R-Drop：不依赖batch size（SupCon在Exp#9因batch=8失败）、不破坏mask语义（Mixup在Exp#2失败）
- backbone_lr: 2e-5 → **1e-5**（减缓记忆高层特征）
- weight_decay: 0.05 → **0.1**
- cls_dropout: 0.4 → **0.5**
- **Stochastic Depth**: `drop_path_rate=0.2`（timm原生支持）
- label_smoothing: 0.1 → **0.0**（LS与class_weight冲突，会将少数类标签向均匀分布平滑）
- epochs: 25，早停patience=8

**关键实现**：
```python
# R-Drop: 在train_one_epoch中
logits1 = model(images, metadata, ...)
logits2 = model(images, metadata, ...)  # 第二次前向，不同dropout
ce_loss = (criterion(logits1, labels) + criterion(logits2, labels)) / 2
kl_loss = compute_kl_loss(logits1, logits2)  # 对称KL散度
loss = ce_loss + alpha_rdrop * kl_loss  # alpha_rdrop=0.5
```

**预期**：val F1 +1-3% → 0.64-0.67

### Exp#21：BERT部分微调

**假设**：冻结的通用BERT无法编码"回声不均匀""形态欠规则"等医学术语的诊断含义

**改动**（在Exp#20的recipe + CV基础上，升级到Exp#18架构）：
- **解冻BERT最后2层**（layer 10-11），约18M参数
- BERT LR = **5e-6**（比backbone低2倍，比head低40倍）
- **梯度累积**：effective_batch_size = 16（accumulate 2 steps × batch 8）
- 解冻层的attention_dropout提高到0.2
- 保留R-Drop正则化

**显存预估**：BERT 2层反向传播额外约4GB，总计~12GB，单GPU可行

**预期**：val F1 +2-4% → 0.66-0.70

**风险**：若过拟合，回退到冻结BERT + 尝试下载医学中文BERT（MC-BERT等）

---

### Exp#22：SwinV2-Small骨干升级

**假设**：SwinV2-Tiny (27.6M) 容量不足以学习benign和no_tumor的细微差异；SwinV2-Small (49.0M) 的stage-3层数从6增加到18，提供更丰富的空间-语义表征

**改动**（在Exp#21最佳recipe基础上）：
- `swinv2_tiny_window8_256` → **`swinv2_small_window8_256`**（timm可用，已验证）
- 特征通道完全相同 [96, 192, 384, 768]，**UNet decoder、CrossAttention、GatedFusion等全部无需改动**
- backbone_lr: → **5e-6**（更大模型需更保守的微调）
- warmup: 10 → **12 epochs**
- 保留R-Drop + Stochastic Depth

**参数量**：49.0M (backbone) + ~7M (heads) + 110M (BERT frozen) = ~166M total, ~56M trainable

**预期**：val F1 +1-3% → 0.67-0.72

---

### Exp#23：少数类过采样 + Focal Loss

**假设**：class_weight只改变梯度幅度，不增加benign样本多样性；需要在采样和损失两个层面同时处理不平衡

**改动**：
- **WeightedRandomSampler**：benign采样概率0.5，no_tumor采样概率0.5（每batch近似均衡）
- **分类损失改为Focal Loss**：alpha=[0.6, 0.4]（benign权重更高），gamma=2.0
- **Benign专属增强**：对benign样本额外施加更强旋转(±30°)和弹性变形
- **特征级Cutout**：在f2_proj特征图上随机mask 10%的空间位置（仅训练时），不影响分割分支
  - 为什么不在输入级Mixup：Exp#2证明输入级混合破坏mask语义

**预期**：Benign F1从0.52提升到0.55-0.60，整体val F1 +1-2%

---

### Exp#24：临床特征优化

**假设**：wall_thickness (55.2%覆盖率) 的中位数填充引入噪声（Exp#15表明10D alone反而降分）

**改动**：
- **移除wall_thickness**（覆盖率太低）
- **添加缺失指示器**：对缺失率>5%的特征（echo_type、lesion_count、us_diameter），各加一个binary is_missing标志
- 最终特征：9D基础 + 3D缺失指示器 = **12D**
- 缺失值改用**零填充**（z-score后0=均值，配合指示器无歧义）
- meta_hidden: 64 → **96**

**预期**：val F1 +0.5-1.5%

---

### Exp#25：多样性架构集成

**假设**：Exp#10的3-seed同构集成失败（F1=0.6326），因为模型高度相关；不同架构的互补性远优于同架构不同种子

**方案**：训练4个互补模型，4GPU并行：
| GPU | 模型 | 架构 | 模态 |
|-----|------|------|------|
| 0 | Model A | **SwinV2-Small** + 全Exp#18架构 | 图像+文本+临床 |
| 1 | Model B | **SwinV2-Tiny** + SegAttn + 临床(无文本) | 图像+临床 |
| 2 | Model C | **SwinV2-Small** + CrossAttn(无门控) | 图像+文本 |
| 3 | Model D | **ConvNeXt-Tiny** + 临床 | 图像+临床(CNN骨干) |

- 集成策略：**Stacking** — 在val fold预测上训练LogisticRegression学习权重
- 阈值在val集上优化

**预期**：集成 val F1 +2-4% → 目标 **0.72-0.75**

---

### Exp#26：TTA + 概率校准

**假设**：模型已训练了对几何变换的不变性，TTA可降低预测方差

**方案**（仅推理时）：
- **5视角TTA**：原图、水平翻转、垂直翻转、+10°旋转、-10°旋转
- 图像和mask同步变换，文本/元数据不变
- 平均softmax概率
- **Temperature Scaling**：在val集上拟合温度参数T校准输出概率
- 应用于单最优模型 + Exp#25集成

**预期**：val F1 +0.5-1.5% → 最终目标 **0.73-0.76**

## 验证方式

每个实验完成后：
1. 检查5折val F1的均值和方差（方差应<0.03）
2. 检查训练曲线：train acc是否仍快速饱和（>95%说明过拟合未解决）
3. 检查benign类的P/R/F1是否改善（核心目标）
4. 最终仅用Exp#25/26的集成模型在test集评估一次
