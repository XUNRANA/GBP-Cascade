# 类别平衡实验对比分析：1:3 vs 1:1

> Exp#18（原始 1:3 不平衡）vs Exp#20b（物理欠采样 1:1 平衡）
> 模型架构完全相同：SwinV2-Tiny + BERT CrossAttention + GatedTrimodalFusion

---

## 一、实验设置对比

| 配置项 | Exp#18 (1:3 不平衡) | Exp#20b (1:1 平衡) |
|--------|--------------------|--------------------|
| **训练集** | 1229 张 (benign=309, no_tumor=920) | **618 张 (benign=309, no_tumor=309)** |
| **测试集** | 523 张 (benign=129, no_tumor=394) | **258 张 (benign=129, no_tumor=129)** |
| **训练比例** | benign:no_tumor = 1:2.98 | **benign:no_tumor = 1:1** |
| **测试比例** | benign:no_tumor = 1:3.05 | **benign:no_tumor = 1:1** |
| **平衡策略** | class_weight=[1.986, 0.668] 加权损失 | **物理欠采样 + class_weight=[1.0, 1.0]** |
| 模型架构 | SwinV2SegGuidedCls4chTrimodal | 完全相同 |
| Backbone | SwinV2-Tiny@256 (27.6M) | 完全相同 |
| BERT | bert-base-chinese (冻结) | 完全相同 |
| 融合方式 | GatedTrimodalFusion (img+text+meta) | 完全相同 |

**唯一区别**：训练/测试数据量和类别比例、分类loss的class_weight。

## 五、深度分析

### 1. 为什么 Benign 类大幅提升？

**根本原因：训练信号均衡化**

在 Exp#18 (1:3) 中：
- 每个 batch (8张) 平均仅含 ~2 张 benign + ~6 张 no_tumor
- 虽然 class_weight=1.986 增加了 benign 的梯度幅度，但**模型看到的 benign 样本多样性不变**
- 模型倾向于学习 no_tumor 的特征模式（因为见得多），对 benign 的特征捕捉不充分
- 最终阈值需降到 0.430 才能"人为"提升 benign 召回率

在 Exp#20b (1:1) 中：
- 每个 batch 平均 ~4 张 benign + ~4 张 no_tumor
- 模型每次参数更新同时受到两类梯度的等量驱动
- **模型真正学会了区分 benign 的特征**，而非靠阈值偏移
- 阈值 0.530 接近 0.5，说明模型置信度校准良好

### 2. 为什么 No_tumor 类大幅下降？

- 训练集从 920 张 no_tumor 减少到 309 张（丢失 **66.4%** 的样本）
- No_tumor 类内部也有多样性（不同息肉形态、不同超声表现），丢弃样本导致覆盖不全
- 测试集也从 394 张缩减为 129 张，评估方差增大

### 3. 为什么整体 F1 macro 下降？

**注意：两个实验的测试集不同，F1 macro 不可直接对比**

- Exp#18 测试集 523 张 (129:394)，F1 macro 受 no_tumor(394张) 权重更大的影响
- Exp#20b 测试集 258 张 (129:129)，两类等权
- 如果在相同的 258 张平衡测试集上评估 Exp#18，其 F1 macro 大概率会低于 0.6846

### 4. 阈值分析

| 实验 | 默认阈值F1 | 最优阈值 | 最优阈值F1 | 阈值偏移量 |
|------|-----------|---------|-----------|-----------|
| Exp#18 | 0.6714 | 0.430 | 0.6846 | -0.070 |
| Exp#20b | 0.6464 | 0.530 | 0.6625 | +0.030 |

- Exp#18 阈值偏移 -0.070，说明模型系统性地对 benign 预测不足，需要大幅降低阈值才能提升 benign 召回
- Exp#20b 阈值仅偏移 +0.030，模型对两类的预测置信度更均衡

## 六、关键发现

### 发现1：Benign 的识别瓶颈是数据不平衡，不是模型能力

Exp#18 的 Benign F1 仅 0.52，一直被认为是模型能力瓶颈。但 Exp#20b 证明：**给模型均衡的训练信号，Benign F1 可以达到 0.67（+28.8%）**。这说明模型架构有足够能力区分两类，问题出在训练数据分布上。

### 发现2：class_weight 不等于真正的数据平衡

Exp#18 的 class_weight=[1.986, 0.668] 只是放大了 benign 的梯度，但：
- **不增加样本多样性**：模型看到的 benign 图像种类没有变多
- **不改变 batch 组成**：每个 batch 仍然 benign 偏少
- **不改变特征空间的覆盖**：模型对 benign 的特征表示空间学习不充分

真正的 1:1 平衡训练（即使是欠采样）在 Benign 类上效果远优于 class_weight 加权。

### 发现3：数据量与平衡性的权衡

| 方案 | 训练量 | Benign F1 | No_tumor F1 | 整体 F1 |
|------|--------|-----------|-------------|---------|
| 1:3 + class_weight | 1229张 | 0.5217 | **0.8474** | **0.6846** |
| 1:1 欠采样 | 618张 | **0.6717** | 0.6534 | 0.6625 |

- 如果**目标是 Benign 识别**（临床上漏检良性肿瘤更危险），1:1 方案更优
- 如果**目标是整体 F1 macro**，1:3 + class_weight 方案整体更优
- 最佳策略可能是**两者结合**：保留全部数据 + 更强的平衡采样策略

## 七、后续优化建议

### 方向1：保留全部数据 + 均衡采样（兼顾数据量和平衡性）

```
方案: WeightedRandomSampler + class_weight=[1.0, 1.0]
     + 对 benign 样本施加更强数据增强 (增加多样性)
预期: 结合两者优点, Benign F1 > 0.60, No_tumor F1 > 0.80
```

### 方向2：Benign 专属强增强

- 对 benign 样本额外施加弹性变形、更大旋转角度、CutMix
- 增加 benign 的等效多样性，弥补样本量不足
- 配合 1:1 采样，不丢弃 no_tumor 数据

### 方向3：Focal Loss 替代 CE

- Focal Loss 自动降低易分样本的权重，聚焦于难分的 benign 样本
- 配合均衡采样，可能进一步提升 benign 精确率

### 方向4：两阶段训练

1. 第一阶段：全部数据 + 1:3 正常训练 → 学好 no_tumor 特征
2. 第二阶段：冻结 backbone，1:1 采样微调分类头 → 校准 benign 决策边界

## 八、总结

| 维度 | Exp#18 (1:3) | Exp#20b (1:1) | 优势方 |
|------|-------------|---------------|--------|
| Benign 精确率 | 0.5323 | **0.6544** | **Exp#20b (+22.9%)** |
| Benign 召回率 | 0.5116 | **0.6899** | **Exp#20b (+34.9%)** |
| Benign F1 | 0.5217 | **0.6717** | **Exp#20b (+28.8%)** |
| No_tumor F1 | **0.8474** | 0.6534 | **Exp#18** |
| 整体 F1 macro | **0.6846** | 0.6625 | Exp#18 (但测试集不同) |
| 模型校准 | 阈值偏移大(0.430) | 阈值接近0.5(0.530) | **Exp#20b** |
| 训练数据利用 | 全部1229张 | 仅618张(丢弃50%) | **Exp#18** |

**核心结论**：1:1 类别平衡训练证明了 Benign 类的识别潜力远未被 Exp#18 挖掘。未来应在保留全部数据的前提下，通过更智能的采样策略（如 WeightedRandomSampler + Benign 增强）来获得两者的最优平衡。
