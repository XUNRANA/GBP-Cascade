## 1. 实验概述

### 1.1 任务定义

**三分类胆囊病变风险分层**: malignant(0) / benign(1) / no_tumor(2)

AI 定位为**风险分层工具**，核心目标:
1. **安全底线**: 恶性肿瘤零漏诊 (malignant sensitivity 尽可能高)
2. **临床价值**: 在安全底线之上，最大化息肉患者的"避免不必要手术率"
3. **灰区处理**: 通过 0~1 连续风险分数 + 可调阈值，适配不同临床场景

### 1.2 数据集

| 划分 | malignant | benign | no_tumor | 总计 |
|---|---:|---:|---:|---:|
| Train | 272 | 271 | 541 | 1,084 |
| Test | 114 | 114 | 228 | 456 |

- 患者级无泄露 (train/test)
- malignant **无病灶分割标注** (仅有胆囊 rectangle)
- malignant 文本覆盖仅 36%

### 1.3 技术要求 (来自 0414.txt)

1. 非对称误分类代价权重
2. 线性风险分数 0~1 + 可调阈值
3. 恶性无分割标签 → seg loss = 0
4. 保证一定数量的息肉被正确分类
5. 每个 batch 三类均衡

## 3. 各实验详细结果

### 3.1 Smoke-1 (Exp-A): 最小三分类基线

**脚本**: `20260414_task3_SwinV2Tiny_segcls_smoke_1.py`

**配置**:
- SwinV2-Tiny@256 + 4ch (RGB + Gallbladder ROI)
- 标准 3-class CE (class weight 按频率), 无代价敏感
- 无 ordinal score, 无阈值搜索
- 无 BalancedBatchSampler (随机 shuffle)
- Epochs: 30, BatchSize: 8, Backbone LR: 2e-5, Head LR: 2e-4

**最终测试 (Epoch 14 最优权重)**:

```
              precision    recall  f1-score   support

   malignant     0.8911    0.7895    0.8372       114
      benign     0.3558    0.5088    0.4188       114
    no_tumor     0.6927    0.5833    0.6333       228

   macro avg     0.6465    0.6272    0.6298       456
```

- Seg Dice: 0.8303 | Lesion IoU: 0.7122

**暴露的问题**:
- benign precision 极低 (35.6%), 大量其他类被误判为 benign
- no_tumor recall 不够 (58.3%), 很多息肉被误归为 benign/malignant
- 没有代价敏感 → 模型无法区分误分类的严重程度
- 没有连续风险分数 → 无法通过阈值灵活调整

## 4. 关键发现与分析

### 4.1 benign recall 是核心瓶颈

这是贯穿所有实验的核心问题:

| 实验 | benign recall | benign precision | benign F1 |
|---|---|---|---|
| Smoke-1 | **50.9%** | 35.6% | 41.9% |
| Exp-B (softmax) | 36.8% | 36.2% | 36.5% |
| Exp-B (ordinal, opt) | 24.6% | 35.9% | 29.2% |
| Exp-B (ordinal, def) | 28.1% | 38.1% | 32.3% |

**根因分析**: 
- benign 和 no_tumor 的风险分数高度重叠 (benign mean=0.229 vs no_tumor mean=0.167)
- 114 个 benign 样本中有 79~84 个被误分为 no_tumor
- 两者在超声影像上的视觉特征极为相似（都是良性病变）
- 代价敏感损失虽然保护了 malignant，但未能拉开 benign/no_tumor 的特征距离

### 4.2 代价敏感损失的效果

**正面效果**:
- malignant precision 从 89.1% (Smoke) → 95.7% (Exp-B ordinal)
- no_tumor recall 从 58.3% (Smoke) → 82.5% (Exp-B ordinal)
- 避免不必要手术率达到 82.5%

**代价**: 
- benign recall 反而下降 (50.9% → 24.6~36.8%)
- 代价矩阵迫使模型优先保护 malignant 和 no_tumor，benign 被"夹在中间"

### 4.3 Ordinal Score 的价值

- 提供了 0~1 连续风险分数，支持阈值灵活调整
- 默认阈值 (0.33, 0.66) 的 F1=0.6423 优于 val 搜索的 (0.37, 0.73) 的 F1=0.6305
- 阈值调整允许在 "malignant sensitivity vs 避免手术率" 之间权衡:
  - 放宽 t2 (0.73→0.66): malignant sensitivity 从 78.9% → 81.6%，但漏诊数仅少 1 个
  - 这说明 ordinal score 对恶性患者有较好的区分力

### 4.4 分割性能

| 实验 | Seg Dice | Lesion IoU |
|---|---|---|
| Smoke-1 | **0.8303** | **0.7122** |
| Exp-B | 0.7982 | — |

分割性能稳定，且 malignant seg loss 自动置零机制工作正常。

## 5. 下一步方向

### 5.1 优先: 重跑 Exp-C (使用 pretrained 权重)

Exp-C 因 HuggingFace SSL 问题回退到随机初始化，需要:
1. 解决 pretrained 权重下载/缓存问题
2. 重新训练完整 60 epochs
3. 预期: Seg-Guided Attention 聚焦病灶区域 + 10D 临床 metadata 提供先验知识，有望改善 benign recall

### 5.2 Exp-D: 加入文本分支

- BERT 编码超声描述文本
- 需处理 malignant 文本覆盖仅 36% 的问题 (可缺失模态)
- 参考 Exp#18 的 CrossAttention 架构

### 5.3 Exp-E: 完整三模态门控融合

- 整合图像 (Seg-Guided Attention) + 文本 (BERT) + 临床 metadata
- 门控三模态融合 (Gated Trimodal Fusion)
- 对标旧任务 Exp#18 的 F1=0.6846 (二分类)

### 5.4 可选: 改善 benign recall 的专项策略

- 增大 benign 在 BalancedBatchSampler 中的比例 (如 2:3:3)
- 调整代价矩阵中 benign→no_tumor 的惩罚 (当前 2.0 → 可增大到 3.0)
- 对比学习预训练 (拉大 benign vs no_tumor 特征距离)
- 二阶段: 先分 malignant vs non-malignant, 再分 benign vs no_tumor
