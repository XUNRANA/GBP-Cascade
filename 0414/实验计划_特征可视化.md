# 0414 特征可视化实验计划

> 本文档是 `实验计划.md` (训练路线图) 的**事后解释性补充**。
> 核心目标: 围绕当前 best 三分类模型诊断 **benign ↔ no_tumor** 特征纠缠, 并引入可解释的经典影像组学 biomarker。
> 参考需求源: `0421png.txt`, `实验计划.md §4.3 / §9 Exp-F`。

## 1. 背景与目标

### 1.1 为什么再画一轮图

当前 best 三分类模型 (`20260414_task3_SwinV2Tiny_segcls_3`) 在分类头前提取的 **320D 融合特征** 已做过一次 t-SNE:

- 输出: `0414/logs/feature_analysis_best_softmax_segcls3/tsne_pre_classifier_feat.png`
- silhouette = **0.115** (分离弱)
- 质心距离:
  - malignant ↔ benign = 8.76
  - malignant ↔ no_tumor = 9.85
  - **benign ↔ no_tumor = 1.92** ← 严重纠缠

结论: 恶性在特征空间已充分分开, 但**良性与息肉几乎重合**, 与 smoke 测试报告的 "benign recall=50% 是瓶颈" 相吻合。
单张 t-SNE 只能说明 "有纠缠", 无法回答:

- 纠缠发生在模型的哪一层?
- 是否存在某种模态 / 属性 (size、flow、morph) 能将它们切开?
- 换成可解释的经典特征 (纹理、形态) 是否有统计显著的 biomarker?

本计划回答上述三问, 为后续 Exp-F (SupCon) 或临床报告提供证据。

### 1.2 目标

| # | 目标 | 产出 |
|---|---|---|
| G1 | 观察特征**在模型内的演化**: 从浅层到深层, 类间分离何时建立, 何时良性/息肉开始可分 | Phase 1.1 多层 t-SNE 网格 |
| G2 | 对 best 模型的同一组样本, 用**多种投影方法**和**多维属性着色**诊断纠缠区域 | Phase 1.2 / 1.3 |

## 3. Phase 1 — 深度特征多视角分析

对象: `logs/20260414_task3_SwinV2Tiny_segcls_3/20260414_task3_SwinV2Tiny_segcls_3_best.pth` + test 集 456 张。

### 3.1 多层 t-SNE 网格 (8 面板)

在 `SwinV2SegGuidedOrdinalMetaModel.forward` 的下列节点注册 hook, **一次 forward 同时抓 8 份特征**, 每份独立做 t-SNE。

| # | 节点 | Shape | 池化 | 含义 |
|---|---|---|---|---|
| 1 | backbone `f0` | (B, 96, 64, 64) | GAP → 96 | SwinV2 最浅层, 纹理/边缘 |
| 2 | backbone `f1` | (B, 192, 32, 32) | GAP → 192 | 中低层, 局部结构 |
| 3 | backbone `f2` | (B, 384, 16, 16) | GAP → 384 | 中高层, 语义成型 |
| 4 | backbone `f3` | (B, 768, 8, 8) | GAP → 768 | 最深层, 全局语义 |
| 5 | `cls_proj(f2)` | (B, 256, 16, 16) | GAP → 256 | 分类投影后, **未加** seg-guided attention |
| 6 | seg-guided `cls_feat_img` | (B, 256) | (已池化) | **加了** 分割注意力后的图像特征 |
| 7 | 融合 `cls_feat` | (B, 320) | — | concat(image_256, meta_64) |
| 8 | `cls_head[0]` 输出 | (B, 128) | — | Penultimate, Linear+GELU 后 |

**实现要点**:

- 对 CNN 特征 (#1-#5) 使用 `adaptive_avg_pool2d(x, 1).flatten(1)` 得到固定维向量
- 所有 8 份特征都先 `StandardScaler` 再 t-SNE
- 同一 (random_state=42, perplexity=30) 保证可比
- 每个面板标题显示 `silhouette` 和 `benign↔no_tumor centroid L2`

**期望结果**: 浅层 (f0/f1) 三类几乎不可分; 深层逐步出现 malignant 岛; benign vs no_tumor 在所有层都可能仍纠缠, 这恰是下一步 Exp-F 的动机。

### 3.2 投影方法对比 (320D cls_feat)

同一 320D 特征, 并排三图:

| 方法 | 参数 | 用途 |
|---|---|---|
| PCA | n=2 | 线性可分性基线 |
| t-SNE | perplexity=30, init=pca | 局部结构 |
| UMAP | n_neighbors=15, min_dist=0.1 | 兼顾局部 + 全局 |

量化指标: silhouette, KMeans(k=3) 结果与真标签的 ARI。

### 3.3 多属性上色 (3 × 3 网格, 共用一张 t-SNE 坐标)

同一张 t-SNE 的 2D 坐标, 换 9 种颜色编码。用来同时回答 "是谁?""预测对不对?""模型有多确信?""临床属性是否已经隐含在几何结构里?"。

| 行 \ 列 | 1 | 2 | 3 |
|---|---|---|---|
| **分类** | 真实类 (3 色离散) | 预测类 (3 色离散) | 正确/错误 (绿/红二值) |
| **置信** | ordinal score s (RdYlGn 连续) | softmax max prob (viridis 连续) | 预测熵 (magma 连续) |
| **临床** | size_bin (0/1/2) | flow_bin (0/1/2) | morph_bin (0/1/2) |

**输出解读模板** (写到 JSON + Markdown 摘要):

- 若 "临床行" 的某个属性与真标签行呈现相似 pattern, 说明模型已经隐式编码该属性
- 若 "预测熵" 高亮区域正好位于 benign ↔ no_tumor 边界, 说明模型自知模糊, 适合交回医生 (对应黄灯)

### 3.4 类间 / 类内距离分布

4 张子图, 布局 2 × 2:

- **Violin**: 每类内部每个样本到类质心的 L2 距离分布 (3 把小提琴并排)
- **Box + swarm**: 6 组两两样本距离 (mm / mb / mn / bb / bn / nn); 在 bn 组放大查看离群点
- **Heatmap**: 3 × 3 质心欧氏距离矩阵, 单元格数字标注, 双色阶
- **Stacked Histogram**: 每样本 silhouette score 直方图, 按类 stacked

### 3.5 判别性维度分析 (320 维逐维)

- **Bar**: 对每一维独立做 3 类 ANOVA F 检验, 取 top-30 画条形图
- **KDE 叠加**: top-6 维度按类别的概率密度 (3 条曲线叠在一张子图上, 共 6 子图)
- **Corr heatmap**: top-20 维度的 Pearson 相关矩阵 (识别冗余)

判读: 若 top-20 维度相关性高, 说明模型实际上只用了少数正交方向 → 支持后续的维度约减或对比学习正则。

### 3.6 "灰区" 临床复核 (单张图, 对应 §4.3 三段式)

基于 3.3 的 ordinal score 面板进一步注释:

- 在 2D 坐标上画两条等值线 (s = t1, s = t2) 或对应等值带
- 分三类 marker:
  - `o` 正常点
  - `*` malignant → no_tumor 漏诊点 (临床红线)
  - `x` 高置信度 (max prob > 0.9) 但预测错的点
- 色条用 ordinal score

输出一张 "临床风险分布图", 直接对标 `实验计划.md §4.3` 表格中的绿灯/黄灯/红灯区。

## 4. Phase 2 — 经典影像组学特征 (PyRadiomics)

### 4.1 环境

```bash
pip install pyradiomics umap-learn
# SimpleITK 已装 (版本不限), matplotlib / seaborn / scipy / sklearn / pandas 都已装
```

### 4.2 特征提取管线

对 **test 集 456 张** (与 Phase 1 同一批样本, 可直接 join) 执行:

1. 读 PNG → PIL.Image → numpy (H, W) uint8 灰度
2. 读同名 `.json` → `generate_gallbladder_mask(shapes, W, H)` 得到胆囊 rectangle mask (所有类都有)
3. 同步生成 `generate_lesion_mask` 作为次 ROI, 仅对有 polygon 的 benign / no_tumor (malignant 没有 lesion polygon)
4. `SimpleITK.GetImageFromArray` 包装图像和 mask → `RadiomicsFeatureExtractor.execute(img, mask)`
5. 从返回的 OrderedDict 中过滤 `diagnostics_*` 字段, 只保留纯特征 (约 107 个)
6. 汇总为一张 CSV:
   ```
   image_path, patient_id, true_class, pred_class, ordinal_score,
   original_firstorder_*, original_shape2D_*, original_glcm_*, original_glrlm_*,
   original_glszm_*, original_gldm_*, original_ngtdm_*
   ```

**参数固定** (写入 `extractor_config.yaml`, 保证复现):

```yaml
imageType:
  Original: {}          # 不跑 wavelet / LoG, 保持 107 维
setting:
  binWidth: 25          # 灰度 bin 数
  force2D: true         # 超声是 2D
  force2Ddimension: 0
  interpolator: sitkBSpline
  normalize: true       # 图像内部归一化
  normalizeScale: 100
  label: 1
featureClass:
  firstorder:
  shape2D:
  glcm:
  glrlm:
  glszm:
  gldm:
  ngtdm:
```

**异常处理**:

- mask 像素数 < 32 → 记录到 `skipped.csv`, 不参与后续统计
- 某个特征值 NaN/Inf → 同列中位数填充并记录
- 若某一特征族全为 NaN (极小 ROI), 警告后跳过该特征

### 4.3 降维可视化

对 456 × 107 矩阵 **Z-score 标准化** (对量纲差异至关重要), 然后 4 张图:

| 图 | 说明 |
|---|---|
| `2.3_pca_scree.png` | 累积 variance 曲线, 标出 80% / 90% / 95% 阈值 (对应 PC 数量) |
| `2.3_pca_2d.png` | PC1 vs PC2 散点, 3 类着色; 补一幅 PC1 vs PC3, 右侧附加载荷表 |
| `2.3_tsne.png` | perplexity=30, random_state=42, 与 Phase 1 参数一致, 便于横向比较 silhouette |
| `2.3_umap.png` | n_neighbors=15, min_dist=0.1 |

每张图标题中标注 silhouette。写一段对比段落: "深度 320D silhouette = 0.115, 经典 107D silhouette = ?" 放进 `report.json`。

### 4.4 单特征统计检验 (107 维逐特征)

对每一维:

1. **3 类 ANOVA F 检验** (`scipy.stats.f_oneway`) → `p_anova`, `F_stat`
2. **两两 Mann-Whitney U 检验** (`scipy.stats.mannwhitneyu`, two-sided, 非参):
   - `p_mal_ben`
   - `p_mal_no`
   - `p_ben_no` ← **重点: benign vs 息肉**
3. **效应量 Cliff's delta** (非参, 两组样本无假设分布):
   - `delta_mal_ben`, `delta_mal_no`, `delta_ben_no`
   - (|δ|<0.147 为小效应, 0.147 ~ 0.33 中等, ≥ 0.474 大)
4. **FDR 校正** (Benjamini-Hochberg, α=0.05) 对 anova 和三个两两 p 值分别做 → `q_*`
5. 每类均值 / 标准差

输出 `biomarker_table.csv`:

```
rank,feature,family,p_anova,q_anova,
p_mal_ben,q_mal_ben,delta_mal_ben,
p_mal_no,q_mal_no,delta_mal_no,
p_ben_no,q_ben_no,delta_ben_no,
mean_mal,std_mal,mean_ben,std_ben,mean_no,std_no,
sig_ben_no_fdr,interpretation_zh
```

按 `q_ben_no` 升序取 top-20。 `interpretation_zh` 自动填文字, 例如:

> "`glcm_Contrast`: benign 均值 0.45 ± 0.12, no_tumor 0.28 ± 0.08; benign 显著更高 (q=1.3e-5, Cliff δ=0.52, 大效应); **候选 biomarker**"

### 4.6 ROI 对比实验 (可选, 仅 benign + no_tumor 子集)

分别用 **胆囊 rectangle** 和 **lesion polygon** 两套 mask 各提一份 107 维, 比较:

- 整体 silhouette (降维图)
- benign vs no_tumor 的 top-10 biomarker q 值分布
- 得出结论: 对 ben vs no 区分, 胆囊 ROI 够用 / 还是必须要病灶级 ROI

## 5. Phase 3 — 深度 × 经典联合分析 (可选)

### 5.1 联合 t-SNE

1. 对同一组 456 个样本, 把 deep (320D) 和 radiomics (107D) 各自 Z-score 后 concat → 427D
2. 再做一次 t-SNE / UMAP, 对比 silhouette
3. 若联合后仍 < 单独 deep, 说明 radiomics 里没有 DL 漏掉的新信息; 反之则可把 radiomics 作为辅助特征加进模型

### 5.2 CCA (Canonical Correlation Analysis)

- `sklearn.cross_decomposition.CCA(n_components=10).fit(deep, radiomics)`
- 输出前 5 个 canonical correlation 系数 + 载荷
- 对前 3 个 canonical 方向画 deep_score vs radiomics_score 散点, 若高相关, 说明 DL 已经隐式学到了该方向上的经典特征

## 9. 与既有实验计划的关系

- `实验计划.md` = 训练/建模路线图 (Exp-A ~ Exp-F), **本计划不动**
- `实验计划_特征可视化.md` (本文件) = 解释性分析, 是训练计划的**事后补充**:
  - 诊断 benign ↔ no_tumor 特征纠缠 → 决定是否做 Exp-F (SupCon)
  - 提供 biomarker 列表 → 下一步可考虑做特征工程或作为辅助损失
  - 给论文 Figure 章节提供素材 (多层 t-SNE、属性着色、biomarker 火山图、临床灰区图)
