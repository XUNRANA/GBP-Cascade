# 0414 三分类联合分类+分割实验计划

## 1. 项目背景与临床定位

### 1.1 数据集的临床本质

本数据集中的所有病例均为**经超声、CT、MRI 重重检查后仍无法排除肿瘤，最终选择手术并获得病理结果**的患者。其中：

- 大部分中晚期恶性肿瘤影像特征明显，模型也能较准确识别
- 剩余手术患者中，极少数为恶性，少数为良性，**大多数为不需要手术的普通息肉**
- 这些息肉患者本可随访观察，但因医生无法确定排除肿瘤而被迫手术（"防御性医疗"）

因此，这是一个**充满临床灰色地带的"骨头数据集"**，不是特征明显的理想数据。

### 1.2 AI 的定位

**不要把 AI 定位为完美的病理分类器，而应定位为"风险分层与安全过滤工具（Risk Stratification & Triage Tool）"。**

核心目标：
1. **安全底线**：保持恶性肿瘤的极高敏感度（接近零漏诊）
2. **临床价值**：在安全底线之上，尽可能多地将真正的普通息肉"捞出来"，让他们避免不必要的手术
3. **灰区处理**：对模型无法确定的病例（良性 vs 息肉边界），输出"高不确定性"标记，交回医生决策

**一句话**：AI 的成功不是"分类有多准"，而是"在不漏诊任何恶性的前提下，帮助多少息肉患者免挨一刀"。

## 4. 三分类任务定义

### 4.1 严重程度排序映射

三个类别具有**临床严重程度梯度**：

```
no_tumor(息肉) < benign(良性肿瘤) < malignant(恶性肿瘤)
```

模型内部 ordinal remap：

| 类别 | Excel 原始标签 | 模型内部编码 | 风险等级 |
|---|---|---:|---|
| no_tumor | 2 | 0 | 低风险 |
| benign | 1 | 1 | 中风险 |
| malignant | 0 | 2 | 高风险 |

### 4.2 线性风险分数

分类不仅输出 3 类 softmax，**还要同时输出一个 0~1 的连续风险分数 `s`**（对应 `0414.txt` 第2条要求）：

- `s` 越小 -> 越偏 no_tumor（低风险，可随访）
- `s` 中间 -> 越偏 benign（中风险，需关注）
- `s` 越大 -> 越偏 malignant（高风险，建议手术）

推理时用两个阈值 `(t1, t2)` 做最终分类：

```
s < t1       -> no_tumor（建议随访）
t1 <= s < t2 -> benign（建议密切关注）
s >= t2      -> malignant（建议手术）
```

初始阈值：`t1=0.33, t2=0.66`

**最终阈值必须在验证集上搜索，不能用 test 集选择。** 可配合临床需求设置非对称阈值，例如 `(t1=0.15, t2=0.50)` 以极大降低恶性漏诊率。

### 4.3 临床区间解读

根据风险分数，划分三个临床决策区间：

| 区间 | 分数范围 | 临床建议 | 安全要求 |
|---|---|---|---|
| 低风险（绿灯） | s < t1 | 建议随访，避免手术 | 需极高 NPV（>95%） |
| 不确定（黄灯） | t1 <= s < t2 | 建议增强检查或专家会诊 | 将模棱两可的病例交回医生 |
| 高风险（红灯） | s >= t2 | 建议手术 | 需极高敏感度（>95%） |

## 5. 模型架构设计

### 5.1 设计原则

采用**端到端多任务统一框架**，不使用两阶段级联，避免误差传播（error propagation）问题。

核心思路：**保留 Exp#18 的联合学习主线**，但做关键适配：
1. 第4通道改为 gallbladder ROI mask（非 lesion mask）
2. 分割任务改为条件监督（恶性 loss 置零）
3. 分类从二分类改为三分类 + ordinal score
4. 文本分支支持缺失模态

### 5.2 推荐架构

```
SwinV2-Tiny@256
  + 4ch 输入 (RGB + Gallbladder ROI mask)
  + UNet 分割解码器 (条件 Lesion Seg)
  + Seg-Guided Attention Pooling
  + 3-class Softmax Head
  + 1D Ordinal Score Head (sigmoid -> 0~1)
  + 10D 临床特征编码器 (可选)
  + BERT 文本分支 + Cross-Attention (可选，支持缺失)
  + Gated Multimodal Fusion (可选)
```

### 5.3 各组件说明

**Backbone**：SwinV2-Tiny，4通道输入（第1层 patch embedding 扩展），输出 4 级特征 f0~f3。

**分割解码器**：UNet 风格 decoder (f3->f2->f1->f0 + skip connections)，输出 2 类 lesion mask。
- 所有三类参与前向传播
- 仅 `has_lesion_mask=True` 的样本参与 loss 计算（malignant 自动跳过）

**Seg-Guided Attention**：用分割 softmax 概率图作为 attention weight，对 f2 特征做加权池化，得到 256D image feature。分割越准，分类特征越聚焦于病灶区域。

**Ordinal Score Head**：独立的 `Linear(256->1) + Sigmoid`，输出 0~1 风险分数。

**Metadata Encoder**：`Linear(10->96) -> LN -> GELU -> Dropout -> Linear(96->96)`，10D 临床特征（age, gender, size_mm, size_bin, flow_bin, morph_bin, echo_type, lesion_count, wall_thickness, us_diameter）。

**Text Branch**：BERT-base-chinese (frozen)，[CLS] -> 128D projection。通过 Cross-Attention 增强 f2 空间特征。对无文本样本输入空串，由 gate 自动降权。

**Gated Fusion**：concat(img_256D + text_128D + meta_96D) -> gate network -> 加权求和 -> 256D fused feature。

### 5.4 为什么这样设计

| 设计决策 | 原因 |
|---|---|
| 端到端而非两阶段 | 避免恶性漏诊的误差级联传播 |
| GB ROI 作第4通道 | 三类都有 GB rectangle，不依赖 lesion polygon |
| 条件分割 loss | 恶性无 lesion 标签，masking loss 解决标签不对称 |
| Ordinal score | 三类有严重程度梯度，线性分数支持阈值调整 |
| Gate fusion | 恶性文本仅36%覆盖，必须支持缺失模态 |

## 6. 损失函数设计

### 6.1 总损失公式

```
L_total = L_cls + lambda_ord * L_ord + lambda_seg * L_seg
```

推荐初值：`lambda_ord=0.5, lambda_seg=1.0`

### 6.2 分类损失：代价敏感交叉熵

严格对应 `0414.txt` 第1条要求。不同误分类方向给予不同惩罚权重：

**代价矩阵**（`0414.txt` 原文对应关系）：

| 真实 \ 预测 | no_tumor (息肉) | benign (良性) | malignant (恶性) |
|---|---:|---:|---:|
| **no_tumor** | 0.0 | 1.0 (中等偏小) | 1.5 (中等偏小) |
| **benign** | 2.0 (中等偏大) | 0.0 | 1.0 (中等偏小) |
| **malignant** | **4.0 (最大)** | 1.0 (中等偏小) | 0.0 |

解读：
- `malignant -> no_tumor`（恶性漏诊为息肉）：**代价最大 = 4.0**，临床灾难
- `benign -> no_tumor`（良性误判为息肉）：代价中等偏大 = 2.0，可能延误治疗
- `no_tumor -> benign`（息肉误判为良性）：代价最小 = 1.0，过度关注但可接受
- `no_tumor -> malignant`（息肉误判为恶性）：代价中等偏小 = 1.5，过度手术
- `benign <-> malignant`（良恶性互判）：代价中等偏小 = 1.0

实现方式：逐样本加权 CE loss，对每个样本根据其真实标签和预测分布查表计算惩罚。

### 6.3 排序损失 (Ordinal Loss)

用于约束 ordinal score `s` 符合严重程度梯度：

```
对于 ordinal remap: no_tumor=0, benign=1, malignant=2

方式一 (CORAL): 将3分类转为2个二分类子任务
  P(s >= 1) = sigma(w1^T * feat + b1)  -- 是否 >= benign
  P(s >= 2) = sigma(w2^T * feat + b2)  -- 是否 >= malignant
  L_ord = BCE(P(s>=1), I[y>=1]) + BCE(P(s>=2), I[y>=2])

方式二 (EMD): Earth Mover's Distance
  L_ord = sum_k |CDF_pred(k) - CDF_true(k)|^2

方式三 (Smooth L1):
  target_s = y / (C-1)  -- no_tumor=0.0, benign=0.5, malignant=1.0
  L_ord = SmoothL1(s, target_s)
```

推荐先用**方式三（Smooth L1）**，简单直接，与线性分数要求最吻合。

### 6.4 分割损失

```
L_seg = L_ce_seg + L_dice_seg
```

- `L_ce_seg`：2类 CrossEntropy，权重 `{bg: 1.0, lesion: 5.0}`
- `L_dice_seg`：Binary Dice Loss
- **仅 `has_lesion_mask=True` 的样本参与计算**，malignant 自动跳过

### 6.5 可选：监督对比损失 (SupCon)

对比损失不替代 CE，而是作为**特征空间正则化**：

```
L_total = L_cls + lambda_ord * L_ord + lambda_seg * L_seg + lambda_con * L_supcon
```

在特征空间中拉近同类、推远异类，特别有助于改善 benign vs no_tumor 的特征纠缠问题。可在 Exp-F 中尝试。

## 7. 训练策略

### 7.1 数据划分

- 官方 train/test 不动
- 从 train 中切出**患者级 val**：`train_inner : val = 85 : 15`
- 按患者分层切分，保证三类比例一致
- **test 只做最终一次评估**

### 7.2 Batch 组成（对应 `0414.txt` 第5条）

要求每个 batch 都包含三类，且尽可能保持均衡比例。

`batch_size=8` 时固定为：

| malignant | benign | no_tumor |
|---:|---:|---:|
| 2 | 2 | 4 |

实现：自定义 `BalancedBatchSampler`，每个 batch 按固定比例从各类中采样，不依赖纯 class_weight。

### 7.3 训练配置

| 参数 | 值 | 说明 |
|---|---|---|
| img_size | 256 | 与 Exp#18 一致 |
| backbone_lr | 2e-5 | 预训练 backbone 慢学习 |
| head_lr | 2e-4 | 新 head 快学习 |
| weight_decay | 0.05 | |
| warmup_epochs | 5~10 | 简单模型5, 复杂模型10 |
| num_epochs | 50~100 | 根据复杂度调整 |
| grad_clip | 1.0 | |
| AMP | True | |
| label_smoothing | 0.1 | |

### 7.4 早停与模型选择

- 用 **val macro-F1** 选 best epoch（而非 test）
- 用 **val** 搜索最优 `(t1, t2)` 阈值
- test 只做最终一次评估，报告最终结果

## 8. 评估指标体系

### 8.1 分类基础指标

| 指标 | 说明 |
|---|---|
| macro-F1 (3-class) | **主指标**，三类平均 |
| per-class Precision / Recall / F1 | 单独盯每类表现 |
| malignant -> no_tumor 错分数 | **临床安全红线** |
| Accuracy | 参考 |

### 8.2 临床核心指标

| 指标 | 临床含义 | 目标 |
|---|---|---|
| **malignant Sensitivity** | 恶性肿瘤被正确识别的比例 | >= 95%, 理想 >= 99% |
| **NPV (阴性预测值)** | AI说"息肉建议随访"时，真正是息肉的概率 | >= 95% |
| **避免不必要手术率** | 在恶性零漏诊前提下，能正确识别多少息肉 | 越高越好 |

"避免不必要手术率"的计算方式：
1. 设定阈值使 malignant sensitivity = 100%（或 99%）
2. 在此阈值下，统计被正确分类为 no_tumor 的息肉患者比例
3. 这些患者就是 AI 帮助避免不必要手术的群体

### 8.3 高级评估指标

| 指标 | 说明 | 论文价值 |
|---|---|---|
| **DCA (Decision Curve Analysis)** | 在不同风险阈值下，AI 辅助决策的净获益 | 顶级医学期刊黄金标准 |
| **NRI (Net Reclassification Improvement)** | AI 比纯医生经验能多大程度纠正错误分类 | 量化 AI 临床增益 |
| **校准曲线 (Calibration Curve)** | 模型预测概率与实际概率的吻合度 | 高置信度输出是否可信 |
| **ROC/AUC** | 各类别的 ROC 曲线和 AUC 值 | 标准报告指标 |

### 8.4 分割指标

| 指标 | 范围 |
|---|---|
| Lesion IoU / Dice | 仅在有 lesion polygon 的样本上报 |
| Gallbladder IoU / Dice | 三类都报（如使用 GB seg head） |

### 8.5 报告层级

- **image-level**：逐图像评估
- **patient-level**：同一患者多张图的风险分数取平均后判类（更贴合临床）

## 9. 实验路线图

### 总体原则

**不要一步上最复杂模型。** 逐步增加组件，每步验证增量收益。

### Exp-A：三分类最小可运行基线（已完成 Smoke 测试）

**目标**：跑通三分类联合框架

| 组件 | 配置 |
|---|---|
| Backbone | SwinV2-Tiny@256, 4ch (RGB + GB ROI) |
| 分割 | Lesion head, malignant loss masking |
| 分类 | 3-class softmax (标准 CE) |
| Meta/Text | 无 |
| Ordinal | 无 |

**已有结果**：F1=0.6298, malignant recall=79%, benign recall=50%

### Exp-B：代价敏感分类 + 线性风险分数

**目标**：直接响应 `0414.txt` 的核心要求

在 Exp-A 基础上增加：
- **Cost-sensitive CE**（代价矩阵如第6.2节）
- **Ordinal score head**（Smooth L1 loss）
- 在 val 上搜索最优 `(t1, t2)`
- 自定义 `BalancedBatchSampler`

**预期**：
- malignant -> no_tumor 错分显著减少
- 风险分数 `s` 与严重程度正相关
- 可通过调整阈值适配不同临床场景

### Exp-C：加入临床结构化特征

在 Exp-B 基础上增加：
- 10D 临床元数据（age, gender, size_mm, flow_bin, morph_bin 等）
- MLP 编码器 + late fusion

**预期**：
- 良性/恶性边界更稳定
- 临床先验知识辅助区分

### Exp-D：加入文本分支（可缺失模态）

在 Exp-C 基础上增加：
- BERT-base-chinese (frozen) 文本编码
- Cross-Attention 增强图像特征
- 对无文本样本（尤其 malignant 64%无文本），文本分支输入空串，由 gate 降权

**注意**：malignant 文本覆盖仅 36%，不能把文本当主模态。

### Exp-E：完整三模态门控融合

最终形态，对标旧 Exp#18：

```
SwinV2-Tiny@256
  + 4ch (RGB + GB ROI)
  + UNet Lesion Seg (条件 loss)
  + Seg-Guided Attention
  + BERT Cross-Attention
  + 10D Meta Encoder
  + Gated Trimodal Fusion
  + 3-class Softmax + Ordinal Score
  + Cost-Sensitive Loss
```

### Exp-F（可选）：监督对比学习预训练

**目标**：解决 benign vs no_tumor 特征纠缠

两阶段训练：
1. **Stage 1 (表征学习)**：移除分类头，用 SupCon loss 在 backbone 上训练特征空间
2. **Stage 2 (微调)**：冻结 backbone，接分类/分割头，用 CE + ordinal loss 微调

**适用条件**：Exp-B~E 的 benign recall 仍然不理想时尝试。
