## 2. SupCon 是什么

SupCon（Supervised Contrastive Learning）核心思想：

1. 同类样本在特征空间要靠近。
2. 异类样本在特征空间要远离。

对每个样本 `i`，把同类样本集合记为 `P(i)`，损失可写为：

`L_i = - (1 / |P(i)|) * Σ_{p∈P(i)} log( exp(sim(z_i,z_p)/τ) / Σ_{a!=i} exp(sim(z_i,z_a)/τ) )`

其中：

1. `z` 是归一化后的 embedding。
2. `sim` 通常用点积（归一化后等价余弦相似度）。
3. `τ` 是 temperature，越小越“尖锐”。

这和普通 CE 的区别是：CE 只约束“分类正确”；SupCon 额外约束“特征几何结构”。

## 3. 你项目里的 SupCon 实现（代码级）

### 3.1 特征从哪里来

在 `scripts/20260414_task3_SwinV2Tiny_segcls_4.py`：

1. 先用分割引导注意力得到图像分类特征 `cls_feat`（`B,256`），见 `566-567` 行。  
2. 经过 `proj_head` 做投影：`256 -> 128 -> 128`，见 `540-543` 行。  
3. 对投影特征做 `L2 normalize`，见 `570-571` 行。  

这一步输出 `proj_feat`，并在 `forward` 返回（`583` 行）。

### 3.2 SupConLoss 怎么算

`SupConLoss` 在 `167-214` 行实现，关键步骤：

1. 相似度矩阵：`sim = features @ features.T / temperature`（`190` 行）。  
2. 正样本 mask：同标签且去掉对角（`193-198` 行）。  
3. 稳定化：减每行最大值（`200-201` 行）。  
4. 计算 log-softmax 形式对比概率（`204-205` 行）。  
5. 对每个样本的正样本对取平均（`208-212` 行），再取负均值（`213` 行）。  

### 3.3 SupCon 如何并入总损失

`SegClsOrdinalSupConLoss` 在 `227-272` 行：

1. `cls_loss`：Focal + Cost-sensitive（`255` 行）。  
2. `ord_loss`：Ordinal regression（`257` 行）。  
3. `con_loss`：SupCon（`259` 行）。  
4. 总损失：`seg + λ_cls*cls + λ_ord*ord + λ_con*con`（`268-271` 行）。  

### 3.4 本次消融怎么关掉 SupCon

在 `scripts/20260420_task3_SwinV2Tiny_segcls_4_nocon.py`：

1. 仅将 `lambda_con` 改为 `0.0`（`823` 行）。  
2. 其他配置保持不变（batch、代价矩阵、ordinal 目标、网络结构一致）。  

注意：日志里仍会打印 `con=`，因为 `con_loss` 仍被计算并记录，但乘以 `lambda_con=0` 后不会进入总损失梯度。

## 6. 为什么会出现这种结果（机制分析）

### 6.1 SupCon 约束的是“图像分支”而不是“最终融合特征”

实现上，SupCon用的是 `cls_feat(256D)` 的投影（`566-571` 行），  
而最终分类头输入是 `fused_feat = cls_feat + meta_feat`（`574-581` 行）。

这会导致：

1. SupCon 优化目标与最终决策特征不完全一致。  
2. 当 metadata 对分类有贡献时，SupCon 对图像分支的几何约束不一定等价于最终分类收益。  

### 6.2 小 batch 下 SupCon 的统计噪声较大

当前 batch 是 `2:3:3`（总 8）。  
对 malignant 样本来说，每个 anchor 仅有 1 个正样本，这会让对比估计更不稳定，梯度噪声更大。

### 6.3 目标冲突：分类总体 vs 临床安全

从结果看：

1. 无 SupCon 的 Macro-F1 更高（总体分类好一点）。  
2. 但 malignant sensitivity 与 benign recall 下降（临床偏好方向变差）。  

这说明 SupCon 在这里更像在“拉类别边界形状”，优先了安全/平衡，而不是单纯追分。

### 6.4 Seg Dice 降低的信号

无 SupCon 的 Dice 从 `0.8497` 降到 `0.8024`，幅度较大。  
虽然 SupCon 不直接作用 segmentation head，但共享 backbone，特征分布改变会影响分割分支。

---

## 7. 如何继续做得更稳（建议）

1. 做 `lambda_con` 网格：`0.0 / 0.05 / 0.1 / 0.2 / 0.3`，而不是只做开关。  
2. 增加多随机种子复现实验（至少 3 个种子），确认不是单次波动。  
3. 试一版“在 fused_feat 上做 SupCon”作为对照，验证是否存在目标错位。  
4. 若临床优先，保留 SupCon 并联合阈值调优；若追求纯 Macro-F1，可降低或关闭 SupCon。
