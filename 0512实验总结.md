## 一、临床优先级与 5 底线

### 1.1 用户三条核心诉求 (按重要度)

| 优先级 | 诉求 | 临床意义 |
|:---:|---|---|
| ① | **mal → low = 0** (硬约束) | 绝不能把恶性当低风险随访 — 漏诊会致死 |
| ② | **ben → low 尽量低** | 良性息肉本质上仍是肿瘤,需要管理,不该和正常人混为一档 |
| ③ | **nt → low 尽量高** | 真正没病的人锁低风险池,节省随访资源 |

| 软底线 | nt_to_high | ≤ 5% | (减少正常人假警报) |
| 软底线 | nt_to_low | ≥ 80% | ③ |
| 软底线 | ben_to_low | ≤ 10% | ② |

## 四、10 实验逐一深入分析

### 实验 1 ── 0502 baseline (SwinV2Tiny ordinal trimodal)

**设计动机**: 第一版完整 pipeline,作为后续所有实验的参照基线。

**架构**: SwinV2-Tiny 4ch (RGB+gallbladder mask) + Seg-Guided Attention +
BERT-frozen + 10D 临床特征 + Gated Trimodal Fusion → 256D → risk_head (1D)

**Loss**: `L = BCE_ord(risk, ord_target) + (CE + Dice)(seg)`,ordinal_targets=[1.0, 0.5, 0.0]

**决策**: 单 score 双阈值 (t_low=0.29, t_high=0.5),val grid-search 选

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  114    0    0 ]   ← 100% 抓住所有 mal ✅
   ben    [   18   61   35 ]   ← 35/114=30.7% 良性误为低风险 ❌
   nt     [   25   59  144 ]   ← 25/228=11% 正常人假警报为 high ❌
```

**5 底线**:mal→low=0 ✅ │ high_R=1.0 ✅ │ nt→high=11.0% ❌ │ nt→low=63.2% ❌ │ ben→low=30.7% ❌  **(2/5)**

**优点**: 高敏感 (mal 100% 抓住),硬底线稳。

**缺点**: 三档之间太"均匀",没有特别压制任何一类。3 个软底线全不过。

**临床读法 (假设 100 个病人:25 mal + 25 ben + 50 nt 上线部署)**:
- mal: 25 全部到 high → 100% 召回,正确
- ben: 7 假高 + 13 中 + 5 低 → 20% 良性息肉跑去 low 池,临床上等于"暂不管"
- nt: 5 假高 + 13 中 + 32 低 → 10% 正常人被惊到要做侵入性检查

### 实验 2 ── 0502 A SupCon

**设计动机**: 在 fusion 256D 表示上加 Supervised Contrastive 损失,期望同类样本聚拢、
不同类拉远,从而让阈值更易分。

**架构**: baseline 不变,新增 SupCon head:fused→128→L2 normalize,
对 256D fusion 加 batch 内对比损失。

**Loss**: `L = BCE_ord + λ_supcon · L_supcon + (CE + Dice)(seg)`

**决策**: 同 baseline 单 score 双阈值 (t_low=0.25, t_high=0.5)

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  114    0    0 ]   ← 高敏感保持
   ben    [   26   57   31 ]   ← ben→low 27.2% (比 baseline 30.7% 略低)
   nt     [   22   70  136 ]   ← nt→low 59.6% (比 baseline 63.2% 略低)
```

**5 底线**: mal→low=0 ✅ │ high_R=1.0 ✅ │ nt→high=9.6% ❌ │ nt→low=59.6% ❌ │ ben→low=27.2% ❌  **(2/5)**

**优点**: ben→low 小幅改善 (30.7→27.2%)。

**缺点**: SupCon 在 ultrasound 这类高个体差异+小批量 (batch=8) 的场景下信号弱,
基本和 baseline 同档,改善有限。

**临床读法**: 与 baseline 几乎相同,选择 A 的额外计算成本不值得。

---

### 实验 3 ── 0502 B Calib **(0502 winner, 4/5)**

**设计动机**: baseline ordinal target 用 [1.0, 0.5, 0.0] 时,ben 期望分数 0.5 离 nt 的 0 不够远,
calibration 后 nt 也压不到接近 0。改为 **非对称 ordinal target** [1.0, **0.35**, 0.0] +
post-hoc **温度校准** (LBFGS fit val),压低 nt 分数,把 nt 干净分到 low。

**架构**: baseline 一致,只改 loss 的 ordinal_targets。

**Loss**: `L = BCE_ord(risk, [1.0, 0.35, 0.0][label]) + (CE + Dice)(seg)`

**决策**: 单 score 双阈值,但应用 **T=0.3664** 温度缩放后 (logit/T → sigmoid),
t_low=**0.07** (极低), t_high=0.5。Brier 从 0.039 → 0.023 (减半)。

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  113    1    0 ]   ← 1 个 mal 跌到 medium,high_R=0.991 仍达标
   ben    [    3   26   85 ]   ← ❌ 85/114=74.6% 良性跑到 low — 灾难
   nt     [    3   21  204 ]   ← ✅ 204/228=89.5% 正常人到 low — 优秀
```

**5 底线**: mal→low=0 ✅ │ high_R=0.991 ✅ │ nt→high=1.3% ✅ │ nt→low=89.5% ✅ │ ben→low=74.6% ❌  **(4/5)**

**优点**: 4/5 底线全过,**nt→low 史上最高**。high_precision 也跃升到 0.95+。

**缺点**: ben target 0.35 + 温度 0.366 把 ben 的 score 也一同压低 → **75% 良性息肉
跑到 low**,临床上这些病人本该"3-6 月随访",现在被定为低风险。

**临床读法 (100 病人:25 mal + 25 ben + 50 nt)**:
- mal: 25 全 high → 100% 召回 ✅
- ben: 1 高 + 6 中 + **18 低** → 18 个良性息肉漏管 (临床风险:5-10% 转癌)
- nt: 1 高 + 4 中 + 45 低 → 90% 正常人省事 ✅ — 节流明显

**B 的核心矛盾**: nt 干净分到 low 的代价就是 ben 也被压到 low,**ben target 0.35 设得不够远**。

### 实验 4 ── 0502 C twostage (失败案例)

**设计动机**: 先 stage1 学连续 risk (mal=1.0, ben=0.5, nt=0),再 stage2 学 mal vs ben。
**期望**: stage2 在 polyp 内细分,ben 被锁在 medium。

**架构**: risk_head (BCE,ordinal target [1.0,0.5,0.0]) + ben_mal_head (BCE on mal vs ben)

**Loss**: `L = BCE_ord + λ_bm · BCE(ben_mal | polyp) + seg`

**决策**: 双阈值 (t_high=0.4 for risk → high; t_med=0.5 for ben_mal → med vs low)

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  114    0    0 ]   ← 高敏感保住
   ben    [   58   25   31 ]   ← ❌ 58 个良性误为 high!
   nt     [   58   57  113 ]   ← ❌ 58 个正常假高,nt→low 仅 49.6%
```

**5 底线**: mal→low=0 ✅ │ high_R=1.0 ✅ │ nt→high=25.4% ❌ │ nt→low=49.6% ❌ │ ben→low=27.2% ❌  **(2/5)**

**优点**: 没有特别优点 — 是十个里最差之一。

**缺点 (根因)**:
- stage2 val AUC **只有 0.665** (随机抛硬币 0.5)
- stage1 输出已经把 mal/ben 区分掉 → stage2 拿到的输入特征里 mal/ben 信号几乎没了 → 学不动
- 设计上 stage1 和 stage2 学的是**相关任务**,信号被前面截留

**临床读法**: 大量正常人和良性息肉同时跑到 high 池 → 假警报严重 → 系统不可用。

### 实验 5 ── 0502 D modal (多模态消融)

**设计动机**: 验证临床特征 (10D metadata) + BERT 文本对最终性能的贡献。
具体做了 modality 输入的对照修改 (D 表示 modality)。

**架构**: 与 baseline 同,只改 fusion 的某个 modality 设置 (具体是关闭/替换某个 branch)。

**Loss**: 同 baseline

**决策**: 单 score 双阈值 (t_low=0.27, t_high=0.5)

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  114    0    0 ]
   ben    [   16   61   37 ]   ← ben→low 32.5% (比 baseline 略差)
   nt     [   14   76  138 ]   ← nt→low 60.5% (比 baseline 略低)
```

**5 底线**: mal→low=0 ✅ │ high_R=1.0 ✅ │ nt→high=6.1% ❌ │ nt→low=60.5% ❌ │ ben→low=32.5% ❌  **(2/5)**

**优点**: 跟 baseline 非常接近,说明被改的那个 modality 对最终输出影响有限。

**缺点**: 没解决任何核心问题,3 个软底线全不过。

**临床读法**: 与 baseline 同,选 D 没有理由,baseline 更纯净。

### 实验 6 ── 0510 E aux-cls

**设计动机**: B 的单 risk 分数下,ben 和 nt 在 [0.1, 0.4] 区间高度重叠,
无法仅靠阈值分开。**显式引入三分类 (mal/ben/nt) 概率作为决策路由信号** —— 即使 risk 低,
只要 p_ben 高就强制走 medium。

**架构**: baseline + `cls_head: 256 → 128 → 3` (新增并行 3-class CE head)

**Loss**: `L = BCE_ord + 0.5 · CE_3class + seg`,non-tumor target [1.0, 0.35, 0.0] 同 B

**决策**: 类别先验路由 + 双阈值回落
```
if p_nt  > 0.7  → low
elif p_mal > 0.3 → high
elif p_ben > 0.3 → medium  (强制!不进 low)
else: 走 risk 双阈值 (t_low=0.05, t_high=0.5)
```
chosen_method = `soft_relaxed_0.10` (val 上 strict 找不到,放宽 ben→low ≤ 10%)

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  114    0    0 ]
   ben    [    2   80   32 ]   ← ben→low 28.1% (强于 B 但远不达标)
   nt     [    3  109  116 ]   ← nt→low 仅 50.9% (大量 nt 走 medium)
```

**5 底线**: mal→low=0 ✅ │ high_R=1.0 ✅ │ nt→high=1.3% ✅ │ nt→low=50.9% ❌ │ ben→low=28.1% ❌  **(3/5)**

**优点**: nt→high 仅 1.3% (路由判 p_nt>0.7 直接锁 low,nt 假高警报极少)。

**缺点**:
- val 学到的 τ_nt=0.7 在 test 上严重过拟合:test 上多数 nt 的 p_nt 没 >0.7
- 落入回落分支的样本仍按 risk 阈值分,ben/nt 又混在一起
- 路由起作用的样本不到 1/3

**临床读法**: 类别先验思想是对的,但**阈值过拟合**让方法在 test 上失效。

### 实验 7 ── 0510 F triple-head

**设计动机**: 放弃 ordinal regression,改为 **3 个完全独立的二分类 head**,
每个 head 专注一档,各有 BCE 损失。

**架构**: baseline 的 risk_head 替换为 3 个并行 head:
- head_high (mal=1 vs others=0)
- head_med (ben=1 vs others=0)
- head_low (nt=1 vs others=0)

**Loss**: `L = BCE_high + BCE_med + BCE_low + seg`,无温度校准

**决策**: argmax(p_high, p_med, p_low) + tau_safe=0.44 fallback (high 必须 > 0.44 否则降级)

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  113    1    0 ]   ← 1 个跌中,high_R=0.991
   ben    [    0   99   15 ]   ← ben→low 13.2% (尚可)
   nt     [    1  180   47 ]   ← ❌ nt 大量被吸进 medium,nt→low 仅 20.6%
```

**5 底线**: mal→low=0 ✅ │ high_R=0.991 ✅ │ nt→high=0.4% ✅ │ nt→low=20.6% ❌ │ ben→low=13.2% ❌  **(3/5)**

**优点**:
- nt→high 0.4% — 史上最低 (head_high 严格)
- ben→low 13.2% — 仅次于 H/I,接近 10% 软底线

**缺点 (致命)**:
- **medium_share = 61.4%** — argmax 把多数样本判 medium
- head_med 输出对 nt 样本也常有高响应 → 大量 nt 走 medium
- 3 个 BCE 没有 ordinal 约束,nt 样本同时被 head_med 和 head_low 竞争,head_med 略胜
- best_epoch=2 — 模型很快过拟合 val,缺乏稳定性

**临床读法**: 极度保守,几乎所有非典型病人都到 medium → 中风险池爆炸,资源压力大。

### 实验 8 ── 0510 G margin

**设计动机**: B 在 score 空间 ben/nt 重叠 ([0.085-0.79] vs [0.058-0.93]),
加 **pairwise margin ranking loss** 显式拉开三类间的 score 间距。

**架构**: 同 baseline (只改 loss)

**Loss**:
```
L = BCE_ord + 0.3 · L_rank + seg
L_rank = ReLU(s(nt) - s(ben) + 0.2) + ReLU(s(ben) - s(mal) + 0.2)
```
margin = 0.2,λ_rank = 0.3

**决策**: 单 score 双阈值 + 温度校准 T=0.414 (比 B 的 0.366 温和)

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  114    0    0 ]
   ben    [    5   64   45 ]   ← ben→low 39.5% (改善但仍灾难)
   nt     [   11   53  164 ]   ← nt→low 71.9% (没到 80%)
```

**5 底线**: mal→low=0 ✅ │ high_R=1.0 ✅ │ nt→high=4.8% ✅ │ nt→low=71.9% ❌ │ ben→low=39.5% ❌  **(3/5)**

**优点**: nt→high=4.8% 卡线达标,nt→low=71.9% 接近 80%。

**缺点 (根因)**:
- λ_rank=0.3 太小,ranking loss 几乎被 BCE 淹没
- chosen_method = `baseline_fallback` (G 的 strict search 把 t_high 升到 0.78,ben→low 反而到 36%,
  最终退回 baseline 双阈值)
- T=0.414 比 B 温和,压低分力度变弱,nt→low 反而比 B 低 (71.9 vs 89.5)

**临床读法**: G 是 B 的"温和版" — ben 比 B 略好但仍 39%,nt 比 B 差。两头不靠。

### 实验 9 ── 0510 H cascade **(ben→low 完美)**

**设计动机**: 修正 0502 C 的失败 (stage1+stage2 强相关)。改为:
- **stage1: tumor vs no_tumor** (这两类差异显著,易学,期望 AUC ≥ 0.99)
- **stage2: mal vs ben** (在 tumor 池内细分)

物理隔离 nt 和 polyp,**ben 在 stage2 不可能进 low**。

**架构**: baseline 的 risk_head 替换为:
- tumor_head (256→128→1, BCE on y_tumor=mal+ben vs y=nt)
- mal_head (256→128→1, BCE on y_mal=mal vs y=ben,**loss 用 y_tumor 加权**:no_tumor 样本对 stage2 不贡献梯度)

**Loss**: `L = BCE(p_tumor) + 1.0 · BCE_weighted(p_mal | tumor) + seg`

**决策**:
```
双温度校准: T_tumor=2.287 (大!), T_mal=1.012
if p_tumor < τ1=0.16  → low      (no_tumor 直接走 low)
elif p_mal > τ2=0.5   → high     (mal 直接走 high)
else                  → medium   (ben 强制 medium, 绝不进 low)
```

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  114    0    0 ]   ← 高敏感保住
   ben    [    9  102    3 ]   ← ✅ ben→low 仅 2.6% (B 的 1/30)
   nt     [   13  198   17 ]   ← ❌ nt→low 仅 7.5%, nt→high=5.7% 略超 (13/228)
```

**5 底线**: mal→low=0 ✅ │ high_R=1.0 ✅ │ nt→high=5.7% ⚠️临界 │ nt→low=7.5% ❌ │ ben→low=2.6% ✅  **(3/5)**

**优点**: **ben→low = 2.6%** — 这是 B 的 74.6% 的 **1/29**!临床意义巨大:
良性息肉几乎全被锁在 medium 池接受随访。

**缺点 (致命)**:
- **val tumor_head AUC 仅 ≈ 0.85** (从 T_tumor=2.287 极大温度反推:原始 logit 过自信)
- tumor_head 区分 nt vs polyp 能力不够 → 大量 nt 的 p_tumor ≥ 0.16,被推 medium
- **nt→low 仅 7.5%** — 198/228 = 87% 的 nt 跑到 medium

**临床读法 (100 病人)**:
- mal: 25 全 high ✅
- ben: 2 高 + **22 中** + **1 低** → 22 个良性息肉规范随访,1 个漏管 — **临床上接近完美**
- nt: 6 高 + 43 中 + 1 低 → 43 个正常人被推 medium → 大量随访负担

**H 的核心 trade-off**: ben→low 几乎清零,但代价是 **nt 几乎全部进 medium**。
真正的瓶颈是 stage1 tumor_head AUC 上不去。

### 实验 10 ── 0510 I B+H fusion (B + H 融合尝试)

**设计动机**: H 的 ben→low=2.6% 极成功,但 nt→low=7.5% 灾难。
**B 的非对称 ordinal target + H 的 cascade 决策同时上**,期望 ordinal 监督让 fusion 表示更紧凑,
间接提升 tumor_head AUC。

**架构**: 3 个并行 head 共享 256D fusion:
- risk_head (1D, ordinal supervision by B 的 [1.0, 0.35, 0.0])
- tumor_head (1D, BCE on y_tumor)
- mal_head (1D, BCE on y_mal | tumor)

**Loss**: `L = 2.0 · BCE_ord(risk) + BCE(tumor) + 1.0 · BCE_weighted(mal|tumor) + seg`

**决策**: **完全走 H 的 cascade** (risk_logit 只用于诊断不参与决策)
```
双温度: T_risk=0.508, T_tumor=2.290, T_mal=0.317 (❗ 过激进)
if p_tumor < τ1=0.12  → low
elif p_mal > τ2=0.3   → high
else                  → medium
```

**Test-112 混淆矩阵**:
```
                高   中    低
   mal    [  106    8    0 ]   ← ❌ 8 个 mal 跌到 medium,high_R=0.930 破硬底线!
   ben    [    0  114    0 ]   ← ✅ ben 全锁 medium,ben→low = 0%
   nt     [    2  217    9 ]   ← nt→low 仅 3.9% (cascade nt 问题没解决)
```

**5 底线**: mal→low=0 ✅ │ **high_R=0.930 ❌** │ nt→high=0.9% ✅ │ nt→low=3.9% ❌ │ ben→low=0.0% ✅  **(3/5)**

**优点**:
- **ben→low = 0%** — 史上唯一一个 0% 良性误为低!
- nt→high = 0.9% (228 nt 仅 2 个假高)

**缺点 (致命)**:
- **T_mal = 0.317 严重过校准** — mal_head 在 stage2 被极度压低,
  8 个真 mal 的 p_mal 没过 τ2=0.3,被压到 medium
- **high_recall 跌到 0.93 < 0.95 → 破硬底线**
- nt→low 3.9% (cascade nt→low 问题没解决,反而更差,因为 τ1 设到 0.12 更严)
- val mal 子集 (~40 例) 太小,温度 fit 严重过拟合

**临床读法**: ben→low=0 看似完美,但 **mal 漏诊率 7% (8/114) 不可接受** — 这相当于
8 个恶性病例被告知"中等风险",可能延误手术。**临床上不可上线**。

**根因**: I 试图融合 B 的 ordinal + H 的 cascade,但 cascade 决策器对 mal_head 极度敏感,
val 上 T_mal LBFGS fit 不稳定 → 部署时 T=0.317 过激进。

## 五、决策器架构 × 临床指标 — 总规律

把 10 个实验按决策器结构分两类,trade-off 一目了然:

| 决策器类型 | 实验 | nt→low (优先级 ③) | ben→low (优先级 ②) |
|---|---|---:|---:|
| **单 score + 双阈值** | baseline, A, B, D, G | **59% – 89%** ✅ 都接近过线 | **27% – 75%** ❌ 都灾难 |
| **多 head / cascade / argmax** | C, E, F, H, I | **4% – 51%** ❌ 都不过 | **0% – 28%** ✅ 都改善 |

### 5.1 物理解释

**单 score 双阈值**(B 是极致):
- 单一 risk 分数 + 强温度校准 → nt 全压向 0 → 落入 low 阈值之下
- 但 ben target 0.35 也跟着压低 → ben 也滑进 low
- 优先级 ③ ✅ / 优先级 ② ❌

**Cascade / 多 head**(H 是极致):
- 决策路径上专门隔离 ben (强制走 medium)
- 但 tumor vs no_tumor 边界在 ultrasound 上**不是线性可分** → tumor_head AUC 上不去 (0.85)
- 大量 nt 被判 polyp → 走 medium
- 优先级 ② ✅ / 优先级 ③ ❌

### 5.2 根本瓶颈

> **stage1 tumor_head val AUC 仅 ≈ 0.85** (期望 ≥ 0.99)
>
> 这是 cascade 决策器一切问题的源头。不解决,nt→low 永远上不去。
