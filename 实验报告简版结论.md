# Task2 实验简版结论

## 1. 任务与数据

- 任务：`Task2 = benign(良性肿瘤) vs no_tumor(非肿瘤性息肉)` 二分类
- 原始训练集：`1303` 张
  - `benign=309`
  - `no_tumor=994`
- 原始测试集：`564` 张
  - `benign=129`
  - `no_tumor=435`
- 数据类别明显不平衡，约为 `1:3`

说明：

- 本页默认讨论的是“原始分布测试集”上的结果
- `1:1` 平衡 train/test 的实验只作为补充分析，不作为主排名依据
- 当前流程仍是 `train/test` 两份数据，测试集被用于选最佳 epoch，因此所有结果更适合横向比较，不应视为严格最终泛化性能

## 2. 最核心结论

- 当前原始分布测试集上的最优模型已经更新为：
  - `ConvNeXtTiny_adamw_warmupcosine_classweight_2`
  - `Macro-F1 = 0.6325`
  - `Acc = 0.7606`
- 少数类 `benign` 抓取能力最强的是：
  - `SwinTiny_adamw_warmupcosine_classweight_2`
  - `benign recall = 0.4496`
  - `benign f1 = 0.4234`
- 经典 CNN 里最稳的参考线仍然是：
  - `Resnet34_weakaug_classweight_3`
  - `Macro-F1 = 0.6225`

一句话总结：

- 首轮统一 baseline 说明了任务难点
- 第二轮模型专用 recipe 说明了真正的上限
- 当前最重要的结论不是“哪种模型名字更强”，而是“模型专用 recipe 会明显改变结论”

## 4. 第二轮 recipe 的实际价值

第二轮不是“小修小补”，而是实质性提升：

| 模型 | 首轮 baseline F1 | 第二轮 recipe F1 | 提升 |
| --- | ---: | ---: | ---: |
| `ConvNeXtTiny` | 0.4945 | 0.6325 | `+0.1380` |
| `SwinTiny` | 0.4354 | 0.6192 | `+0.1838` |
| `ViTB16` | 0.5455 | 0.6151 | `+0.0696` |
| `EfficientNetB0` | 0.5994 | 0.6115 | `+0.0121` |

这说明：

- `ConvNeXt / Swin / ViT` 首轮表现差，不代表模型本身不行
- 问题更主要出在统一 baseline recipe 不适配这些模型
- 第二轮有效 recipe 主要包括：
  - `AdamW`
  - backbone / head 差分学习率
  - `warmup + cosine`
  - `class weight`
  - `label smoothing`
  - 贴近模型特点的弱增强

## 5. 现阶段可以明确回答的几个问题

### 5.1 为什么早期 baseline 容易失败

- 数据不平衡明显，模型容易倒向多数类 `no_tumor`
- 统一使用 `Adam + lr=1e-3 + 100 epoch`，对不同模型并不公平
- 没有针对少数类做足够处理时，`macro-F1` 很难上去
- 部分模型其实不是“过拟合”，而是“根本没按合适 recipe 训起来”

### 5.2 为什么第二轮能提升

- `class weight` 直接改善了 `benign` 学习
- `warmup + cosine` 抑制了训练前期不稳定和后期失控
- 差分学习率让预训练 backbone 不至于被过快破坏
- 弱增强提高泛化，但不像强增强那样破坏超声图像结构

### 5.3 为什么不能用 `1:1` test 来下主结论

- 平衡测试集会让指标看起来更“均衡”
- 但这不代表模型在真实分布上更强
- 这次实验已经验证：`1:1` 训练出来的权重回到原始 test 后，整体结果反而变差

## 6. 当前最推荐的推进路线

### 主线

- `ConvNeXtTiny_adamw_warmupcosine_classweight_2`

建议继续尝试：

- `img320`
- `focal loss`
- `weighted sampler`
- `TTA`

### 辅助主线

- `SwinTiny_adamw_warmupcosine_classweight_2`

原因：

- 目前 `benign recall / benign f1` 最强
- 很适合继续冲少数类指标

### 稳定参考线

- `Resnet34_weakaug_classweight_3`

原因：

- 结果稳定
- 便于判断后续新实验到底是真提升，还是 recipe 波动

## 7. 必须保留的方法学提醒

- 当前所有结果都建立在 `test` 反复参与选最佳 epoch 的流程上
- 因此：
  - 当前结果适合做阶段性横向比较
  - 后续正式定稿前，建议补 `val` 或至少在 `train` 内部做验证划分
- 不建议再把 `test` 改成 `1:1` 作为主评估集

## 8. 最适合汇报时说的结论

- 当前最优模型是 `ConvNeXtTiny_adamw_warmupcosine_classweight_2`，`Macro-F1 = 0.6325`
- 当前少数类识别最强模型是 `SwinTiny_adamw_warmupcosine_classweight_2`
- 第二轮结果证明：模型专用 recipe 比单纯换 backbone 更重要
- 后续最值得投入的方向不是继续盲目换模型，而是围绕 `ConvNeXt / Swin / ResNet34` 三条线做分辨率、loss 和采样策略优化
