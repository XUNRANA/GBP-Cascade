# 0502 风险分层模型实验总结

## 1. 实验目标
将原三分类任务（malignant / benign / no_tumor）调整为**风险分层任务**（高/中/低风险），核心目标是：
- 保证恶性病例不被分到低风险（安全底线）；
- 在满足安全底线前提下，提高低风险（随访）人群的纯度；
- 对 `0502dataset111`（1:1:1）与 `0502dataset112`（1:1:2）做稳定性验证。

## 2. 实验设置（0502）
- 主脚本：`0502/scripts/20260502_task_risk_SwinV2Tiny_ordinal_trimodal_1.py`
- 工具库：`0502/scripts/risk_utils.py`
- 主训练/主评估集：`0502dataset111`
- 稳定性验证集：`0502dataset112`
- 风险分数：`risk_score ∈ [0,1]`（1D sigmoid head）
- 序目标：`malignant=1.0, benign=0.5, no_tumor=0.0`
- 阈值策略：双阈值 `t_low, t_high`（`score>=t_high` 判高风险，`score<=t_low` 判低风险）

## 3. 评估方法（本实验怎么评）
### 3.1 阈值选择方法（在 Val 上）
先以 `Val ROC-AUC(malignant vs others)` 选最优权重，再在 val 上做**带约束双阈值搜索**：
- `malignant -> low` 数量 `<= 1`
- 高风险召回 `>= 0.95`
- 中风险占比 `<= 0.35`
- 在满足约束时，优先最大化低风险精确率（low_precision）。

本次最终阈值来自 `thresholds.json`：
- `best_epoch = 6`
- `best_val_auc = 1.0000`
- `t_low = 0.29`, `t_high = 0.50`
- `profile_used = primary`

### 3.2 测试评估口径
对 Test-111 / Test-112 统一使用 val 选出的阈值，输出：
1. 三档混淆矩阵（真实标签 × 预测风险档）  
2. 安全口径（二值：高风险 vs 非高风险）  
3. 关键安全指标：`mal_to_low`、`high_recall`、`high_precision`、`binary_roc_auc`  
4. 每档统计（high/medium/low 的 precision/recall/share）  
5. 校准指标（Brier score）

## 4. 实验结果（现在这版模型表现）
### 4.1 总体结果
- 训练已完成，产物完整（best 权重、阈值、eval_111、eval_112、混淆矩阵图）。
- 两个测试集都满足安全底线：`mal_to_low = 0`，`high_recall = 1.0`。
- 恶性识别能力极强（二值 ROC-AUC 接近 1）。
- 主要瓶颈仍在 benign 与 no_tumor 的边界区分（中/低风险互串明显）。

### 4.4 低风险类别的详细解析（no_tumor）
1. 预测为低风险的人群中是否混入中高风险：  
   - Test-111 预测低风险共 108 例：`malignant=0, benign=35, no_tumor=73`  
   - Test-112 预测低风险共 179 例：`malignant=0, benign=35, no_tumor=144`  
   - 结论：预测低风险中**有中风险（benign）混入**，但**没有高风险（malignant）混入**。
2. 真实低风险（no_tumor）被成功预测为低风险的比例：  
   - Test-111：`73/114 = 64.04%`  
   - Test-112：`144/228 = 63.16%`  
   - 合并：`217/342 = 63.45%`

### 4.5 三种类别识别成功率（成功预测 / 实际病例数）
> 口径：malignant→高风险、benign→中风险、no_tumor→低风险

## 5. 结论
这版风险分层模型已经达到“**先保安全**”目标：恶性不漏到低风险，高风险召回稳定达标；但“**中低风险精细分层**”仍是当前主矛盾，特别是 benign 与 no_tumor 的可分性。

## 6. 下一步改进（3条）
### 路线1（最高优先级）：强化 benign vs no_tumor 边界学习
- **改什么**：加入边界难例挖掘（hard mining）与 benign/no_tumor 对比约束（例如 pairwise ranking 或 margin 对比损失）。
- **预期收益**：降低中低风险互串，提升 medium/low 档 precision 与 recall。
- **验收指标**：在不降低 `high_recall` 与 `mal_to_low` 的前提下，`medium precision` 与 `low precision` 同时提升。

### 路线2：做不确定性感知阈值与校准
- **改什么**：在风险分数后增加温度校准/分段校准，并对阈值附近样本输出“不确定”标记用于临床复核。
- **预期收益**：减小数据分布变化时阈值漂移，提升跨数据集稳定性。
- **验收指标**：Brier score 下降，111/112 两集关键指标波动幅度缩小。

### 路线3：双阶段决策（安全筛查 + 细分）
- **改什么**：阶段一专注恶性高召回筛查；阶段二仅在非高风险人群内细分 benign/no_tumor。
- **预期收益**：保持安全底线同时提升中低风险细分能力。
- **验收指标**：`mal_to_low=0`、`high_recall>=0.95` 保持不变，且二阶段下 medium/low 档混淆显著下降。
