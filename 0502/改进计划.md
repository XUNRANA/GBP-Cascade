# 0502 风险分层模型 —— 改进计划（4 个并行实验）

> 基于 baseline `20260502_task_risk_SwinV2Tiny_ordinal_trimodal_1` 的诊断分析,设计 4 个**互不依赖、可并行启动**的改进实验。

---

## 一、Baseline 诊断（哪里最痛）

### 关键指标快照

| 指标 | Test-111 | Test-112 |
|---|---:|---:|
| mal_to_low (硬底线 ≤1) | **0** ✅ | **0** ✅ |
| high_recall (硬底线 ≥0.95) | **1.0000** ✅ | **1.0000** ✅ |
| binary ROC-AUC | 0.9995 | 0.9994 |
| **high_precision** | 0.8028 | **0.7261** ⚠️ |
| **low_precision** | 0.6759 | 0.8045 |
| medium_share | 0.269 | 0.263 |
| Brier score | 0.0871 | 0.0933 |
| **benign 识别率（→中风险）** | **53.51%** ⚠️ | 53.51% ⚠️ |
| **no_tumor 识别率（→低风险）** | **64.04%** ⚠️ | 63.16% ⚠️ |

**安全底线无懈可击,但中低风险细分 ≈ 抛硬币级别** —— benign 一半识别不出来。

### 三档混淆矩阵
```
Test-111            high   med   low
malignant     [114    0     0]   ← 完美
benign        [ 18   61    35]   ← 18 个被错划到高(过度手术)
no_tumor      [ 10   31    73]   ← 10 个被错划到高
```

### 根因分析（4 条）

**根因 1 ── benign 评分内部分布严重发散**
- benign 的 score 范围 `[0.042, 0.970]`,std=0.168
- 也就是说 benign 群体里**有近 16% 的样本评分 > 0.5 直接被推成高风险**
- 18 例 benign→high 全部来自这一段尾巴

**根因 2 ── 校准曲线两极化（bimodal）**
- 校准 bin 计数:`[31, 37, 47, 52, 33, 10, 10, 5, 2, 115]`
- 两端样本最多(0.05 段 31 个,0.95 段 115 个),中间空荡荡
- bin_mean_true 在 0~0.85 全是 0,只有 0.85+ 才出现真值 0.5/1
- → 模型对"中间地带"没有清晰的概率语义,benign↔no_tumor 在分数上没拉开

**根因 3 ── 过拟合严重(epoch 6 后训练全是负贡献)**
- best 出现在 **epoch 6**(Val AUC=1.0)
- 之后 50+ 个 epoch:val AUC 在 0.94~0.99 抖动,后期长期 fall back 到 `unconstrained` profile(M→低=3 漏 3 例恶性)
- 60 epoch 设置严重过长,白训了 90% 时间

**根因 4 ── Test-112 上 high_precision 大幅下降**
- 111 → 112:high_precision 0.80 → 0.73(掉 7pt)
- benign 在 112 上数量没变,但 no_tumor 翻倍后的"硬负样本"更多 → 假阳更多
- 说明阈值对样本分布敏感,模型决策边界**不够稳健**

## 二、改进设计：4 个并行实验

> 每个实验**独立目录、独立日志、互不依赖**,可同时挂 4 张卡跑。
> 结束后用统一评估口径在 Test-111 / Test-112 上对比 baseline。

### 实验 A:边界对比学习(SupCon + Hard Mining)

**目标**:在 feature 空间把 benign vs no_tumor 拉开,直接攻克 benign 53% 识别率瓶颈。

**核心改动**:
- 在 `fused` (256D) 之后接一个 **projection head** (256→128)
- 加 **SupCon loss**(监督对比),pull 同类 / push 异类,**显式强化 benign↔no_tumor 边界**
- 加 **benign-no_tumor pair-wise hard mining**:每 batch 强制采 3 对 score 接近的 b-n 难例

**损失函数**:
```
L = lambda_ord * BCE(risk_score, ord_target)        # 保留
  + lambda_seg * (Dice + CE)                          # 保留
  + lambda_supcon * SupCon(proj_feat, labels)        # 新增, lambda=0.5
```

**评估关注**:
- benign 识别率(关键指标!) →期望 53% → 65%+
- no_tumor 识别率 → 期望 64% → 75%+
- TSNE 可视化 cls_feat:期望 benign 簇/no_tumor 簇的 silhouette score 提升

**脚本**:`20260505_task_risk_A_supcon.py` + `risk_utils_A.py`(新增 SupCon)

---

### 实验 B:非对称 Target + 早停 + 温度校准

**目标**:从训练目标和正则化两端同时下手,缓解过拟合 + 修校准。

**核心改动**:
1. **ordinal target 从 `(1.0, 0.5, 0.0)` 改为 `(1.0, 0.35, 0.0)`**
   - 把 benign 拉低,使其在概率空间更接近 no_tumor
   - 强迫模型学 benign 内部细粒度差异(否则 BCE 被压平)
   - 同时尝试 `(1.0, 0.6, 0.0)` 作为对比组(写在 config 里一键切)
2. **训练量收紧**:`num_epochs=25, patience=5`
   - baseline best @ epoch 6,根本不需要 60 epoch
   - patience=5 :val AUC 5 个 eval 不升就停
3. **训练后温度校准(Temperature Scaling)**:
   - 在 val 上用 LBFGS 拟合 1 个温度参数 T
   - 推理时 `risk_score = sigmoid(logit / T)`
   - 期望把 score 分布从 bimodal 拉平,提升 ordinal 中间段的概率语义

**评估关注**:
- Brier score:期望 0.087 → 0.05~0.06
- 校准曲线 bin_mean_true 在 0.3~0.7 段出现非零(不再两极)
- high_precision 提升(0.80→0.85+)

**脚本**:`20260505_task_risk_B_calib.py`(基于 baseline 改 config + 加 temperature.py)

---

### 实验 C:双阶段决策(Two-Stage Cascade)

**目标**:把"安全筛查"和"benign vs no_tumor 细分"完全解耦,各自最大化。

**核心改动**:
- **阶段 1(完全复用 baseline)**:用现有 risk_score 做高风险二分类
  - `score >= t_high` → 高风险(直接输出)
  - `score < t_high` → 进入阶段 2
- **阶段 2(新训练)**:在非恶性样本(benign + no_tumor)上单独训练一个**二分类 head**
  - 输入:trimodal fused feat (256D)
  - 输出:`p_benign ∈ [0,1]`
  - Loss:BCE(p_benign, is_benign)
  - 只用 benign + no_tumor 样本训练,**不让恶性参与梯度**
- **推理**:阶段 1 过滤完高风险后,阶段 2 用 `t_med` 切分
  - `p_benign >= t_med` → 中风险
  - `p_benign < t_med` → 低风险

**为什么有效**:
- baseline 一个 ordinal head 同时背"识别恶性"和"分 benign/no_tumor",前者太强势把后者梯度淹没
- 阶段 2 只看 b/n 二分类,模型容量完全用于这个边界

**评估关注**:
- benign 识别率 → 期望 53% → 70%+
- no_tumor 识别率 → 期望 64% → 78%+
- 高风险召回保持 1.0(完全继承阶段 1)
- 同步报告 stage-2 head 的 ROC-AUC(b vs n)作为内部诊断

**脚本**:`20260505_task_risk_C_twostage.py` + 阶段 2 单独训练函数 `train_stage2_head()`

---

### 实验 D:模态重加权 + 文本/临床增强(Modal Rebalance)

**目标**:benign vs no_tumor **影像本身就难分**,必须让 BERT 文本和 10D 临床特征发挥更大作用。

**baseline 的隐患**:
- gated trimodal fusion 是数据驱动的门控,但训练初期图像分支梯度强,门控会"被一步教坏",学不到平衡的多模态利用
- 文本和临床特征对 b vs n 的鉴别力没充分发掘

**核心改动**:
1. **Modality Dropout**:训练时每个 batch **随机 drop 一个模态**(p=0.15)
   - 强迫 fusion head 不依赖任何单一模态
   - drop 时该模态向量置零(已对应 mask 位置在 attention 里关掉)
2. **门控权重正则**:在 gated fusion 输出上加 **entropy regularization**(鼓励 3 模态权重均衡)
   - 损失:`L_gate = -lambda_gate * H(gate_weights)`,lambda=0.05
3. **文本数据增强**:训练时对 BERT input_ids 做 **15% token masking**(类似 BERT 预训练 MLM 风格)
   - 让模型不依赖某个固定关键词,学整体语义
4. **临床特征 noise injection**:训练时给 10D 临床特征加 N(0, 0.05) 高斯噪声
   - 提升临床头泛化,缓解 112 上的分布漂移
5. **诊断输出**:每个 epoch 记录三模态平均 gate 权重(image / text / clin),写日志方便观察

**评估关注**:
- Test-111 → Test-112 的 high_precision 下降幅度:期望从 -7pt 缩到 -2pt 内(更稳健)
- 三模态门控权重日志:期望最终 image≈0.5, text≈0.3, clin≈0.2 而不是 image≈0.85
- benign 内 score std 缩小(从 0.168 → 0.12 左右,说明 benign 更聚拢)

**脚本**:`20260505_task_risk_D_modal.py` + 模型 forward 改写支持 modality dropout

## 五、推荐启动顺序(若只能串行)

按"投入产出比"排序:

1. **实验 C(双阶段)** —— 改动局部,风险最低,**预期收益最大**(benign 直接 +15pt)
2. **实验 B(校准+早停)** —— 改动最少(只动 config),立即收益(过拟合直接修掉)
3. **实验 A(SupCon)** —— 中等改动,理论收益高,但需要调 lambda
4. **实验 D(模态平衡)** —— 改动最大,期望收益是**稳定性而非精度**,放最后

但用户已说"4 个并行" → 直接 4 卡同时启动,A/B/C/D 互不阻塞。
