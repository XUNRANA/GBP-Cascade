> 核心原则：**所有 Transformer 实验必须利用 JSON 病灶标注信息**，ROI-only 仅作 CNN 对照。

## 一、前期实验回顾（0319 仅图片，无标注）

| 模型 | Acc | Macro-F1 | Benign Recall | Benign F1 |
|------|-----|----------|---------------|-----------|
| ConvNeXt-Tiny img320 | 0.7660 | **0.6372** | 0.3721 | 0.4211 |

| ResNet34 weakaug+cw | 0.7358 | 0.6225 | 0.4109 | 0.4157 |

**瓶颈**：Benign recall ≤ 0.45，Macro-F1 卡在 0.64。

## 二、标注信息关键发现

| 统计项 | benign | no_tumor | 差异 |
|--------|--------|----------|------|
| 病灶面积占比 | **2.09%** ± 1.75% | **1.39%** ± 1.10% | benign大50% |
| 有病灶标注 | 100% | 99.9% | 几乎全覆盖 |
| 胆囊ROI占比 | 73.8% | 75.7% | 基本相同 |

**核心信号**：benign 病灶更大、形态不同，但面积极小（1-2%），纯图片模型难以捕捉 → 必须显式引入标注。

## 三、标注利用的 5 种方式

```
方式 A：ROI 裁剪
  gallbladder 矩形框裁剪 → 去除 25% 背景（仅用矩形框，最浅层利用）

方式 B：4 通道输入（RGB + Lesion Mask）
  所有非 gallbladder 的多边形 → 二值 mask → 作为第 4 通道
  模型显式看到"病灶在哪里"

方式 C：双分支（全局 ROI + 病灶局部裁剪）
  分支A 看全局上下文，分支B 聚焦病灶细节
  融合时加入病灶面积比等结构化特征

方式 D：Mask-Guided Attention
  病灶 mask 生成 attention bias → 引导 Transformer 自注意力聚焦病灶区域
  不改变输入通道，而是在注意力层注入空间先验

方式 E：多任务（分类 + 辅助分割）
  编码器同时学分类和病灶分割 → 迫使特征保留病灶边界信息
```

#### 实验 6/9：双分支融合（方式 C）
```
Branch A (全局): ROI裁剪 → Swin-V2/MaxViT backbone → global_feat (768-d)
Branch B (局部): 病灶 bbox(padding 30%) 裁剪 → 共享 backbone → local_feat (768-d)

结构化特征:
  - lesion_area_ratio = 病灶面积 / 胆囊面积
  - lesion_count = 病灶数量
  - lesion_type_onehot = [polyp, pred, adenoma, tubular_adenoma]  # 4维

融合:
  [global_feat, local_feat, struct_feat] → MLP → 2类
  concat (768 + 768 + 7) → Linear(1543, 256) → GELU → Dropout → Linear(256, 2)
```

#### 实验 7：Mask-Guided Attention（方式 D）
```
不改变输入(仍为3通道RGB ROI裁剪)，在 Transformer 内部注入空间先验：

1. 病灶 mask resize 到 feature map 尺寸 (e.g., 8×8 for patch=16, img=256)
2. mask_bias = mask * λ (λ=可学习标量，初始化为1.0)
3. 在 self-attention 中: attn = softmax(QK^T/√d + mask_bias)

效果: 病灶区域的 patch token 获得更高注意力权重
优势: 不改变输入维度，不需要修改第一层conv，预训练权重完全复用
```

#### 实验 8：多任务学习（方式 E）
```
Swin-V2 encoder →
  Head A: GAP → Linear → 2类分类 (主任务, loss weight=1.0)
  Head B: FPN decoder → 1ch sigmoid → 病灶分割 (辅助, loss weight=0.3)

Loss = CrossEntropy_cls + 0.3 × (BCE_seg + Dice_seg)

分割GT: 病灶多边形 → 二值mask
目的: 分割梯度迫使 encoder 学习病灶边界/形态特征 → 反哺分类任务
推理时只用 Head A，无额外开销
```
