# 0322 Task2 实验详解

## 1. 总体架构与代码组织

### 1.1 代码框架

所有实验共享一个工具库 `task2_json_utils.py`，每个实验只需编写一个独立 `.py` 文件，通过 4 个回调函数接入通用训练框架：

```
task2_json_utils.py          ← 公共工具库
├── JSON 标注解析              (load_annotation, get_gallbladder_rect, generate_lesion_mask, crop_roi)
├── 数据集类                   (GBPDatasetROI, GBPDatasetROI4ch, GBPDatasetFull4ch)
├── Transform                 (SyncTransform, StrongSyncTransform, build_roi_*_transform)
├── 模型工具                   (adapt_model_to_4ch, split_backbone_and_head)
├── 训练工具                   (train_one_epoch, evaluate, FocalLoss, mixup_cutmix_data, ...)
├── 采样与阈值                 (build_weighted_sampler, find_optimal_threshold, evaluate_with_threshold)
└── 实验运行器                 (run_experiment)

20260323_task2_XXX_N.py       ← 每个实验脚本
├── Config 类                  (超参数、路径)
├── build_model()              (构建模型)
├── build_dataloaders(cfg)     (数据集 + DataLoader)
├── build_optimizer(model,cfg) (优化器)
└── main() → run_experiment(cfg, build_model_fn, build_dataloaders_fn, build_optimizer_fn, ...)
```

`run_experiment()` 是通用训练循环，负责：日志记录 → 加载数据 → 构建模型 → 类别权重计算 → 训练循环 (warmup + cosine LR) → 每 N epoch 评估 → 保存最优模型 → 最终测试 + 阈值优化。

### 1.2 运行方式

所有实验在 `conda activate gbp` 环境下，使用 `CUDA_VISIBLE_DEVICES=1` 单卡运行：

```bash
# 批次1: Exp #1-6 (初始实验)
source activate gbp && CUDA_VISIBLE_DEVICES=1 nohup bash run_0322_batch1_gpu1.sh > run_0322_batch1.log 2>&1 &

# 批次2: Exp #7-8 (强增强实验)
source activate gbp && CUDA_VISIBLE_DEVICES=1 nohup bash run_0322_batch2.sh > run_0322_batch2.log 2>&1 &

# 批次3: Exp #7(重跑),9,10,11 (修复+改进实验)
source activate gbp && CUDA_VISIBLE_DEVICES=1 nohup bash run_0322_batch3_improved.sh > run_0322_batch3.log 2>&1 &
```

每个实验通过 lock 文件机制防止重复运行，训练完成后自动将脚本复制到日志目录归档。

## 2. 所有实验共同点

以下是 11 个实验**全部共享**的基础设置：

| 项目 | 设置 |
|------|------|

| **种子** | seed=42 (reproducibility) |
| **RGB 归一化** | ImageNet 标准: mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225] |

## 3. 实验分三阶段

### 阶段一 (Exp #1-6): 初始探索

目标：对比不同模型架构 + 不同 JSON 标注利用方式。
共同设置：60 epochs, warmup=5, min_lr_ratio=0.05, eval_interval=3, 普通 shuffle DataLoader。

### 阶段二 (Exp #7-8): 强增强

目标：解决阶段一发现的严重过拟合问题 (train acc>93%, test F1~0.60)。
改进：100 epochs, warmup=8, min_lr_ratio=0.01, StrongSyncTransform, Mixup/CutMix, dropout=0.3, 全图替代 ROI 裁剪。

### 阶段三 (Exp #9-11): 修复 + 高级策略

目标：修复阶段二发现的 Mixup 类权重 BUG + WeightedRandomSampler + Focal Loss + 阈值优化。

### Exp #1: ConvNeXt-Tiny + ROI 裁剪 (CNN Baseline)

**定位**：CNN baseline，量化"JSON 标注的 ROI 裁剪"相比"0319 全图 resize"的效果。

**模型结构**：
```
ConvNeXt-Tiny (torchvision 预训练)
├── features: ConvNeXt backbone (4 stages, [96,192,384,768] channels)
│   ├── Stage 0: 96ch, 3×ConvNeXt Block (7×7 depthwise conv + LayerNorm + 1×1 MLP)
│   ├── Stage 1: 192ch, 3×ConvNeXt Block
│   ├── Stage 2: 384ch, 9×ConvNeXt Block
│   └── Stage 3: 768ch, 3×ConvNeXt Block
├── avgpool: AdaptiveAvgPool2d → [B, 768]
└── classifier[2]: Linear(768, 2)         ← 替换最后一层
```
- 参数量: ~28.6M
- 输入: **3 通道** (ROI 裁剪后 RGB), img_size=**320**

**数据处理流程**：
```
原始图像 → 读取 JSON gallbladder 矩形框 → crop_roi(padding=2%) → Resize(320)
→ [训练] HFlip(p=0.5) + VFlip(p=0.2) + Affine(±10°, translate±4%, scale 0.9-1.1) + ColorJitter(±15%)
→ [测试] 仅 Resize(320)
→ ToTensor → Normalize(ImageNet) → [3, 320, 320]
```

**超参数**：backbone_lr=5e-5, head_lr=5e-4, batch=8, epochs=60

**结果**：F1=0.5890, benign recall=27.1%

### Exp #2: ResNet34 + ROI 裁剪 (CNN Reference)

**定位**：轻量 CNN 参考线，用于对比更大模型是否有优势。

**模型结构**：
```
ResNet34 (torchvision 预训练)
├── conv1: Conv2d(3,64,7,stride=2) + BN + ReLU + MaxPool
├── layer1: 3×BasicBlock (64ch, 3×3 conv ×2)
├── layer2: 4×BasicBlock (128ch)
├── layer3: 6×BasicBlock (256ch)
├── layer4: 3×BasicBlock (512ch)
├── avgpool: AdaptiveAvgPool2d → [B, 512]
└── fc: Linear(512, 2)                    ← 替换最后一层
```
- 参数量: ~21.3M
- 输入: **3 通道** (ROI 裁剪后 RGB), img_size=**320**

**数据处理**：与 Exp #1 完全相同 (ROI crop + 弱增强)。

**超参数**：backbone_lr=5e-5, head_lr=5e-4, batch=**16** (ResNet34 更轻可用更大 batch), epochs=60

**结果**：F1=0.6028, benign recall=39.5%

**对比 Exp #1**：ResNet34 反而比 ConvNeXt 高 1.4 个点 F1！说明小数据上轻量模型不易过拟合。

### Exp #3: SwinV2-Tiny + ROI 4ch (核心实验)

**定位**：首次引入 **4 通道输入** (RGB + 病灶 mask)，验证 JSON 病灶标注对分类的帮助。

**模型结构**：
```
Swin-V2-Tiny (timm 预训练: swinv2_tiny_window8_256)
├── patch_embed: Conv2d(4,96,4,stride=4)  ← adapt_model_to_4ch() 扩展为 4 通道
│   ├── 前 3 通道: 加载 ImageNet 预训练权重
│   └── 第 4 通道 (mask): Kaiming 正态初始化
├── layers: 4 个 SwinTransformerV2 Stage
│   ├── Stage 0: 96ch, depth=2, heads=3, window=8×8, shifted window attention
│   ├── Stage 1: 192ch, depth=2, heads=6 (PatchMerging downsample)
│   ├── Stage 2: 384ch, depth=6, heads=12
│   └── Stage 3: 768ch, depth=2, heads=24
├── norm: LayerNorm
├── head:
│   ├── global_pool: AdaptiveAvgPool
│   └── fc: Linear(768, 2)
```
- 参数量: ~27.6M
- 输入: **4 通道** [RGB + lesion_mask], img_size=**256** (Swin 窗口大小 8 需要 256)

**4 通道适配 (`adapt_model_to_4ch`)**：
```python
# 找到模型第一个 Conv2d(in_channels=3) 层
# 创建新 Conv2d(4, out, k, s, p)
# weight[:, :3] = 旧预训练权重
# weight[:, 3:]  = Kaiming 正态初始化 (mask 通道从零学起)
```

**数据处理流程 (4ch SyncTransform)**：
```
原始图像 → 读取 JSON → 生成 lesion_mask (所有非 gallbladder 多边形 → 二值 mask)
                      → crop_roi(padding=2%) → 对 RGB 和 mask 施加相同几何变换
→ [训练] Resize(256) + HFlip(p=0.5) + VFlip(p=0.2) + Affine(±10°,±4%,0.9-1.1x) | ColorJitter(±15%, 仅RGB)
→ [测试] 仅 Resize(256)
→ RGB: ToTensor → Normalize(ImageNet) → [3,256,256]
→ Mask: ToTensor (0/1 二值) → [1,256,256]
→ cat([RGB, mask]) → [4, 256, 256]
```

**关键技术**：SyncTransform 确保 RGB 和 mask 做完全一致的几何变换 (翻转、旋转、仿射)，颜色扰动仅作用于 RGB。

**超参数**：backbone_lr=**2e-5** (Transformer 用更小 lr), head_lr=2e-4, batch=8, epochs=60

**结果**：F1=**0.6172**, benign recall=40.3% — 所有单模型中最佳之一

### Exp #4: MaxViT-Tiny + ROI 4ch

**定位**：CNN-Transformer 混合架构，测试 Block+Grid 双重注意力机制。

**模型结构**：
```
MaxViT-Tiny (timm: maxvit_tiny_tf_224.in1k)
├── stem: Conv2d(4,...) + ...           ← 4ch 适配
├── stages: 4 stages, 每个 stage 包含:
│   ├── MBConv Block (移动端卷积: depthwise + SE + pointwise)
│   ├── Block Attention (局部窗口注意力)
│   └── Grid Attention (全局稀疏注意力)
├── head:
│   ├── global_pool
│   └── fc: Linear(512, 2)
```
- 参数量: ~30.9M
- 输入: **4 通道**, img_size=**224** (MaxViT 默认)

**与 Exp #3 的不同**：
| | Exp #3 (SwinV2) | Exp #4 (MaxViT) |
|---|---|---|
| 架构 | 纯 Transformer | CNN + Transformer 混合 |
| 注意力 | Shifted window (局部) | Block + Grid (局部+全局) |
| img_size | 256 | 224 |
| backbone_lr | 2e-5 | **5e-5** (稍大) |

**结果**：F1=0.5826, benign recall=23.3% — 不如 SwinV2

### Exp #5: DeiT-III-Small + ROI 4ch

**定位**：数据高效 ViT，专为中小数据集优化的 Transformer。

**模型结构**：
```
DeiT-III-Small (timm: deit3_small_patch16_224.fb_in1k)
├── patch_embed: Conv2d(4, 384, 16, stride=16)  ← 4ch 适配, 384 维 embedding
│   将 224×224 → 14×14=196 个 patch token
├── cls_token: [1, 1, 384]                      ← 可学习分类 token
├── pos_embed: [1, 197, 384]                    ← 位置编码
├── blocks: 12×TransformerBlock
│   每个 Block: LayerScale + MHSA(6 heads) + FFN(384→1536→384)
├── norm: LayerNorm
└── head: Linear(384, 2)
```
- 参数量: ~21.9M
- 输入: **4 通道**, img_size=**224**

**与 Exp #3 的不同**：
| | Exp #3 (SwinV2) | Exp #5 (DeiT-III) |
|---|---|---|
| 架构类型 | 层次化 (4 stage, 分辨率递减) | 平坦化 (12 层, 分辨率不变) |
| 注意力范围 | 窗口内局部 (8×8) | 全局 (所有 196 个 patch) |
| 特征维度 | 96→192→384→768 | 固定 384 |
| 数据效率技术 | 无 | LayerScale + 蒸馏训练 |
| backbone_lr | 2e-5 | 2e-5 |

**结果**：F1=0.5964, benign recall=21.7%

### Exp #6: SwinV2-Tiny 双分支 (全局+局部+结构化特征)

**定位**：最有创新性的设计 — 同时利用全局 ROI、局部病灶裁剪、手工特征三种信息。

**模型结构 (`DualBranchSwin`)**：
```
输入: (global_img[3,256,256], local_img[3,256,256], struct_feat[2])

Branch A (全局):  global_img → Swin-V2 backbone(shared, num_classes=0) → [B, 768]
Branch B (局部):  local_img  → Swin-V2 backbone(shared, 同一个)      → [B, 768]

Fusion:
  concat([768, 768, 2]) → [B, 1538]
  → Linear(1538, 512) → GELU → Dropout(0.3)
  → Linear(512, 128)  → GELU → Dropout(0.2)
  → Linear(128, 2)    → 输出
```

**双分支数据处理**：
```
原始图像 + JSON →
  Branch A (全局): gallbladder ROI 矩形框裁剪 (padding=2%) → Resize(256) + 弱增强
  Branch B (局部): 病灶多边形 bbox 裁剪 (padding=30%) → Resize(256) + 弱增强
  struct_feat: [lesion_area / gallbladder_area, lesion_count / 5.0]
```

**这个实验与其他所有实验的根本不同**：
1. **3 通道输入** (不用 mask 第 4 通道，而是用双分支)
2. **共享 backbone** — 同一个 SwinV2 处理全局和局部图像，减少参数
3. **手工结构化特征** — 病灶面积比、病灶数量直接作为数值输入
4. **自定义训练循环** — 不用通用 `run_experiment()`，因为输入是 tuple 格式
5. batch_size=**4** (两个图像同时前传，显存翻倍)

**结果**：F1=0.5960, benign recall=23.3% — 过拟合严重 (train acc 98.8%)，小数据不适合这么复杂的架构

### Exp #7: SwinV2-Tiny + 全图 4ch + 强增强 + Mixup/CutMix

**定位**：针对阶段一过拟合问题的全面改进实验。

**相比 Exp #3 的 5 大改进**：

| 改进项 | Exp #3 (旧) | Exp #7 (新) |
|--------|-------------|-------------|
| **图像裁剪** | ROI 裁剪 (丢失 25% 背景) | **全图** (保留所有上下文) |
| **数据增强** | SyncTransform (弱) | **StrongSyncTransform** (强) |
| **正则化** | 无额外 dropout | **Mixup/CutMix** + head dropout=0.3 |
| **训练轮数** | 60 epochs | **100 epochs** |
| **最小学习率** | min_lr_ratio=0.05 | **0.01** (更慢衰减) |

**StrongSyncTransform vs SyncTransform 对比**：

| 增强操作 | SyncTransform (弱) | StrongSyncTransform (强) |
|---------|-------------------|-------------------------|
| 裁剪 | 固定 Resize | **RandomResizedCrop** (scale 0.7-1.0) |
| 水平翻转 | p=0.5 | p=0.5 |
| 垂直翻转 | p=0.2 | p=**0.3** |
| 旋转 | ±10° (在 affine 中) | **±20°** (独立, p=0.5) |
| 仿射 | 固定: ±10°, ±4%, 0.9-1.1x | p=0.5: ±5°, ±6%, 0.9-1.1x, **shear±5°** |
| 颜色扰动 | p=0.4: brightness±15%, contrast±15% | p=**0.6**: brightness±**30%**, contrast±**30%**, saturation±**20%** |
| 模糊 | 无 | **GaussianBlur**(k=3, p=0.2) |
| 擦除 | 无 | **RandomErasing**(p=0.2, scale 2-15%) |
| 噪声 | 无 | **GaussianNoise**(p=0.3, std=0.03) |

**Mixup/CutMix 机制**：
```python
# 每个 batch 随机选择 Mixup 或 CutMix (各 50% 概率)
# Mixup:  image_mixed = λ * image_A + (1-λ) * image_B     (λ ~ Beta(0.4, 0.4))
# CutMix: 随机矩形区域替换                                  (λ ~ Beta(1.0, 1.0))
# 标签: one-hot soft labels, loss = -(soft_label * log_softmax * class_weight)
```

**数据集使用 `GBPDatasetFull4ch`** — 不做 ROI 裁剪，直接用全图：
```
原始图像 → 生成 lesion_mask → StrongSyncTransform(图像, mask) → [4, 256, 256]
```

**模型**：`timm.create_model("swinv2_tiny_window8_256", ..., drop_rate=0.3)` — timm 内置的 head dropout

**超参数**：backbone_lr=2e-5, head_lr=2e-4, batch=8, epochs=100, warmup=8

**结果**：F1=**0.6173**, benign recall=41.9%

> **注意**：这个实验的 Mixup 存在 BUG — soft CE loss 没有应用类别权重，在阶段三修复。

### Exp #9: SwinV2-Tiny + WeightedSampler + 修复 Mixup 权重

**定位**：修复阶段二的 Mixup 类权重 BUG + 引入 WeightedRandomSampler 解决类别不平衡。

**相比 Exp #7 的 2 大改进**：

**改进1 — 修复 Mixup 类权重 BUG**：
```python
# 之前 (BUG): soft CE 没有类别权重，benign 类在 Mixup 时完全失去额外权重
loss = -(soft_labels * log_probs).sum(dim=1).mean()

# 修复后: 加入 class_weight
w = criterion.weight.unsqueeze(0)  # [1, C], benign=1.99, no_tumor=0.67
loss = -(soft_labels * log_probs * w).sum(dim=1).mean()
```

**改进2 — WeightedRandomSampler**：
```python
# 之前: shuffle=True → 每个 batch 平均 benign:no_tumor ≈ 1:3 (随机)
# 现在: WeightedRandomSampler → 每个 batch 平均 benign:no_tumor ≈ 1:1 (均衡)

# 实现: 每个样本的采样权重 = 1/该类样本数
# benign 样本权重 = 1/309, no_tumor 样本权重 = 1/920
# replacement=True, 有放回采样
```

**其他设置**：与 Exp #7 完全相同 (SwinV2 4ch, StrongSyncTransform, Mixup, 100ep, dropout=0.3)

**结果**：F1=0.6052 (默认阈值), **阈值优化后 F1=0.6240** (threshold=0.745)

### Exp #10: SwinV2-Tiny + Focal Loss + Balanced Sampler + 阈值优化

**定位**：最激进的改进方案 — 用 Focal Loss 替代 CE，不搭配 Mixup，训练后搜索最优阈值。

**相比 Exp #9 的 3 大变化**：

**变化1 — Focal Loss 替代 CrossEntropyLoss**：
```python
# CrossEntropyLoss: loss = -w_c * log(p_c)
# Focal Loss:       loss = -α_c * (1-p_c)^γ * log(p_c)
#                   γ=2: 对容易分类的样本降权，聚焦难分样本
#                   α_c = class_weight (benign=1.99, no_tumor=0.67)
```

**变化2 — 去掉 Mixup/CutMix**：
Focal Loss 需要 hard label 来计算 `(1-p_c)^γ`，Mixup 的 soft label 与 Focal Loss 逻辑冲突。

**变化3 — 去掉 Label Smoothing**：
label_smoothing=0.0 (Focal Loss 本身已自带正则化效果)

**阈值优化机制**：
```python
# 训练结束后，加载最优权重，在测试集上搜索最优阈值：
for thresh in range(0.15, 0.75, 0.005):
    preds = (P(benign) >= thresh) → 预测 benign, 否则 no_tumor
    计算 F1(macro)
# 默认 threshold=0.5，实际最优可能在 0.3-0.7
```

**结果**：F1=0.6045 (默认), **阈值优化后 F1=0.6173** (threshold=0.395)

> Focal Loss 在 epoch 10 时 benign recall 高达 **75.2%**！但后续过拟合严重 (train acc 97.6%)，最终 recall 降回 39%。

### Exp #11: ConvNeXt-Tiny + Focal Loss + Balanced Sampler

**定位**：与 Exp #10 完全相同的策略 (Focal Loss + BalancedSampler)，用 CNN 验证策略通用性。

**模型**：ConvNeXt-Tiny 4ch (同 Exp #8 结构), img_size=320

**与 Exp #10 的不同**：
| | Exp #10 (SwinV2) | Exp #11 (ConvNeXt) |
|---|---|---|
| 架构 | Transformer | CNN |
| img_size | 256 | 320 |
| backbone_lr | 2e-5 | 5e-5 |
| head_lr | 2e-4 | 5e-4 |

**结果**：F1=0.6021 (默认), **阈值优化后 F1=0.6131** (threshold=0.535)

---

### Ensemble: 多模型集成

**方法**：加载 6 个最优模型的 softmax 概率取平均 → 阈值优化。

**参与模型**：Exp #3, 7, 8, 9, 10, 11

**最佳组合**：Exp #3 (SwinV2 ROI 4ch) + Exp #9 (SwinV2 Balanced Mixup), threshold=0.53 → **F1=0.6423**
