## 第4页: 数据处理与输入设计

### 输入方式对比

| 输入方式 | 描述 | 示意 |
|---------|------|------|
| **ROI 3ch** | 裁剪胆囊ROI区域，RGB 3通道 | 丢失周围组织上下文 |
| **ROI 4ch** | 裁剪胆囊ROI + 病灶mask作第4通道 | 保留病灶位置信息 |
| **全图 4ch** | 完整超声图 + mask通道 (resize 256×256) | 保留全部上下文 |
| **双分支** | 全图 + ROI 两路输入，共享backbone | 理论最优，实践复杂 |

### 4通道输入实现 (`adapt_model_to_4ch`)
- 复制 ImageNet 预训练权重到前3通道 (保留RGB知识)
- 第4通道 (mask) 使用 Kaiming 初始化 (从头学习)
- 适用于所有架构 (SwinV2, ConvNeXt, DeiT, MaxViT, ResNet)

### 数据增强策略

| 弱增强 (SyncTransform) | 强增强 (StrongSyncTransform) |
|------------------------|---------------------------|
| RandomHorizontalFlip | RandomResizedCrop (0.7~1.0) |
| Resize 256 | RandomRotation ±20° |
| Normalize | RandomAffine (shear ±15°) |
| | GaussianBlur (随机) |
| | RandomErasing (20%) |
| | GaussianNoise (随机) |
| | ColorJitter (亮度/对比度) |

**同步增强**: 几何变换同时应用于 image 和 mask；颜色变换仅应用于 RGB 通道

## 第5页: 模型架构对比

### 测试的5种Backbone架构

| 模型 | 类型 | 参数量 | 预训练 | 特点 |
|------|------|--------|--------|------|
| **SwinV2-Tiny** | Window-Attention Transformer | 28M | ImageNet-1K | 局部窗口注意力，适合小数据 |
| **ConvNeXt-Tiny** | 现代CNN | 29M | ImageNet-1K | Transformer风格CNN |
| **DeiT3-Small** | ViT Transformer | 22M | ImageNet-1K | 数据高效训练 |
| **MaxViT-Tiny** | Hybrid Transformer | 31M | ImageNet-1K | 多轴注意力 |
| **ResNet-34** | 经典CNN | 21M | ImageNet-1K | CNN baseline |

### 架构实验结论
- **SwinV2-Tiny 一致性最优**: 在 ROI 和全图输入下均表现最好
- **Transformer > CNN**: SwinV2 > DeiT > MaxViT > ConvNeXt > ResNet
- **原因分析**: 窗口注意力机制更适合捕捉息肉的局部纹理特征

| # | 模型 | 关键变化 | F1 (默认) | F1 (阈值优化) |
|---|------|---------|-----------|-------------|
| 12 | SwinV2 双视角+辅助损失 | has_polyp/has_pred/has_adenoma/area辅助任务 | 0.5856 | 0.6029 |

### 重要BUG修复 — Mixup类权重
```python
# BUG: soft CE 未应用类别权重 → benign类在Mixup时失去额外权重
loss = -(soft_labels * log_probs).sum(dim=1).mean()  # ✗

# 修复: 加入类别权重
loss = -(soft_labels * log_probs * weight).sum(dim=1).mean()  # ✓
```
**效果**: benign recall 从 ~20% 提升到 ~42% (最大单次提升)

## 第12页: 第四阶段 — LAT v1 (Lesion-Aware Transformer)

### 设计动机
现有方法对标注信息利用粗糙 (仅mask通道或辅助损失)，需要更精细的特征融合

### LAT v1 架构

```
输入: 超声图 (RGB + Mask 4通道, 256×256)
         ↓
SwinV2-Tiny Backbone → 空间特征 [B, 8, 8, 768]
         ↓
┌─────────────────────────────────────────────┐
│ Token Generation (4 tokens, dim=256)         │
│ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌──────┐│
│ │CLS Token│ │Global   │ │Lesion   │ │Morph ││
│ │(learnable)│ │(mean pool)│ │(mask pool)│ │(12d→256)││
│ └─────────┘ └─────────┘ └─────────┘ └──────┘│
└─────────────────────────────────────────────┘
         ↓
Transformer Encoder (2层, 4头, d=256, pre-norm)
         ↓
CLS output → 分类头 → 2类
CLS output → 投影头 → 128d (SupCon)
```

### 5个创新点
1. **Mask-Guided 空间池化**: 用mask下采样做温度缩放注意力权重
2. **12维形态学特征**: circularity, eccentricity, solidity, convexity 等
3. **Transformer Fusion**: 4 token 自注意力交互 (~1.2M 新参数)
4. **SupCon 辅助损失**: 监督对比学习，拉近同类推远异类
5. **精确度导向阈值**: α×F1 + (1-α)×benign_precision

### v1 结果
| 指标 | LAT v1 (阈值=0.565) | Exp#7 基线 |
|------|---------------------|-----------|
| F1 (macro) | **0.6119** | 0.6173 |
| Benign F1 | 0.4375 | 0.4222 |

**问题**: 移除了 Mixup/CutMix (与SupCon不兼容) → 严重过拟合 (train acc 98%)

## 第13页: 第四阶段 — LAT v2 (修复版)

### v1 → v2 关键改进

| 改动 | v1 | v2 | 原因 |
|------|----|----|------|
| Mixup/CutMix | ✗ 移除 | ✓ 恢复 | 防止过拟合，小数据最关键正则化 |
| SupCon 损失 | ✓ (λ=0.1) | ✗ 移除 | 与 Mixup 软标签不兼容 |
| Dropout | 0.25 | 0.35 | 加强正则化 |
| 输出 | logits + projection | 仅 logits | 简化架构 |

### v2 结果

| 指标 | LAT v2 (阈值=0.245) | Exp#7 基线 | 提升 |
|------|---------------------|-----------|------|
| **F1 (macro)** | **0.6310** | 0.6183 | **+1.3 pp** |
| **Benign F1** | **0.4674** | 0.4222 | **+4.5 pp** |
| Benign Recall | 0.5271 | 0.4186 | +10.9 pp |
| Benign Precision | 0.4198 | 0.4255 | -0.6 pp |
| Accuracy | 0.7036 | 0.7170 | -1.3 pp |

### 分析
- **Benign F1 提升 +4.5pp**: Mask-guided pooling + 形态学特征确实提供了额外信息
- **Benign Recall 大幅提升**: 更多良性肿瘤被正确识别
- **训练稳定**: train acc 在 ~80% (vs v1 的 98%)，过拟合得到控制
- **阈值偏低 (0.245)**: 模型倾向保守预测 benign，需要低阈值"释放"benign预测

## 第17页: 性能天花板分析

### 所有方法收敛到 F1 ≈ 0.64-0.65

无论使用何种技术组合，F1 始终在 0.63-0.65 区间:

| 维度 | 已尝试方法 | F1 范围 |
|------|-----------|---------|
| 模型架构 | SwinV2, ConvNeXt, ResNet, DeiT, MaxViT | 0.58-0.64 |
| 训练策略 | CE, Focal, Mixup, WeightedSampler | 0.59-0.64 |
| 特征提取 | 微调特征, DINOv2自监督, 手工特征 | 0.61-0.65 |
| 集成方法 | 2/5/9模型集成, Stacking, XGBoost | 0.63-0.65 |
| 新架构 | LAT (Transformer Fusion + Morph) | 0.61-0.63 |
| 输入方式 | ROI 3ch/4ch, 全图 4ch, 双分支 | 0.58-0.64 |
| 分辨率 | 256×256, 384×384 | 0.58-0.64 |

### 天花板形成原因

**根本原因 (按重要性排序):**

1. **任务本质难度**: benign vs no_tumor 在超声影像上视觉差异极微小。区别在于息肉的**病理性质** (良性肿瘤 vs 非肿瘤性)，而非影像表现。即使人类放射科医生也需活检确认。

2. **患者零重叠**: 训练/测试来自完全不同患者 → 模型必须学到"疾病本质"而非"个体特征"

3. **训练数据量不足**: benign 仅 309 张 (来自有限患者数)，难以覆盖良性肿瘤的全部变异

4. **标注信息有限**: JSON 仅提供位置/形状，不含组织学级别等关键鉴别信息

## 第18页: 已验证无效的方法

### 避免重复投入的经验总结

| 方法 | F1 变化 | 投入时间 | 结论 |
|------|---------|---------|------|
| DINOv2 自监督特征 | -3 pp | 中 | 通用视觉特征不如任务微调特征 |
| 多架构融合 (SwinV2+ConvNeXt+DeiT) | +1 pp | 高 | 不同架构学到高度重叠信息 |
| PCA 降维 | -2 pp | 低 | 有用信息分散在高维，不可压缩 |
| 标注类型特征 (one-hot) | +0 pp | 低 | 两类标注分布几乎相同 |
| 384 分辨率 | -6 pp | 高 | 更大分辨率 → 更严重过拟合 |
| 多分类器 (XGBoost+LGB+SVM+LR) | +1 pp | 中 | 分类器不是瓶颈，特征才是 |
| 24分类器 Top-K ensemble | +0 pp | 中 | 集成多样性不足 |
| 辅助标注分类损失 | -2 pp | 高 | 辅助任务干扰主任务学习 |
| SupCon 对比学习 | -1 pp | 高 | 小batch对比学习效果差 |

### 核心教训
> **当特征空间中两类样本本质高度重叠时，更强的模型/更多的特征/更好的分类器都无法显著提升性能。瓶颈在数据而非算法。**
