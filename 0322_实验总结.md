# 0322 Task2 实验总结 (完整版)

## 任务概述

- **任务**: Task 2 — 良性肿瘤(benign, label=0) vs 非肿瘤性息肉(no_tumor, label=1) 二分类
- **数据集**: 0322dataset, 训练集 1229 张 (benign=309, no_tumor=920), 测试集 523 张 (benign=129, no_tumor=394)
- **类别不平衡**: benign:no_tumor ≈ 1:3
- **患者重叠**: 训练集与测试集 0% 患者重叠 (完全不同的患者群)
- **目标**: Acc、Precision、Recall、F1 (macro) 均 > 80%
- **JSON标注**: gallbladder ROI 矩形框 + 病灶多边形 (polyp/adenoma 等)

## 关键发现

### 1. Mixup 类权重 BUG (最大影响修复)
- **问题**: soft cross-entropy 没有应用类别权重，Mixup 训练时 benign 类失去额外权重
- **修复**: `loss = -(soft_labels * log_probs * criterion.weight).sum(dim=1).mean()`
- **效果**: benign recall 从 ~20% 提升到 ~42%

### 2. 所有方法收敛到同一天花板 (~0.64-0.65 F1)
无论使用什么模型、特征、分类器或集成策略，F1 都在 0.63-0.65 区间。这包括：
- 5种不同神经网络架构 (SwinV2, ConvNeXt, ResNet, DeiT, MaxViT)
- 多种训练策略 (CE loss, Focal loss, Mixup, WeightedSampler)
- 5种子集成 + TTA
- DINOv2 自监督预训练特征
- XGBoost / LightGBM / SVM / LogReg 多分类器
- 多层 Stacking ensemble
- 8463 维全特征融合

### 3. 数据层面的根本发现

**标注类型分析** — benign 和 no_tumor 的标注几乎完全相同：
| 标注类型 | benign 训练集 | no_tumor 训练集 | 鉴别力 |
|---------|-------------|----------------|--------|
| gallbladder polyp | 57.3% | 69.9% | 弱 (两边都有) |
| pred | 32.4% | 47.7% | 反向弱信号 |
| adenoma | 17.5% | 0.3% | 强，但覆盖率极低 |

**关键洞察**: 两个类别的影像本质上是"有polyp的胆囊" vs "有polyp的胆囊"。区别在于polyp的**病理性质** (良性肿瘤 vs 非肿瘤性)，而非影像表现。adenoma 标注虽然鉴别力极强 (benign 18% vs no_tumor 0.3%)，但覆盖率太低，无法帮助大多数样本。

### 4. ROI 裁剪不如全图
ROI 裁剪丢失约 25% 上下文信息，全图 + mask 作为第4通道是更好的方案。

### 5. 特征重要性
XGBoost 特征重要性分析显示，深度特征主导了分类决策，标注类型特征重要性接近 0。说明标注元数据无法提供深度特征之外的额外信息。

## 未达标原因深度分析

**80% 目标 vs 当前最佳 0.6501 — 差距约 15 个百分点**

### 根本原因 (按重要性排序)

1. **任务本质难度极高**: benign vs no_tumor 本质上是区分"良性肿瘤性息肉"和"非肿瘤性息肉"，这在超声影像上的视觉差异极其微小。即使人类放射科医生也需要依赖活检确认。

2. **患者零重叠**: 训练集和测试集来自完全不同的患者群。模型需要学到"疾病特征"而非"患者特征"，但在小数据量下极难实现。

3. **训练数据不足**: benign 仅 309 张 (来自有限患者)，不足以覆盖良性肿瘤在超声下的全部变异。

4. **标注信息有限**: JSON 标注仅提供病灶位置/形状，不含组织学级别信息。而关键区分依赖的恰恰是组织学特征。

5. **类别不平衡 (1:3)**: 尽管使用了 WeightedSampler 和类别权重，不平衡仍然影响 benign 类的学习。

## 改进建议 (按优先级)

### 优先级 1: 数据层面 (最可能突破瓶颈)

| 方案 | 说明 | 预期效果 |
|------|------|---------|
| **增加 benign 训练数据** | 收集更多良性肿瘤样本至 600+ 张 | ★★★★★ |
| **专家标注质量检查** | 确认 benign vs no_tumor 标签准确性 | ★★★★★ |
| **引入外部数据** | 使用公开胆囊超声数据集做预训练 | ★★★★ |
| **交叉验证** | 5-fold CV 充分利用所有标注数据 | ★★★ |

### 优先级 2: 模型策略 (预期提升有限, 1-3 pp)

| 方案 | 说明 | 预期效果 |
|------|------|---------|
| **SWA (随机权重平均)** | 对训练后期多个 checkpoint 权重平均 | ★★ |
| **知识蒸馏** | 用大模型 (Swin-V2-Base) 蒸馏 | ★★ |
| **对比学习预训练** | SimCLR/BYOL 在全部图片上无监督预训练 | ★★ |

### 优先级 3: 任务重新定义

| 方案 | 说明 |
|------|------|
| **降低目标** | 若临床可接受，目标调整为 F1>70% |
| **多任务学习** | 同时做分类 + 病灶分割 |
| **两阶段方法** | 先分割 → 再对病灶做精细分类 |
| **引入临床信息** | 患者年龄、性别、胆囊壁厚度等临床特征 |

## 结论

> **当前 0322dataset 上 benign vs no_tumor 的二分类, 性能天花板约为 F1=0.64-0.65。**
>
> 这个天花板由数据本身决定：
> - 训练样本少 (benign仅309张)
> - 患者无重叠 (纯泛化)
> - 两类影像表现几乎相同 (区别在病理学而非影像学)
>
> **在不增加/改善数据的情况下，模型性能已接近该数据集的理论上限。**
>
> **最佳策略**: Stacking Ensemble (14预测器) → F1=0.6501, Acc=74.0%
>
> **达到 80% 目标的唯一路径**: 增加高质量标注数据 + 引入临床信息 + 专家标注质量审核。
