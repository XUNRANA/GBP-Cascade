# 0322 Task2 实验总结

## 任务概述

- **任务**: Task 2 — 良性肿瘤(benign, label=0) vs 非肿瘤性息肉(no_tumor, label=1) 二分类
- **数据集**: 0322dataset, 训练集 1229 张 (benign=309, no_tumor=920), 测试集 523 张 (benign=129, no_tumor=394)
- **类别不平衡**: benign:no_tumor ≈ 1:3
- **目标**: Acc、Precision、Recall、F1 (macro) 均 > 80%
- **JSON标注**: gallbladder ROI 矩形框 + 病灶多边形 (polyp/adenoma 等)

## 关键发现

### 1. 发现并修复 Mixup 类权重 BUG
- **问题**: `train_one_epoch` 中 Mixup 的 soft cross-entropy 没有应用类别权重，导致 benign 类在大部分 batch 中完全失去额外权重
- **修复**: `loss = -(soft_labels * log_probs * criterion.weight).sum(dim=1).mean()`
- **效果**: benign recall 从 ~20% 提升到 ~42% (Exp #7 vs Exp #8)

### 2. ROI 裁剪反而不如全图
- ROI 裁剪丢失了约 25% 的上下文信息，而那些背景信息可能包含有用的区分特征
- 全图 + 病灶 mask 作为第4通道是更好的方案

### 3. 严重过拟合
- 所有模型 train acc 快速到 85-97%，但 test F1 停在 0.58-0.62
- 强增强(StrongSyncTransform) + Mixup/CutMix 降低了过拟合但没有本质提升泛化
- Focal Loss 不搭配 Mixup 时过拟合最严重 (train acc 97.6%)

### 4. 阈值优化有帮助但有限
- 默认 0.5 阈值下 benign recall 极低
- 阈值优化平均提升 F1 约 0.01-0.02

### 5. 模型集成效果有限
- 因为各模型学到的特征高度相似，多样性不足
- 最佳 pair ensemble F1=0.6423，仅比单模型提升 0.025

---

## 未达标原因分析

**80% 目标 vs 当前最佳 0.6423 — 差距巨大 (约 16 个百分点)**

### 根本原因

1. **数据量严重不足**: benign 训练样本仅 309 张，且类别比 1:3 不平衡
2. **类间差异极小**: benign (良性肿瘤) vs no_tumor (非肿瘤息肉) 的影像表现非常相似，都有小面积的病灶 (1-2% 图像面积)，肉眼难以区分
3. **病灶面积差异微弱**: benign 平均 2.09% vs no_tumor 1.39%，仅差 0.7 个百分点
4. **标注信息有限**: JSON 标注仅提供病灶位置/范围，无法提供组织学级别的区分信息

## 改进建议 (按优先级)

### 优先级 1: 数据层面 (最可能突破瓶颈)

| 方案 | 说明 | 预期效果 |
|------|------|---------|
| **增加 benign 训练数据** | 收集更多良性肿瘤样本，至少到 600+ 张 | ★★★★★ |
| **引入外部数据** | 使用公开胆囊超声数据集做预训练或混合训练 | ★★★★ |
| **专家标注质量检查** | 确认 benign vs no_tumor 标签准确性，排除标注噪声 | ★★★★ |
| **离线增强扩充** | 对 benign 样本做 offline 增强 (弹性变形、style transfer)，扩充至 3x | ★★★ |

### 优先级 2: 模型/训练策略

| 方案 | 说明 | 预期效果 |
|------|------|---------|
| **多种子集成** | 同一架构、5 个 seed 训练，概率平均 | ★★★ |
| **K-Fold 交叉验证** | 5-fold CV 充分利用所有数据 | ★★★ |
| **知识蒸馏** | 用大模型 (Swin-V2-Base) 蒸馏到小模型 | ★★ |
| **TTA (测试时增强)** | 测试时翻转/旋转取平均 | ★★ |
| **SWA (随机权重平均)** | 对训练后期多个 checkpoint 权重平均 | ★★ |

### 优先级 3: 特征工程

| 方案 | 说明 | 预期效果 |
|------|------|---------|
| **病灶形态学特征** | 提取面积、周长、圆形度、凸包率等手工特征 + ML 分类器 | ★★★ |
| **多尺度融合** | 全图 + ROI + 病灶局部三级裁剪拼接 | ★★ |
| **注意力约束** | 用 mask 做 Guided Attention，强制模型关注病灶区域 | ★★ |

### 优先级 4: 任务重新定义

| 方案 | 说明 |
|------|------|
| **降低目标** | 如果临床场景可接受，将目标调整为 F1>70% |
| **引入多任务学习** | 同时做分类 + 病灶分割，分割任务提供额外监督信号 |
| **两阶段方法** | 先检测/分割病灶 → 再对病灶区域做精细分类 |

---

## 结论

> **当前 0322dataset 上 benign vs no_tumor 的二分类，F1 达到 80% 非常困难。**
> 根本原因是 benign 训练样本量不足 (309张) + 类间视觉差异极小。
> 在不增加数据的情况下，模型性能天花板约在 F1=0.64-0.65 (ensemble + 阈值优化)。
> **建议优先从数据层面突破**: 增加 benign 样本、确认标注质量、引入外部数据。
