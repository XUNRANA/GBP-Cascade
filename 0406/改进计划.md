# 0406 改进计划 — 基于 Exp#4 最优模型

## 当前最优模型回顾 (0402 Exp#4)

| 项目 | 值 |
|------|-----|
| **架构** | SwinV2-Tiny@256 + 4ch(RGB+mask) + UNet Seg Decoder + Seg-Guided Attention + 6D Metadata |
| **F1 (macro)** | 0.6707 (阈值优化后 0.6741) |
| **Benign** | Precision=0.5000, Recall=0.5116, F1=0.5057 |
| **No_tumor** | Precision=0.8389, Recall=0.8325, F1=0.8357 |
| **分割质量** | Dice=0.9851, IoU=0.9706 |
| **最佳 Epoch** | 10 (之后持续过拟合) |

### 核心瓶颈

1. **严重过拟合**: Epoch 10 达峰后持续下降, 训练准确率 99.8% vs 测试 75.3%
2. **Benign precision 仅 0.50**: 一半预测为 benign 的样本实际是 no_tumor
3. **非视觉信息决定性**: Metadata 贡献 +6% F1, 说明病理区分依赖临床特征而非纯影像

### 已尝试且失败的方向 (避免重复)

| 方向 | 实验 | F1 | 失败原因 |
|------|------|-----|---------|
| 监督对比学习 SupCon | Exp#9 | 0.6172 | 小 batch 正负样本对不足 |
| 输入级 Mixup | Exp#2 | 0.5905 | 混合图像破坏分割 mask 语义 |

| 3-Seed 简单集成 | Exp#10 | 0.6326 | 同分布种子, 模型多样性不足 |
| Focal Loss | 0319 | - | 不如 class weight 有效 |

## 四个新实验设计

---

### Exp#11: Exp#4 + 12D 形态学特征扩展

**核心思路**: Metadata 贡献了 +6% F1, 证明非视觉特征对此病理区分至关重要。当前仅用 6D 临床特征 (age/gender/size_mm/size_bin/flow_bin/morph_bin), 完全忽略了**病灶形态学信息** — 而形态学是放射科鉴别良性肿瘤 vs 非肿瘤性息肉的经典手段。

**改进方案**:
- 利用 `task2_lat_v1.py` 中已实现的 `compute_morph_features()` 函数, 从原始 mask 提取 12D 形态学描述符
- 在数据增强**之前**从原始 PIL mask 计算, 确保反映真实病灶形态
- 与原有 6D 临床特征拼接, 构成 18D 扩展元数据

**12D 形态学特征**:

| 编号 | 特征 | 含义 | 对分类的意义 |
|------|------|------|-------------|
| 0 | area_ratio | 病灶面积/图像面积 | 良性肿瘤通常更大 |
| 1 | circularity | 圆度 (1=完美圆) | 良性肿瘤形态更规则 |
| 2 | eccentricity | 离心率 (0=圆, 1=线段) | 形状拉伸程度 |
| 3 | solidity | 密实度 (面积/凸包面积) | 良性肿瘤边界更光滑 |
| 4 | extent | 填充度 (面积/bbox面积) | 区分团块vs弥漫 |
| 5 | aspect_ratio | 长宽比 | 病灶生长方向 |
| 6 | pa_ratio | 周长/sqrt(面积) | 边界复杂度/粗糙度 |
| 7 | convexity | 凸度 (凸包周长/周长) | 边缘规则性 |
| 8 | equiv_diameter | 等效直径 (归一化) | 绝对大小 |
| 9 | norm_perimeter | 归一化周长 | 边界长度 |
| 10 | num_components | 连通分量数 | 多灶 vs 单灶 |
| 11 | largest_ratio | 最大分量/总面积 | 主灶占比 |

**配置变更** (相对 Exp#4):

```python
meta_dim = 18        # 6 clinical + 12 morphological (原 6)
meta_hidden = 128    # 更大容量处理 18D 输入 (原 64)
num_epochs = 30      # Exp#4 在 epoch 10 达峰, 无需 100 轮 (原 100)
eval_interval = 2    # 更频繁评估 (原 5)
```

**实现要点**:
- 新建 Dataset 类, 在 `__getitem__` 中调用 `compute_morph_features(mask_pil)` (增强前)
- `meta_tensor = torch.cat([clinical_6d, morph_12d])` → 18D
- 对 morph 特征同样做 z-score 归一化 (训练集统计量)
- 模型 fusion_in = 256 + 128 = 384 → cls_mlp(384→128→2)

**预期效果**: +1~3% F1。形态学特征直接编码病灶形状信息, 尤其 circularity 和 solidity 对区分良性肿瘤 (形态规则) vs 非肿瘤息肉 (形态不规则) 有诊断价值。

**GPU**: GPU 0 | **训练时间**: ~10 分钟 (30 epochs)

### Exp#12: Exp#4 + 验证集划分 + Early Stopping + EMA

**核心思路**: Exp#4 最致命的问题是**没有验证集** — 直接在测试集上选最优 epoch, 导致 (1) 报告 F1 有乐观偏差, (2) 没有早停机制, 训练 100 轮中 90 轮在过拟合。Exp#7 尝试了 EMA 但同时改了太多 (强增强 + 更大 dropout + 更大 weight_decay), 导致退步。本实验**仅隔离训练流程改进**, 架构和增强与 Exp#4 完全相同。

**改进方案**:
1. **分层验证集划分**: 训练集 85/15 分层划分 (1045 train / 184 val, 保持 1:3 类别比)
2. **Early Stopping**: 监控 val F1 macro, patience=10
3. **EMA**: decay=0.9995 (比 Exp#7 的 0.999 更温和, 避免干扰)
4. **模型选择**: 保存 val F1 最优的 EMA checkpoint, 最终在 test 集上评估

**配置变更** (相对 Exp#4):

```python
num_epochs = 50          # 早停通常在 15-20 轮触发 (原 100)
eval_interval = 1        # 每轮评估, 确保精确早停 (原 5)
val_ratio = 0.15         # 验证集比例 (新增)
patience = 10            # 早停耐心值 (新增)
ema_decay = 0.9995       # EMA 衰减率 (新增)
# 其余所有参数与 Exp#4 完全相同
```

**实现要点**:
- 使用 `sklearn.model_selection.StratifiedShuffleSplit` 划分 train/val
- 创建 3 个 DataLoader: train (有增强), val (无增强), test (无增强)
- 训练循环: train → eval(val) → 检查 best val F1 → 保存/计数patience → 早停
- EMA: 每个 training step 后更新, val 评估时用 EMA 权重
- 最终: 加载 best EMA checkpoint → test 集评估 + 阈值优化
- 导入 `ModelEMA` 类 from `seg_cls_utils_v3.py`

**预期效果**: +0.5~2% F1 (诚实提升)。主要收益不是绝对数值跳跃, 而是: (1) F1 报告不再乐观偏差, (2) 模型稳定在最优 epoch 附近, (3) EMA 平滑权重提升泛化。

**GPU**: GPU 1 | **训练时间**: ~8 分钟 (约 20 轮早停)

### Exp#13: Exp#4 + Feature-Level Mixup (特征空间混合)

**核心思路**: 输入级 Mixup 在 Exp#2 中灾难性失败 (F1=0.5905), 因为混合图像破坏了分割 mask 的语义目标。但 Mixup 是最有效的过拟合对抗手段之一, 而过拟合正是 Exp#4 的核心问题。**关键洞察: 在 seg-guided attention 输出之后的特征空间做 Mixup**, 此时 256D cls_feat 已是紧凑的样本表示, 混合它不影响分割分支。

**原理图**:

```
Input 4ch ──→ SwinV2 Encoder ──→ UNet Decoder ──→ seg_logits (不受影响!)
                    │
                    └──→ f2 ──→ Seg-Guided Attention ──→ cls_feat (256D)
                                                              │
                                                    ┌────────┴────────┐
                                                    │  Feature Mixup  │  ← 仅在此处混合!
                                                    │  (训练时, p=0.5) │
                                                    └────────┬────────┘
                                                              │
                                              + meta_feat (64D) → 320D
                                                              │
                                                         cls_mlp → logits
                                                              │
                                                    Soft CE Loss (混合标签)
```

**Mixup 细节**:
- 训练时以 p=0.5 概率触发
- `lambda ~ Beta(0.4, 0.4)` (偏向两端, 避免过度模糊)
- 在 320D 融合特征 (cls_feat + meta_feat) 上混合
- 分类损失使用 soft cross-entropy (混合后的 one-hot 标签)
- 分割损失完全不受影响 (使用原始未混合的 mask)

**配置变更** (相对 Exp#4):

```python
num_epochs = 50          # Mixup 正则化可能允许训练更久 (原 100)
eval_interval = 2        # (原 5)
mixup_prob = 0.5         # Mixup 触发概率 (新增)
mixup_alpha = 0.4        # Beta 分布参数 (新增)
# 其余与 Exp#4 相同
```

**实现要点**:
- 修改模型 forward, 拆分为两阶段:
  1. `forward_features()`: backbone → seg → attention → 返回 (seg_logits, cls_feat_320d)
  2. `forward_cls(cls_feat)`: cls_mlp → cls_logits
- 训练循环中:
  ```python
  seg_logits, cls_feat = model.forward_features(images, metadata)
  if training and random.random() < mixup_prob:
      lam = np.random.beta(alpha, alpha)
      idx = torch.randperm(B)
      cls_feat = lam * cls_feat + (1-lam) * cls_feat[idx]
      labels_mixed = lam * F.one_hot(labels, 2) + (1-lam) * F.one_hot(labels[idx], 2)
  cls_logits = model.forward_cls(cls_feat)
  seg_loss = seg_criterion(seg_logits, seg_targets)  # 原始, 未混合
  cls_loss = soft_cross_entropy(cls_logits, labels_mixed)  # 混合标签
  ```

**预期效果**: +1~3% F1。Feature-level Mixup 专门正则化分类头 (过拟合瓶颈), 同时保护表现优异的分割分支。平滑决策边界特别有助于提升 benign precision (减少 no_tumor 被误判为 benign)。

**GPU**: GPU 2 | **训练时间**: ~15 分钟 (50 epochs)

### Exp#14: 5-Fold CV + TTA + Morph 集成 (生产级流水线)

**核心思路**: 综合前三个实验的成果, 构建最终生产级推理流水线。两个关键创新:

1. **5-Fold 交叉验证**: Exp#10 的 3-seed 集成失败 (F1=0.6326) 因为所有模型看到相同训练数据, 仅种子不同 → 模型高度相关。K-Fold 让每个模型看到不同训练子集, 产生**真正多样化**的模型
2. **TTA (测试时增强)**: 对每张测试图像生成多个增强视角, 平均预测, 降低预测方差 — 完全免费的推理时提升

**流水线架构**:

```
训练阶段 (5-Fold):
  Fold 1: Train(980样本) → Val(249样本) → Best Model_1 (with EMA + Early Stopping)
  Fold 2: Train(980样本) → Val(249样本) → Best Model_2
  Fold 3: Train(983样本) → Val(246样本) → Best Model_3
  Fold 4: Train(983样本) → Val(246样本) → Best Model_4
  Fold 5: Train(983样本) → Val(246样本) → Best Model_5

推理阶段 (5 Models × 5 TTA Views):
  每张测试图 → 5种增强视角:
    ① 原图
    ② 水平翻转
    ③ 垂直翻转
    ④ 水平+垂直翻转
    ⑤ 轻微旋转 (±5°)
  → 5个模型 × 5个视角 = 25 次前向传播
  → 概率平均 → 阈值优化 → 最终预测
```

**配置**:

```python
# 模型 = Exp#11 架构 (Exp#4 + 18D 扩展 metadata)
meta_dim = 18            # 6 clinical + 12 morphological
meta_hidden = 128

# 训练
n_splits = 5             # 5-Fold CV
num_epochs = 30          # 每 fold
eval_interval = 1
patience = 8             # 早停
ema_decay = 0.9995

# TTA
tta_views = 5            # 增强视角数
```

**实现要点**:
1. `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)` 划分训练集
2. 每个 fold:
   - fold 内训练集统计量做 meta 归一化
   - 训练 with Early Stopping + EMA
   - 保存 best EMA checkpoint
3. TTA 推理函数:
   ```python
   def predict_with_tta(model, img_4ch, metadata):
       views = [img_4ch,                        # 原图
                hflip(img_4ch),                  # 水平翻转
                vflip(img_4ch),                  # 垂直翻转
                vflip(hflip(img_4ch)),           # 双翻转
                rotate(img_4ch, angle=5)]        # 轻微旋转
       probs = [softmax(model(v, metadata)) for v in views]
       return mean(probs)
   ```
4. 集成推理: 5 模型 × 5 TTA → 25 次前向 → 概率平均
5. OOF 预测: 每个样本在其 val fold 的预测, 构成完整训练集的 OOF F1 估计

**预期效果**: +2~5% F1。K-Fold 提供模型多样性 (不同于失败的 seed 集成), TTA 降低预测方差, 形态学特征补充非视觉判别信息。OOF F1 还提供更可靠的泛化性能估计。

**GPU**: GPU 0-3 (5 fold 可分配到多卡) | **训练时间**: ~50 分钟 (5 folds × 10 分钟)

## 实验总览

| 编号 | 名称 | 改进方向 | 针对问题 | 预期 F1 提升 |
|------|------|---------|---------|-------------|
| **Exp#11** | + 12D 形态学特征 | 特征扩展 | Benign precision 低 (缺少形状鉴别力) | +1~3% |
| **Exp#12** | + Val Split + Early Stop + EMA | 训练流程 | 严重过拟合 + 测试集选模型偏差 | +0.5~2% |
| **Exp#13** | + Feature-Level Mixup | 正则化 | Epoch 10 即过拟合 | +1~3% |
| **Exp#14** | 5-Fold CV + TTA + Morph | 集成推理 | 模型多样性不足 + 推理方差 | +2~5% |
