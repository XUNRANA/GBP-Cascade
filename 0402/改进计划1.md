## Exp#4 的弱点分析

虽然 Exp#4 超越了基线, 但仍有明显问题:

| 问题 | 证据 | 可能的解决方案 |
|------|------|-------------|
| **Epoch 10 就到峰值** | 后 90 epoch 浪费 | 更强正则化 / Early Stopping / 短训练 |
| **benign precision 只有 0.50** | 一半预测为 benign 的样本其实是 no_tumor | 更好的特征区分 / 对比学习 |
| **过拟合严重** | Train 99% vs Test 75% | DropPath / 更强 augmentation / 更大 weight decay |
| **Seg-Guided Attn 依赖 seg 质量** | seg Dice=0.985 时效果好, 更低时效果差 | 多尺度 attention / 加 base attention |

## 改进策略

### 策略 A: 更强正则化 (治疗过拟合)

Exp#4 的核心问题是过拟合。不加 Mixup (因为和分割冲突), 需要其他正则化手段:

```
可选的正则化手段:
  1. DropPath (Stochastic Depth) — SwinV2 原生支持, drop_path_rate=0.2→0.3
  2. 更大 weight_decay — 5e-2 → 1e-1
  3. Label Smoothing 加大 — 0.1 → 0.2
  4. 更强 Dropout — cls_dropout 0.4 → 0.5, meta_dropout 0.2 → 0.3
  5. Feature Dropout (在 f2 特征上加 spatial dropout)
  6. 更强数据增强 — RandAugment, 更大旋转角度, GridDistortion
  7. EMA (Exponential Moving Average) — 训练时维护 EMA 权重, 推理用 EMA
  8. Early Stopping — 10-15 epoch patience
```

### 策略 B: 多尺度 Seg-Guided Attention

当前 Exp#4 只用 f2 (16x16) 的 seg attention, 可以融合多尺度特征:

```
f3 (8x8, 768ch) → seg_attn → pool → feat_coarse
f2 (16x16, 384ch) → seg_attn → pool → feat_fine
concat(feat_coarse, feat_fine, metadata) → fusion MLP → 2 class
```

### 策略 C: 减少训练 Epoch + Cosine Restart

既然 Epoch 10 就到峰值, 两个思路:
1. 只训练 30 epoch, 加大 LR warmup
2. 用 Cosine Annealing with Warm Restart, 让模型有多次"重新开始"的机会

### 策略 D: 对比学习 (拉开 benign vs no_tumor)

benign precision 低说明特征空间中两类混叠。加入 Supervised Contrastive Loss:
```
Loss = seg_loss + lambda_cls * cls_loss + lambda_con * SupConLoss
```

### 策略 E: Ensemble (多 seed / 多模型)

Exp#4 的 Epoch 10 是单一最优点, 不同 seed 可能有不同最优点:
```
Seed 42: best at Epoch 10
Seed 123: best at Epoch X
Seed 456: best at Epoch Y
→ 3 模型投票 / 概率平均
```

## Round 3: 4 个并行实验

### Exp#7: Exp#4 + 强正则化 (GPU 0) — 最有希望

**核心思路**: 在 Exp#4 基础上加全套正则化, 解决过拟合问题。

```
Exp#4 原配置 + 以下改动:
  - drop_path_rate = 0.3 (SwinV2 stochastic depth)
  - weight_decay = 0.1 (加倍)
  - label_smoothing = 0.15
  - cls_dropout = 0.5
  - meta_dropout = 0.3
  - 更强 augmentation: 旋转±30°, 弹性变形, GridDistortion
  - EMA decay = 0.999
  - 50 epochs (不需要 100 了)
  - eval_interval = 2 (更频繁评估)
```

**预期**: F1 > 0.6707, 峰值出现更晚 (Epoch 15-25), 曲线更平滑

---

### Exp#8: Exp#4 + 多尺度 Attention (GPU 1)

**核心思路**: 用 f2 + f3 两个尺度的 seg-guided attention, 捕获粗细两种特征。

```
SwinV2-Tiny@256 + 4ch + metadata

分类头改造:
  f3 (768ch, 8x8) → proj → seg_attn → pool → 256d (粗粒度)
  f2 (384ch, 16x16) → proj → seg_attn → pool → 256d (细粒度)
  concat(coarse_256, fine_256, meta_64) → MLP(576→256→2)

其他同 Exp#4:
  - lambda_cls = 2.0
  - 50 epochs
  - drop_path_rate = 0.2
```

**预期**: 多尺度特征提供更全面的病灶描述, benign precision 提升

---

### Exp#9: Exp#4 + 对比学习 (GPU 2)

**核心思路**: 加入 Supervised Contrastive Loss, 拉开 benign 和 no_tumor 的特征距离。

```
SwinV2-Tiny@256 + 4ch + metadata + Seg-Guided Attn

额外加入:
  - projection head: 256 → 128 → 64 (for SupCon)
  - SupConLoss on cls features (after seg-guided attention pool)
  - Loss = seg_loss + 2.0 * cls_loss + 0.5 * supcon_loss

  - 50 epochs
  - temperature = 0.07
  - cls_dropout = 0.4
```

**预期**: benign 和 no_tumor 的特征更好分离, precision/recall 更均衡

---

### Exp#10: Exp#4 Ensemble (3 seed) (GPU 3)

**核心思路**: 用 3 个不同 seed 训练 Exp#4, 取最优 epoch 的概率平均。

```
Seed 42 → 30 epochs → best model
Seed 123 → 30 epochs → best model
Seed 456 → 30 epochs → best model
→ 推理时 3 模型概率平均 → 投票

每个 seed 训练 30 epoch (Epoch 10 就到峰值, 30 足够)
总训练时间: 3 × 8 min = ~25 min
```

**预期**: Ensemble 平滑单模型的随机性, F1 稳定提升 1-2%

## Round 4 预案 (如果 Round 3 不理想)

### 方向 A: 知识蒸馏
- 用 Exp#4 作为 teacher, 训练更鲁棒的 student
- Teacher 提供 soft labels, Student 学到更平滑的决策边界

### 方向 B: Test-Time Augmentation (TTA)
- 推理时对测试图片做 N 次增强 (翻转/旋转/缩放)
- 概率平均, 不需要重新训练
- 在 Exp#4 最优权重上直接应用

### 方向 C: 更大模型
- SwinV2-Small 或 SwinV2-Base
- 更多参数可能更好利用 seg-guided attention

### 方向 D: 传统特征 + 深度特征融合
- 从分割 mask 提取形态学特征 (面积, 圆度, 纹理)
- 与 CNN 特征拼接后用 XGBoost 做最终分类
