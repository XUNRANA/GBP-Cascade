# 0402 分割+分类联合训练实验总结

## 1. 实验背景

### 1.1 动机
模仿 `unet-valjustclass` 项目的分割+分类联合训练思路，探索：**让 SwinV2-Tiny 同时学习病灶分割和良性/非肿瘤分类，分割能否帮助提升分类性能？**

### 1.2 基线对比
| 基线实验 | 方法 | F1 (macro) | Acc | 备注 |
|---------|------|-----------|-----|------|
| Exp#7 (0323) | SwinV2-Tiny + 4ch(RGB+mask) + metadata + Mixup | **0.6371** | 0.7533 | 当前最佳 |
| Exp#7 (阈值优化) | 同上, threshold=0.505 | **0.6388** | — | — |

### 1.3 核心改变
- **输入**: 4ch (RGB + 病灶mask) → **3ch RGB** (mask 不再作为输入)
- **新增**: UNet 分割解码器，病灶 mask 变成分割目标
- **目标**: 分割迫使模型学习病灶空间特征 → 提升分类

## 2. Exp#1: 基础双头联合训练

### 2.1 配置
| 项目 | 值 |
|------|-----|
| 模型 | SwinV2-Tiny encoder + UNet decoder + GAP cls head |
| 输入 | 3ch RGB, 256x256 |
| 分割 | 2类 (背景/病灶), CE+Dice loss, 病灶权重=5.0 |
| 分类 | 2类 (benign/no_tumor), CE loss, label_smoothing=0.1 |
| **lambda_cls** | **1.0** |
| Mixup | **无** |
| 参数量 | 34.6M (encoder 27.6M + decoder+cls 7.1M) |
| 优化器 | AdamW, backbone_lr=2e-5, head_lr=2e-4 |
| 训练 | 100 epochs, warmup 8ep, cosine decay |

### 2.2 最终结果
| 指标 | Exp#1 | 基线 |
|------|-------|------|
| **F1 (macro)** | **0.6157** | **0.6371** |
| F1 (阈值优化) | 0.6215 (thresh=0.450) | 0.6388 |
| Acc | 0.7228 | 0.7533 |
| benign P/R/F1 | 0.43 / 0.40 / 0.41 | — |
| no_tumor P/R/F1 | 0.81 / 0.83 / 0.82 | — |
| **Seg Lesion Dice** | **0.8643** | 无 |
| **Seg Lesion IoU** | **0.7632** | 无 |

### 2.3 训练曲线分析

**分割学习曲线 (非常好)**:
```
Epoch  1: Dice=0.18 → Epoch 10: 0.84 → Epoch 50: 0.89 → Epoch 100: 0.91
```
分割在 ~20 epoch 就基本收敛，泛化良好 (train 0.91 vs test 0.86)。

Train/Test 差距 27 个百分点，严重过拟合。

### 2.4 问题诊断

| 问题 | 证据 | 根因 |
|------|------|------|
| 分类没超基线 | F1: 0.6157 < 0.6371 | 多重因素叠加 |
| 严重过拟合 | Train 99.5% vs Test 72% | 无 Mixup, 无 metadata |
| benign recall 低 | 39.5% | 类别不平衡 (3:1) |
| 分割主导训练 | seg_loss 早期 >> cls_loss | lambda_cls=1.0 太低 |
| 分类头信息不足 | 只用 f3 (8x8, 768ch) | 丢失了空间细节 |

## 3. Exp#2: 多尺度分类头 + Mixup + 高 lambda

### 3.1 配置 (vs Exp#1 的改变)
| 项目 | Exp#1 | Exp#2 |
|------|-------|-------|
| **lambda_cls** | 1.0 | **3.0** |
| **Mixup/CutMix** | 无 | **有 (img+mask同步)** |
| **分类头** | GAP(f3) → MLP | **多尺度 GAP(f3+d3+d2+d1) → MLP** |
| **Dropout** | 0.3 | **0.4** |
| 参数量 | 34.6M | 34.9M |

### 3.2 最终结果 (截止 Epoch 84, 仍在运行)
| 指标 | Exp#2 (截止ep84) | Exp#1 | 基线 |
|------|---|---|---|
| **Best F1** | **0.5905** (Epoch 10) | **0.6157** | **0.6371** |
| Best Acc | 0.7495 (Epoch 60) | 0.7228 | 0.7533 |
| Seg Dice (train) | ~0.83 | ~0.91 | — |

### 3.3 Test F1 完整轨迹
```
Ep5:  0.496 → Ep10: 0.591 (best!) → Ep15: 0.573 → Ep20: 0.565
Ep25: 0.575 → Ep30: 0.553 → Ep35: 0.587 → Ep40: 0.568
Ep45: 0.564 → Ep50: 0.554 → Ep55: 0.568 → Ep60: 0.562
Ep65: 0.564 → Ep70: 0.574 → Ep75: 0.555 → Ep80: 0.567
```

### 3.4 问题诊断

| 问题 | 证据 | 根因 |
|------|------|------|
| **F1 在 Epoch 10 就到峰值** | 后续 90 个 epoch 都没超过 0.5905 | Mixup 过度正则化 |
| **F1 剧烈震荡** | 0.49~0.59 之间波动 | Mixup 引入太大噪声 |
| **分割也受损** | train Dice 只到 0.83 (vs Exp#1 的 0.91) | Mixup 混合 mask 语义不合理 |
| **benign recall 极低** | 最低到 0.22 (Ep80) | 模型严重偏向 no_tumor |
| Train Acc 也不稳定 | 80%~85% 之间波动 | 学习不收敛 |

**核心结论: Mixup/CutMix 对分割+分类联合训练有害！**
- 混合两张图的病灶 mask 没有语义意义
- Mixup 产生的 soft label 让分类更难学
- 分割和 Mixup 是冲突的正则化策略

## 4. 综合对比与发现

### 4.1 三方对比表

| | 基线 (纯分类+4ch) | Exp#1 (seg+cls) | Exp#2 (seg+cls+mixup) |
|---|---|---|---|
| F1 (macro) | **0.6371** | 0.6157 | 0.5905 |
| F1 (阈值优化) | **0.6388** | 0.6215 | — |
| Acc | **0.7533** | 0.7228 | ~0.73 |
| Seg Dice | — | **0.8643** | ~0.82 |
| 过拟合程度 | 中 | 严重 | 最严重 |
| 训练稳定性 | 好 | 好 | 差 (震荡) |

### 4.2 关键发现

**正面发现:**
1. 分割任务本身学得很好 (Dice 0.86), 证明标注质量高、模型能力足够
2. UNet decoder + skip connections 在 SwinV2 上工作正常
3. 分割确实能引导 encoder 学到有意义的空间特征

**负面发现:**
1. **分割没有直接帮助分类** — F1 都没超过基线
2. **去掉第4通道 mask 输入是关键损失** — 基线把 mask 作为输入，直接告诉模型"病灶在哪"
3. **Mixup 与分割冲突** — 混合 mask 破坏了语义
4. **分类头设计不够好** — 只用 GAP 的分类头不如基线的方式

### 4.3 为什么分割没帮到分类？

根本原因分析:

1. **信息瓶颈**: 基线把 mask 作为第4通道输入 = 直接告诉模型"病灶在哪里"。分割+分类方式需要模型自己学会定位病灶，这增加了学习难度但没带来新信息。

2. **任务耦合太弱**: Exp#1 里分割头和分类头只共享 encoder，分割学到的空间特征没有直接被分类头利用 (分类头只做 GAP，丢失了空间信息)。

3. **训练资源竞争**: 分割 loss 在早期主导优化方向，encoder 被优化去做像素级任务而非全局分类。

4. **缺少正则化**: 去掉 Mixup 后没有替代正则化手段，导致分类严重过拟合。

## 5. 下一步方向

> 详见 `改进计划.md`

核心思路:
- **不要放弃 mask 输入** — 用 4ch 输入 + 分割作为辅助任务/正则化器
- **分割引导分类** — 用 seg 预测作为 attention map
- **保留有效手段** — metadata, 阈值优化
- **利用 4x A100 并行** — 每轮跑 4 个实验
