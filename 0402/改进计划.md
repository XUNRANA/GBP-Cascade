# 0402 分割+分类改进计划

> 基于 Exp#1 / Exp#2 的分析结果，设计下一轮 4 个并行实验 (4x A100)

---

## 核心教训回顾

| 教训 | 来源 |
|------|------|
| mask 作为输入通道有价值，不应去掉 | 基线 F1=0.637 > 去掉后 0.616 |
| 分割本身学得很好 (Dice=0.86) | Exp#1 |
| Mixup/CutMix 与分割冲突，不要同时用 | Exp#2 F1=0.59, 震荡严重 |
| 分类头只做 GAP 会丢失空间信息 | Exp#1 分类过拟合 |
| lambda_cls=1.0 太低导致分割主导训练 | Exp#1 loss 分析 |
| metadata (age/gender/size/flow/morph) 有帮助 | 基线对比 |


## Round 1: 4 个并行实验 (4x A100)

### Exp#3: 4ch输入 + 分割辅助正则化 (GPU 0) -- 最有希望

**核心思路**: 保留基线的 4ch 输入 (RGB + mask)，但额外加一个分割解码器作为 **辅助任务/正则化器**。分割头不影响分类头的输入，而是通过共享 encoder 来正则化特征学习。

```
输入: [RGB + lesion_mask] (4ch, 同基线)
      ↓
SwinV2-Tiny encoder (4ch adapted)
      ↓
┌─────────────┐    ┌──────────────────┐
│ 分类头 (主)   │    │ UNet分割头 (辅助)  │
│ GAP → MLP    │    │ decoder → seg    │
│ → 2class     │    │ → 2class mask    │
└─────────────┘    └──────────────────┘

Loss = cls_loss + lambda_seg * seg_loss
     (主任务)      (辅助正则化)
```

**关键配置:**
- 输入: 4ch (RGB + mask), 256x256
- lambda_seg = 0.5 (分割是辅助, 不要主导!)
- 分类: CE + class weights + label_smoothing=0.1
- 分割: CE + Dice, 病灶权重=5.0
- 不用 Mixup
- metadata: 无 (先验证分割辅助的效果)
- 100 epochs

**为什么最有希望:**
- 保留了基线的全部信息 (4ch 输入)
- 分割头作为正则化器，防止 encoder 过拟合
- 分割 loss 权重低 (0.5)，不会抢占分类的训练
- 等于 "基线 + 免费的正则化"

---

### Exp#4: 4ch输入 + 分割引导注意力 + metadata (GPU 1) -- 全特性

**核心思路**: 在 Exp#3 基础上加两个增强：
1. 用分割预测作为 attention map 引导分类
2. 加回 metadata 融合

```
输入: [RGB + lesion_mask] (4ch) + metadata (6维)
      ↓
SwinV2-Tiny encoder (4ch)
      ↓
┌──────────────────────────────┐
│ UNet decoder → seg_logits    │
│ seg_prob → attention map     │
│ attention × encoder_feat     │
│ → weighted pool → concat     │
│ → + metadata_feat            │
│ → fusion MLP → 2class       │
└──────────────────────────────┘

Loss = seg_loss + lambda_cls * cls_loss
```

**关键配置:**
- 输入: 4ch + metadata (6维)
- lambda_cls = 2.0
- 分割注意力: seg prob 下采样 → 加权 f2 特征
- metadata: age, gender, size_mm, size_bin, flow_bin, morph_bin
- base_attention = 0.1 (防止全0)
- dropout = 0.4
- 100 epochs

---

### Exp#5: 3ch输入 + 分割引导注意力分类 (GPU 2) -- 纯学习

**核心思路**: 不给模型 mask 输入，让模型完全通过分割来学习定位病灶，然后用分割结果引导分类。这是 Exp#1 的改进版。

```
输入: RGB (3ch)
      ↓
SwinV2-Tiny encoder
      ↓
UNet decoder → seg_logits → lesion prob map
                ↓
    下采样到 f2 分辨率 (16x16)
                ↓
    attention = lesion_prob + 0.1 (base)
                ↓
    f2_feat × attention → weighted pool
                ↓
    MLP → 2class
```

**关键配置:**
- 输入: 3ch RGB, 256x256
- lambda_cls = 2.0
- 分割注意力池化
- dropout = 0.4
- 无 Mixup
- 100 epochs

**vs Exp#1 的改进:**
- 分类头改为 seg-guided attention pooling (不再是粗暴的 GAP)
- lambda_cls = 2.0 (Exp#1=1.0)
- 更大 dropout

---

### Exp#6: 两阶段训练 — 先分割再分类 (GPU 3) -- 解耦训练

**核心思路**: 彻底解耦分割和分类的训练，避免任务竞争。

**Stage 1 (50 epochs): 只训练分割**
```
输入: RGB (3ch)
SwinV2-Tiny encoder + UNet decoder
Loss = seg_CE + seg_Dice (只有分割loss)
→ 学到强大的空间特征
```

**Stage 2 (50 epochs): 冻结encoder，只训练分类头**
```
输入: RGB (3ch)
SwinV2-Tiny encoder (冻结!) + 新的分类头
分类头: f2_feat + seg_attention → MLP
Loss = cls_CE (只有分类loss)
→ 在已学好的特征上训练分类
```

**关键配置:**
- Stage 1: 50ep, lr=2e-5 (全模型), 只有 seg_loss
- Stage 2: 50ep, lr=5e-4 (只有 cls head), encoder 冻结
- 分类头: seg-guided attention pooling
- 无 Mixup
- 两阶段分别保存最优模型

**为什么值得试:**
- 消除了分割和分类之间的训练竞争
- Stage 1 让 encoder 完全专注于学空间特征
- Stage 2 在已经很好的特征上训练分类，不会破坏分割能力
- 如果效果好，说明问题出在联合训练的优化冲突

## Round 1 实验总览

| GPU | 实验 | 输入 | 分割 | 分类头 | metadata | Mixup | 关键变量 |
|-----|------|------|------|--------|----------|-------|---------|
| 0 | Exp#3 | 4ch | 辅助(lambda=0.5) | GAP→MLP | 无 | 无 | 4ch + 分割正则化 |
| 1 | Exp#4 | 4ch+meta | 引导注意力 | seg-attn→MLP | 有 | 无 | 全特性集成 |
| 2 | Exp#5 | 3ch | 引导注意力 | seg-attn→MLP | 无 | 无 | 纯分割引导 |
| 3 | Exp#6 | 3ch | 两阶段 | seg-attn→MLP | 无 | 无 | 解耦训练 |

**预期排序**: Exp#4 > Exp#3 > Exp#5 > Exp#6

**每个实验约 25 分钟 (100ep), 4 GPU 并行 = 25 分钟出所有结果**

## Round 2 预案 (如果 Round 1 不理想)

### 方向 A: 放弃分割，回归纯分类优化
- 在基线上加更强的正则化 (DropPath, 更大 weight_decay)
- 尝试 SwinV2-Small 或 SwinV2-Base
- 更多 seed ensemble

### 方向 B: 分割特征提取 + 传统分类器
- 用训练好的分割模型提取特征 (encoder features)
- 用 XGBoost / LightGBM 做分类
- 把分割 mask 的形态学特征 (面积, 圆度, 纹理) 作为额外特征

### 方向 C: 深度监督 + 对比学习
- 在 decoder 每一级加辅助分类头 (deep supervision)
- 用对比学习拉开 benign 和 no_tumor 的特征距离
- SupCon loss + CE loss
