# 20260319 Task2 本轮实验总结

## 1. 本轮实验目标

本轮实验的核心目标是：

- 在 `Task2: benign vs no_tumor` 上，为不同 backbone 补齐更合适的训练 recipe
- 找到当前最强主线模型
- 判断 `img320`、`Focal Loss`、`WeightedRandomSampler` 是否还能继续带来收益
- 用多 seed 验证 `ConvNeXt img320` 的结果是否稳定

本轮实验仍然基于当前统一数据：

- train: `1303` 张，`benign=309`，`no_tumor=994`
- test: `564` 张，`benign=129`，`no_tumor=435`

方法学提醒：

- 当前仍然只有 `train/test`
- `test` 参与了最优 epoch 选择
- 因此本轮结果适合做横向比较，但整体仍略偏乐观

## 3. 本轮实验最重要的结果

### 3.1 当前最优单次结果

如果按原始 `test` 分布、单次实验、`Macro-F1` 排名，本轮最佳是：

- `20260319_task2_ConvNeXtTiny_adamw_warmupcosine_classweight_img320_3`
- `Macro-F1 = 0.6372`
- `Acc = 0.7660`
- `benign recall = 0.3721`
- `benign f1 = 0.4211`

这说明：

- `ConvNeXt` 在模型专用 recipe 下已经成为当前最强主线
- `img320` 相比 `ConvNeXt _2` 的 `0.6325` 仍然有小幅正收益

### 3.2 当前少数类最强结果

如果只看少数类 `benign` 的抓取能力，本轮仍然没有超过：

- `20260319_task2_SwinTiny_adamw_warmupcosine_classweight_2`
- `benign recall = 0.4496`
- `benign f1 = 0.4234`

也就是说：

- `Swin + classweight` 仍然是冲 `benign` 的最好辅助线
- 本轮 `Swin + Focal Loss` 并没有超过它

### 3.3 ConvNeXt img320 多 seed 的稳定性

对 `ConvNeXt img320 + classweight` 的 4 个 seed 做统计后：

| 指标 | mean | std | min | max |
| --- | ---: | ---: | ---: | ---: |
| `Acc` | 0.7700 | 0.0071 | 0.7624 | 0.7784 |
| `Macro-F1` | 0.6312 | 0.0041 | 0.6282 | 0.6372 |
| `benign precision` | 0.4976 | 0.0228 | 0.4742 | 0.5256 |
| `benign recall` | 0.3430 | 0.0256 | 0.3178 | 0.3721 |
| `benign f1` | 0.4051 | 0.0118 | 0.3961 | 0.4211 |
| `no_tumor f1` | 0.8573 | 0.0060 | 0.8514 | 0.8643 |

这组结果说明：

- 这条主线是稳定的，不是单个 seed 偶然冲高
- 但它也明显进入了平台区，继续提升的空间已经不大
- 真正波动相对更大的不是总 `Macro-F1`，而是 `benign recall`

## 4. 结果分析

### 4.1 本轮真正起作用的改动

本轮最有效的变化，不是继续换 backbone，而是：

- 给 `ConvNeXt / Swin / ViT / EfficientNet` 换成更贴模型的 recipe
- 统一使用 `AdamW`
- 使用 backbone/head 差分学习率
- 使用 `warmup + cosine`
- 保留轻量增强
- 对不平衡任务加入 `classweight`

这一轮带来的提升是实质性的，不是小修小补。

### 4.2 为什么 ConvNeXt 现在是主线

`ConvNeXt` 的特点是：

- 总体 `Macro-F1` 最高
- `Acc` 也不低
- `benign f1` 在第一梯队
- 多 seed 波动较小

更重要的是，它不像某些模型那样只能通过偏向多数类拿高分，而是整体比较均衡。

### 4.3 为什么 img320 对 ConvNeXt 有帮助，但不是暴涨

`img320` 相比 `224` 确实是正收益：

- `Macro-F1: 0.6325 -> 0.6372`
- `benign f1: 0.4156 -> 0.4211`

但提升幅度不大，原因很可能是：

- 当前 recipe 已经把大部分“低垂果子”摘掉了
- 更高分辨率只能补一点局部细节，不会改变任务本质难度
- 当前任务的主矛盾仍然是少数类 `benign` 学习难

### 4.4 为什么 Swin 的 Focal Loss 没有带来提升

`Swin + Focal Loss` 的结果是：

- `Macro-F1: 0.6192 -> 0.6137`
- `benign recall: 0.4496 -> 0.4109`
- `benign f1: 0.4234 -> 0.4061`

这说明在当前这条线里：

- `classweight CE` 比 `Focal Loss` 更合适
- `Focal Loss` 没有把少数类再往上推，反而略微拉低了整体平衡

### 4.5 为什么 ResNet34 的新版没有超过旧版

`ResNet34` 这一轮没有继续涨，反而从旧版 `0.6225` 降到 `0.6148`。

关键原因不是单一的 `img320`，而是这次同时改了多项：

- `img320`
- `AdamW`
- `diff lr`
- `warmup cosine`
- `label smoothing`

所以当前只能得出：

- 这一整套改法对 `ResNet34` 不如旧主线
- 不能简单下结论说“ResNet34 不适合 320”

如果还想继续验证 `ResNet34`，应该做纯 ablation：只改分辨率，不改 optimizer recipe。

### 4.6 WeightedRandomSampler 的真实效果

`WeightedRandomSampler` 这版结果是：

- `Acc = 0.7837`
- `Macro-F1 = 0.6350`
- `benign precision = 0.5467`
- `benign recall = 0.3178`
- `benign f1 = 0.4020`

它的特点很清楚：

- `Acc` 更高
- `Macro-P` 更高
- `no_tumor` 类表现更强
- 但 `benign recall` 并没有提高，反而低于当前最好 `seed42`

所以这版 sampler 更像：

- 一条“更保守、偏多数类”的方案
- 不是比 `classweight` 更好的少数类方案

## 5. 当前最合理的结论

### 5.1 主线模型

当前最推荐的主线仍然是：

- `ConvNeXtTiny + AdamW + diff lr + warmup cosine + weakaug + classweight + label smoothing + img320`

如果需要单个代表结果，可以引用：

- `seed42`：`Macro-F1 = 0.6372`

如果需要更稳的报告口径，更推荐写成：

- 4-seed mean `Macro-F1 = 0.6312 ± 0.0041`

### 5.2 辅助主线

当前最适合作为少数类辅助线的仍然是：

- `SwinTiny_adamw_warmupcosine_classweight_2`

理由：

- 它仍然保持最高的 `benign recall / benign f1`
- 本轮 `Focal Loss` 并没有超过它

### 5.3 稳定参考线

当前经典 CNN 里最稳的参考线仍然是：

- `20260318_task2_Resnet34_weakaug_classweight_3`

它仍然有意义，因为后续任何新改动都应该拿它做对照，判断是真提升还是 recipe 偏移。

## 6. 后续改进建议

### 6.1 优先做的事

1. 固定当前主线为 `ConvNeXt img320 + classweight`
2. 后续报告优先使用多 seed 均值，而不是只报单个最好 seed
3. 把 `WeightedRandomSampler` 降级为补充对照，不再作为主线方向

### 6.2 还值得继续试的低成本方向

- `ConvNeXt img320 + TTA`
  - 这是当前最值得补的低成本实验
- `val` 划分或 train 内部验证
  - 当前最大的流程缺口不是模型，而是没有独立验证集
- 如果后续要继续做少数类优化，优先从验证集阈值调优入手

### 6.3 当前不建议继续投入的方向

- 继续在 `Swin + Focal Loss` 上深挖
- 继续在 `WeightedRandomSampler` 上扩大量实验
- 再大范围换新的 backbone

因为这几条线当前都没有表现出比现有主线更高的性价比。

## 7. 一句话总结

本轮实验已经把结论压得比较清楚了：

- `ConvNeXt img320 + classweight` 是当前最强、最稳的主线
- 多 seed 证明这条线有效，但提升已经进入平台区
- `WeightedRandomSampler` 没有带来更好的少数类结果
- `Swin` 仍然适合作为冲 `benign` 的辅助线
- 如果想再提升，重点不该是继续盲目换模型，而应该是 `TTA + 更规范的验证流程 + 更稳的报告方式`
