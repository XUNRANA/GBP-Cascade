## 1. 背景与目标

### 1.1 0502 已完成实验回顾

| 实验 | 核心改动 | 关键结果(Test-112) | 评价 |
|---|---|---|---|
| Baseline | SwinV2 + Seg-Guided + Trimodal Fusion | high_p=0.726, Brier=0.093 | 基准 |
| A SupCon | + 边界对比损失 | high_p=0.704, Brier=0.089 | 持平 |
| **B Calib** | + 非对称 target + 早停 + 温度校准 | **high_p=0.950**, Brier=0.018 | 🥇 综合最优 |
| C TwoStage | risk + ben/mal 双 head | high_p=0.496(灾难) | ❌ 失败 |
| D Modal | Modality Dropout + Gate 熵 | high_p=0.792, Brier=0.079 | 略好于 baseline |

### 1.2 B Calib 的隐藏问题(本轮新发现)

虽然 B 在多数指标上大胜,但**混淆矩阵分析揭示了一个新问题**:

```
B Test-112 混淆矩阵(行=真实标签,列=预测分档):
                     high   med    low
malignant (114)  →   113    1      0    ✅ 几乎全进 high
benign    (114)  →    3    26     85    ❌ 75% 落入 low (临床错误!)
no_tumor  (228)  →    3    21    204    ✅ 89% 进 low
```

**临床问题**: 良性息肉 (benign polyp) 是肿瘤,理论上应进 medium band 接受随访或择期手术。
B 把 75% 的 benign 推进了 low band(仅随访),会导致**漏诊与延误**。

### 1.3 本轮 3 个新目标

观察混淆矩阵后,本轮实验聚焦于以下 3 个目标:

| 目标 | 期望 | 临床意义 |
|---|---|---|
| 1️⃣ **high band 中尽量不出现 no_tumor** | nt → high ≤ 5% | 避免正常人被推手术 |
| 2️⃣ **low band 中尽量都是 no_tumor** | nt → low ≥ 80% | 真低风险=真无肿瘤 |
| 3️⃣ **low band 中尽量少 benign** | ben → low ≤ 10% | 良性息肉≠"放着不管" |

### 1.4 现状对比表(Test-112,5 个核心指标)

| 指标(理想方向) | Baseline | A | B 🥇 | C | D | 期望底线 |
|---|---:|---:|---:|---:|---:|---:|
| mal → low (=0) | 0 ✅ | 0 ✅ | 0 ✅ | 0 ✅ | 0 ✅ | = 0 |
| high_recall (≥0.95) | 1.000 ✅ | 1.000 ✅ | 0.991 ✅ | 1.000 ✅ | 1.000 ✅ | ≥ 0.95 |
| **no_tumor → high (≤5%)** | 11.0% ❌ | 9.6% ❌ | **1.3%** ✅ | 25.4% ❌ | 6.1% ❌ | ≤ 5% |
| **no_tumor → low (≥80%)** | 63.2% ❌ | 59.6% ❌ | **89.5%** ✅ | 49.6% ❌ | 60.5% ❌ | ≥ 80% |
| **benign → low (≤10%)** | 30.7% ❌ | 27.2% ❌ | **74.6%** ❌❌ | 27.2% ❌ | 32.5% ❌ | ≤ 10% |

**没有任何一个 0502 实验同时满足 5 个底线**。B 在前 4 项是天花板,但第 5 项严重恶化;
其他实验在第 3、4 项又不够好。本轮 4 个实验目标是**让至少 1-2 个同时拿下全部 5 个底线**。

## 2. 共享设计(4 个实验都基于 B 的胜利组合)

为不引入无关变量,4 个实验 **同时保留 B 已验证有效的训练优化**:

| 组件 | 来源 | 配置 |
|---|---|---|
| Backbone | baseline / B | SwinV2-Tiny + 4ch (RGB+gallbladder mask) + Seg-Guided Attention |
| Text encoder | baseline | bert-base-chinese (frozen) + cross-attention |
| Meta encoder | baseline | 10D 临床特征 → MLP |
| Fusion | baseline | Gated Trimodal Fusion → 256D |
| Ordinal target | B | (mal=1.0, ben=0.35, nt=0.0) 非对称 |
| 训练 epoch 上限 | B | 25 |
| 早停 | B | patience=5,monitor=val_AUC |
| 温度校准 | B | post-hoc LBFGS on val,T 应用到 test |
| 数据划分 | baseline | GroupShuffleSplit 病人级 |
| Test 集 | baseline | Test-111(平衡)+ Test-112(no_tumor 翻倍) |

**4 个实验只动以下一项**(差异化的核心改动):

| 实验 | 改动维度 | 改动 |
|---|---|---|
| **E** | + 辅助 head | 加 3-class CE head,决策时按类别概率路由 |
| **F** | 替换 head | 把单 risk head 换成 3 个独立二分类 head |
| **G** | + Loss | 加 Margin Ranking Loss 强化 nt < ben < mal 排序 |
| **H** | 替换 head | 改成 2-stage:tumor/nt 检测 → mal/ben 分流 |

## 3. 实验 E:三分类辅助 head + 类别感知决策

### 3.1 设计动机

B 的 score 分布(Test-112):
- malignant: mean=0.997, [0.973, 0.999]  → 远离 0,易分
- **benign**: mean=0.341, [0.085, **0.790**]  → 范围太宽,与 nt 高度重叠
- **no_tumor**: mean=0.263, [0.058, **0.929**]  → 范围太宽,与 ben 高度重叠

ben/nt 在单一 risk 维度上**本质不可分**。
解决思路:**额外学一个三分类 head**,提供独立的类别概率作为决策路由信号。
即使 risk 低,只要 p_benign 高就强制走 medium。

### 3.2 模型修改

```python
class SwinV2SegGuidedRiskAuxCls(SwinV2SegGuidedRiskTrimodal):
    def __init__(self, ...):
        super().__init__(...)
        self.cls_head = nn.Sequential(
            nn.Linear(256, 128), nn.GELU(), nn.Dropout(0.1),
            nn.Linear(128, 3),  # [no_tumor, benign, malignant]
        )

    def forward(self, ...):
        # 调用父类得到 fused (256D) 和 risk_logit
        fused, risk_logit, seg_logits = super().forward_features(...)
        cls_logits = self.cls_head(fused)  # (B, 3)
        return risk_logit, cls_logits, seg_logits
```

### 3.3 损失

```
L = BCE_with_logits(risk_logit, y_ordinal)              ← B 的非对称 target
  + λ_cls × CrossEntropy(cls_logits, y_3class)          ← 新增,λ_cls=0.5
  + λ_seg × Dice+BCE(seg_logits, mask)
```

### 3.4 决策规则(Inference)

```python
def predict_band(risk, p_nt, p_ben, p_mal,
                 t_low, t_high,         # 来自 B 的双阈值 grid search
                 tau_nt, tau_mal, tau_ben):  # 三类先验阈值
    if p_nt > tau_nt:
        return "low"      # 强 nt 先验,锁定低风险
    if p_mal > tau_mal:
        return "high"     # 强 mal 先验,锁定高风险
    if p_ben > tau_ben:
        return "medium"   # 强 ben 先验,绝不进 low (核心!)
    # 三类都不强,回到 risk 双阈值
    if risk >= t_high:    return "high"
    if risk <  t_low:     return "low"
    return "medium"
```

`tau_nt, tau_mal, tau_ben` 通过 val grid search 确定。
**搜索范围**: `tau_nt, tau_mal, tau_ben ∈ [0.3, 0.7]`,步长 0.05(共 9³ = 729 组候选)。
搜索约束(在 0502 基础上**新增软底线**):
- `mal_to_low = 0`(硬)
- `high_recall ≥ 0.95`(硬)
- `benign_to_low ≤ 5%`(新软底线)
- `no_tumor_to_high ≤ 5%`(新软底线)
**搜索目标**: 满足上述约束的所有组合中,maximize `(high_precision + low_precision_nt) / 2`。
若无解,放宽 `benign_to_low ≤ 10%` 重搜。

### 3.5 期望效果

- **目标 1**(nt→high 低):p_nt 高的样本被 hard route 到 low,即使 risk 偶尔高也不会进 high
- **目标 2**(nt→low 高):同上
- **目标 3**(ben→low 低):p_ben 高的样本被 hard route 到 medium,即使 risk 低也不会进 low

## 4. 实验 F:Triple-Head 直接三档分类

### 4.1 设计动机

B 把 ordinal score 当回归学,**单维度**难以同时锐化 high/med/low 三个边界(本质上是 2 个边界 + 1 个连续 score)。改为**直接学 3 档分类**:每档一个独立的二分类 head。

类比:不让模型先学个 "肿瘤恶性程度评分" 再分档,而是让模型直接学 "这是 high band 吗 / medium 吗 / low 吗"。三个 head 各自有专属的 BCE 损失,各自专注。

### 4.2 模型修改

```python
class SwinV2SegGuidedRiskTripleHead(SwinV2SegGuidedRiskTrimodal):
    def __init__(self, ...):
        super().__init__(...)
        # 替换原 risk_head
        self.head_high = nn.Sequential(nn.Linear(256, 128), nn.GELU(), nn.Linear(128, 1))
        self.head_med  = nn.Sequential(nn.Linear(256, 128), nn.GELU(), nn.Linear(128, 1))
        self.head_low  = nn.Sequential(nn.Linear(256, 128), nn.GELU(), nn.Linear(128, 1))

    def forward(self, ...):
        fused, _, seg_logits = super().forward_features(...)
        return (self.head_high(fused), self.head_med(fused), self.head_low(fused),
                seg_logits)
```

**Label 映射**:
- `y_high = (label == malignant)`
- `y_med  = (label == benign)`
- `y_low  = (label == no_tumor)`

(三个 head 的 label 互斥,刚好对应数据集的三类)

### 4.3 损失

```
L = BCE(p_high, y_high) + BCE(p_med, y_med) + BCE(p_low, y_low)
  + λ_seg × Dice+BCE(seg_logits, mask)
```

每档 BCE 权重均为 1.0,seg_loss 权重沿用 baseline。

### 4.4 决策规则(Inference)

```python
band = ["high", "medium", "low"][argmax(p_high, p_med, p_low)]
```

无需阈值搜索,直接 argmax。

**Fallback**(为保 high_recall ≥ 0.95):
```python
if band != "high" and p_high > tau_safe:
    band = "high"   # 即使不是 argmax 最大,只要 p_high 超过 tau_safe 就保守归 high
```

> **关键**: `tau_safe` **不是固定值**,而是在 val 上反推出来 ——
> 在 `tau_safe ∈ [0.10, 0.50]` 范围内(步长 0.02)找最小的 `tau_safe`,
> 满足 val 上 `high_recall ≥ 0.95` 且 `mal_to_low = 0`。
> 文中示例 `tau_safe ≈ 0.30` 仅供参考,实际值由 grid search 决定。

### 4.5 期望效果

- 三档 head 各自独立学习,边界锐化
- benign 在 head_med 上是正样本、在 head_low 上是负样本 → benign 被显式拉离 low
- no_tumor 在 head_low 上是正样本、在 head_high 上是负样本 → nt 被显式拉离 high

### 4.6 风险

- 失去 ordinal 性质(score 不再连续单调),不利于温度校准
- 因此 F 实验中**温度校准退化为对每个 head 单独做 sigmoid scaling**

## 5. 实验 G:Ordinal Margin Ranking Loss

### 5.1 设计动机

B 的 score 三类分布有显著重叠(参见 §3.1)。本质是:模型学到了 mal vs (ben+nt),
但 **ben vs nt 没学开**(因为 ordinal target 给 ben=0.35、nt=0.0,差距仅 0.35)。

加入 **pairwise margin loss**,显式约束:
```
score(malignant) > score(benign)   + margin
score(benign)    > score(no_tumor) + margin
```
让模型在 score 空间把三类**强行拉开**。

### 5.2 模型

与 baseline 完全相同(只改 loss,backbone 和 head 不动)。

### 5.3 损失

```python
def margin_ranking_loss(score, label, margin=0.2):
    # score: (B,) sigmoid 后的 risk score in [0, 1]
    # label: (B,) {0=nt, 1=ben, 2=mal}
    s_nt  = score[label == 0]
    s_ben = score[label == 1]
    s_mal = score[label == 2]
    
    # 所有 (nt, ben) 对: 要求 s_ben - s_nt > margin
    if len(s_nt) > 0 and len(s_ben) > 0:
        diff_nb = s_ben.unsqueeze(1) - s_nt.unsqueeze(0)  # (n_ben, n_nt)
        loss_nb = F.relu(margin - diff_nb).mean()
    
    # 所有 (ben, mal) 对: 要求 s_mal - s_ben > margin
    if len(s_ben) > 0 and len(s_mal) > 0:
        diff_bm = s_mal.unsqueeze(1) - s_ben.unsqueeze(0)
        loss_bm = F.relu(margin - diff_bm).mean()
    
    return loss_nb + loss_bm  # 缺类时该项为 0
```

总损失:
```
L = BCE(risk_logit, y_ordinal) + λ_rank × L_rank + λ_seg × seg_loss
```

**关键超参** (操作的是 **post-sigmoid score**, 不是 logit):
- `λ_rank = 0.3` (相对 BCE 的权重)
- `margin = 0.2` (post-sigmoid score 在 [0, 1] 区间的距离, 即类间至少差 20%)
- 在 logit 空间约等于差 0.8 (sigmoid' 在 0.5 附近 ≈ 0.25, margin_logit ≈ margin_score / 0.25)

### 5.4 决策规则

同 baseline:双阈值 (t_low, t_high)。
但 grid search 加 **新软底线**:`benign_to_low ≤ 5%`, `no_tumor_to_high ≤ 5%`。

### 5.5 期望效果

- score(ben) 被 push 到至少 score(nt) + 0.2,benign 在 [0.4, 0.6] 区间集聚 → 落入 medium
- score(mal) 被 push 到至少 score(ben) + 0.2,malignant 仍在 [0.7+] → 落入 high
- score(nt) 被 push 到 score(ben) - 0.2,no_tumor 在 [0.0, 0.3] → 落入 low

## 6. 实验 H:Two-Stage Tumor Detector + Risk Head(修正 C)

### 6.1 设计动机

0502 实验 C 的失败:stage1 是 risk score(把 ben 标为 0.5),stage2 是 ben/mal 二分类。
两个 head 高度相关、互相干扰,stage2 AUC 仅 0.66,反而拉乱了 stage1 的决策边界。

H 的修正思路:
- **Stage1 改为检测 tumor vs no_tumor**(这两类在影像上差异最大,易学,AUC 应可达 0.99+)
- **Stage2 在 tumor 上判 mal vs ben**(这两类是 polyp 内部细分,需要精细特征)
- **关键**: stage2 的 BCE 用 `y_tumor.float()` 加权,**只在真 tumor 样本上有梯度**
  (no_tumor 样本对 stage2 不贡献梯度,避免拉偏)

物理意义:stage1 把 nt 干净分离掉(对应目标 1+2),stage2 在 polyp 池内分 mal/ben,
ben 不可能跑到 low band(目标 3)。

### 6.2 模型修改

```python
class SwinV2SegGuidedRiskCascade(SwinV2SegGuidedRiskTrimodal):
    def __init__(self, ...):
        super().__init__(...)
        # 替换原 risk_head
        self.tumor_head = nn.Sequential(nn.Linear(256, 128), nn.GELU(), nn.Linear(128, 1))
        self.mal_head   = nn.Sequential(nn.Linear(256, 128), nn.GELU(), nn.Linear(128, 1))

    def forward(self, ...):
        fused, _, seg_logits = super().forward_features(...)
        return self.tumor_head(fused), self.mal_head(fused), seg_logits
```

### 6.3 Label 映射

```
y_tumor = (label != no_tumor) = {0: nt, 1: ben+mal}
y_mal   = (label == malignant) = {0: nt+ben, 1: mal}
```

### 6.4 损失

```python
loss_tumor = BCE_with_logits(tumor_logit, y_tumor)  # 全样本

mal_loss_per = BCE_with_logits(mal_logit, y_mal, reduction='none')
loss_mal = (mal_loss_per * y_tumor.float()).sum() / (y_tumor.float().sum() + 1e-6)
# ↑ 只在真 tumor 上算,no_tumor 样本对 mal_head 无梯度

L = loss_tumor + λ_mal × loss_mal + λ_seg × seg_loss
```
λ_mal = 1.0(目标:tumor 检测和 mal 检测同等重要)

### 6.5 决策规则(Inference)

```python
def predict_band(p_tumor, p_mal, tau1, tau2):
    if p_tumor < tau1:    return "low"     # 不像 tumor → low (不论 p_mal 多少)
    if p_mal   > tau2:    return "high"    # 像 tumor 且像 mal → high
    return "medium"                          # 像 tumor 但不像 mal → benign → medium
```

`tau1` 通过 val 上 **`tumor_recall ≥ 0.98`** 确定(底线:不能漏 tumor)。
`tau2` 通过 val 上 mal grid search 确定,目标 `high_precision` 最大 + `mal_to_low = 0` + `high_recall ≥ 0.95`。

### 6.6 期望效果

- p_tumor < tau1 的样本绝大多数是 nt → 干净的 low band(目标 2)
- p_tumor > tau1 但 p_mal < tau2 的样本是 ben → 绝大多数进 medium(目标 3)
- nt → high 几乎不可能(目标 1):需要 p_tumor > tau1 且 p_mal > tau2,nt 同时满足两个的概率极低

### 6.7 与 0502 C 的对比

| 维度 | 0502 C | 0510 H |
|---|---|---|
| stage1 任务 | risk score(ben=0.5) | tumor vs no_tumor(ben=1) |
| stage2 任务 | ben vs mal | ben vs mal |
| stage2 监督 | 全样本 BCE | tumor mask 加权(no_tumor 无梯度) |
| 决策 | 双阈值在 risk 上 | (p_tumor, p_mal) 双独立信号 |
| 期望 stage2 AUC | 实测 0.66 ❌ | 0.85+(独立任务,不被 stage1 拉偏) |

## 7. 评估口径(与 0502 一致 + 新增软底线)

### 7.1 数据
- 病人级 GroupShuffleSplit train / val / test
- Test-111: 平衡测试集(339 张,mal:ben:nt = 114:111:114)
- Test-112: no_tumor 翻倍(456 张,mal:ben:nt = 114:114:228)

### 7.2 评估指标(每实验在 Test-111/112 上各跑一遍)

**硬底线**(必须满足,否则该实验失败):
- `mal_to_low = 0`(无恶性漏诊到 low band)
- `high_recall ≥ 0.95`(高风险召回)

**软底线**(本轮新增,对应用户的 3 个目标):
- `no_tumor_to_high ≤ 5%`  (目标 1)
- `no_tumor_to_low  ≥ 80%` (目标 2)
- `benign_to_low    ≤ 10%` (目标 3)

**关键性能指标**:
- `high_precision`(高风险预测准不准)
- `low_precision_nt`(低风险中真 nt 占比)
- `medium_share`(中风险样本占比)
- `Brier score` raw / calibrated
- ROC AUC(binary mal vs others)

### 7.3 阈值搜索目标函数

```
maximize  (high_precision + low_precision_nt) / 2
subject to  硬底线 + 软底线 都满足
```

如无解,放宽 `benign_to_low ≤ 15%` 再搜索一次。
